# AI 에이전트 기초

AI 에이전트는 두뇌(LLM) + 지시서(프롬프트) + 손발(도구)로 구성된 자율적 실행 시스템이다. 단순 질의응답과 달리, 목표를 스스로 분해하고 도구를 호출하며 결과를 반영하는 루프를 반복한다.

## 핵심 내용

### AI 에이전트의 3계층 구조

| 계층 | 구성요소 | 역할 |
|---|---|---|
| 두뇌 | LLM (Claude 등) | 판단, 계획, 생성 |
| 지시서 | 프롬프트, CLAUDE.md, Rules | 행동 범위 및 방향 정의 |
| 손발 | 도구 (파일 편집, 웹 검색, 터미널 실행 등) | 실세계 조작 |

### 자율도 레벨 1-5

| 레벨 | 설명 | 예시 |
|---|---|---|
| 1 | 단순 질의응답 | ChatGPT 기본 사용 |
| 2 | 도구 사용 (단일 호출) | 웹 검색 포함 |
| 3 | 멀티스텝 실행 | 파일 읽기 → 수정 → 저장 |
| 4 | 자율 계획 + 실행 | 작업 분해 후 순차/병렬 처리 |
| 5 | 멀티 에이전트 협업 | 에이전트 간 역할 분담 |

Claude Code는 레벨 4-5에 해당하며, 코드베이스 전체를 탐색하고 수정하는 자율 루프를 실행할 수 있다.

### 에이전트 실행 루프 (ReAct 패턴)

```
Think → Act → Observe → Think → Act → ...
```

LLM이 상황을 추론하고(Think), 도구를 호출하며(Act), 결과를 관찰(Observe)하는 사이클을 반복한다. 사람의 개입 없이 목표 달성까지 수십 번 반복할 수 있다.

## 세부 사항

### MCP (Model Context Protocol)

Anthropic이 제안한 표준 프로토콜. LLM과 외부 도구/데이터 소스를 연결하는 공통 인터페이스.

- **목적**: 도구 연결을 표준화하여 생태계 호환성 확보
- **특징**: JSON-RPC 기반, 서버-클라이언트 구조
- **활용**: Claude Code에서 MCP 서버를 추가하면 데이터베이스, API, 외부 서비스에 접근 가능

### A2A (Agent-to-Agent) 프로토콜

에이전트끼리 통신하는 프로토콜. MCP가 도구 연결이라면 A2A는 에이전트 간 협업을 위한 표준.

- 오케스트레이터 에이전트가 서브에이전트에게 작업 위임
- 서브에이전트가 결과를 반환하여 조합

### 하네스 엔지니어링

에이전트가 안전하고 일관되게 작동하도록 제약과 구조를 설계하는 작업.

**하네스의 6요소 (Claude Code 기준):**

1. **CLAUDE.md** — 자연어 지시서 (가장 약한 강제력)
2. **Rules** — 도구별 허용/거부 규칙
3. **Skills** — 재사용 가능한 작업 템플릿
4. **Hooks** — 이벤트 기반 자동 실행 스크립트
5. **Permissions** — 파일/명령어 접근 범위 제한
6. **MCP** — 외부 도구 연결

> 강제력 순서: 플로우 내장 > Hook > 함수/스크립트 > CLAUDE.md 규칙

하네스 엔지니어링은 프롬프트 엔지니어링과 컨텍스트 엔지니어링 이후의 단계다. 프롬프트가 레시피, 컨텍스트가 재료라면 하네스는 에이전트가 튀는 행동을 하지 않도록 경로와 제동 장치를 만드는 것이다. 실무에서는 "부탁"보다 Hook, Permission, 스크립트처럼 실행을 강제하는 장치를 우선한다.

## 관련 맥락

- AI 에이전트의 등장으로 "프롬프트 엔지니어링"에서 "하네스 엔지니어링"으로 패러다임 이동
- Claude Code는 이 개념의 실전 구현체
- 자율도가 높을수록 인간의 감독(human-in-the-loop) 설계가 중요해짐
