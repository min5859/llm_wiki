# 스코어 알고리즘 두 버전 비교 — 컷오프 캘리브레이션 vs 진짜 알파 변화

스크리너 점수 알고리즘을 V2 → V3 로 바꾼 직후 평균 점수가 17점 하락 (47 → 28) 하여 기존 컷오프 62 통과 종목이 0건이 됐을 때, **이게 "효과 있는 재설계" 인지 "단순 평행이동" 인지** 를 정량적으로 판정하는 방법론.

## 문제 설정

- V2 (기존): 평균 ~47, 최대 ~70, 매일 6~21건이 컷오프 62 통과 → 정상 매수 진입
- V3 (재설계 후): 평균 ~28, 최대 ~51, 매일 0건이 컷오프 62 통과 → 매수 정지
- 표면적 결정: 컷오프를 62 → 48 로 낮추면 V2 와 동등한 진입 빈도 복원

**핵심 질문**: 컷오프만 낮추면 V2 와 동일한가, 아니면 V3 가 의미 있는 알파 변화를 만들었는가?

## 비교 방법론 — 4가지 측정

### 1. 순위 상관 (Spearman ρ) — 가장 중요

동일 종목이 V2 와 V3 에서 어느 정도 같은 순위를 받는지.

```python
# 일별 종목 점수 매칭 (V2 = 직전 측정일, V3 = 첫 적용일)
pairs = match_by_stock(v2_scores, v3_scores)  # n=30

def rank(xs): ...  # 동순위 평균 랭크
def pearson(xs, ys): ...

v2_tot_rank = rank([p.v2_total for p in pairs])
v3_tot_rank = rank([p.v3_total for p in pairs])
rho = pearson(v2_tot_rank, v3_tot_rank)
```

- ρ ≈ 1.0 → 완전 평행이동 (컷오프 조정과 동등)
- ρ ≈ 0.0 → 완전 재정렬 (V3 가 진짜 다른 알고리즘)
- 실측: **ρ = +0.835** (강한 순위 보존, but 일부 reordering 존재)

### 2. Top-N 픽 교집합

각 알고리즘이 상위 N 종목으로 뽑는 후보의 겹침. 매수 후보 결정은 결국 Top-N 의 문제이므로 직접 영향 지표.

```python
v2_top = sorted(pairs, key=lambda p: -p.v2_total)[:10]
v3_top = sorted(pairs, key=lambda p: -p.v3_total)[:10]
overlap = set(v2_top) & set(v3_top)
print(f"Top-10 교집합: {len(overlap)}/10")
```

- 실측: 8/10 (80%) — 컷오프 단순 하향만 해도 매수 후보의 80% 는 동일

### 3. 큰 순위 이동 종목 — V3 효과 식별

```python
v2_rank_map = {p.code: i+1 for i,p in enumerate(sorted_by_v2)}
v3_rank_map = {p.code: i+1 for i,p in enumerate(sorted_by_v3)}
moves = sorted(pairs, key=lambda p: abs(v3_rank_map[p.code] - v2_rank_map[p.code]), reverse=True)[:10]
```

큰 순위 이동 (≥ ±10) 이 발생한 종목들이 V3 의 진짜 효과. 이번 사례:

| 종목 | V2 순위 | V3 순위 | Δ |
|------|---------|---------|---|
| SK하이닉스 | 21 | **5** | -16 (대형주 earnings_yield 보정으로 신규 진입) |
| 대한해운 | 22 | **9** | -13 (기술 0점이지만 펀더로 끌어올림) |
| 가온전선 | 10 | 20 | +10 (MA 가중치 감점으로 탈락) |
| DB하이텍 | 4 | 13 | +9 (MA + 펀더 안정성 동시 페널티) |

이런 reordering 이 V3 가 "단순 평행이동" 이 아닌 증거.

### 4. 컴포넌트별 평균 변화

```
| 컴포넌트 | V2 평균 | V3 평균 | 이론최대 V2 | 이론최대 V3 |
| 기술    | 18~23   | 12~14   | 40         | 50         |
| 펀더    | 12~16   | 7~9     | 40         | 30         |
| 리서치  | 8~10    | 7~9     | 20         | 20         |
```

이론최대 vs 실제 분포의 차이가 "의도와 결과의 괴리". 이번 사례에서 기술 점수의 이론 최대를 40 → 50 으로 올렸음에도 실제 최대는 25 → 20 으로 더 낮아짐 = ATR / 강양봉 / 거래량 sweet 조건이 KOSPI 거래량 상위 종목에서 동시 만족되기 어려운 임계 설정.

## 결론 도출 패턴

| ρ / 교집합 결과 | 해석 | 대응 |
|-----------------|------|------|
| ρ > 0.9, 교집합 ≥ 90% | 사실상 평행이동 | 컷오프만 조정, 알고리즘 변경 ROI 의심 |
| 0.7 < ρ < 0.9, 교집합 70~90% | **평행이동 + 가장자리 reordering** | 컷오프 조정 + 큰 이동 종목의 forward return 추적 (이번 사례) |
| 0.3 < ρ < 0.7 | 부분적 재정렬 | 컷오프 조정 + 컴포넌트별 IC 재검증 권장 |
| ρ < 0.3 | 거의 새 알고리즘 | forward test 필수, 컷오프 조정만으로 의사결정 금지 |

## 백테스트의 한계 — 두 알고리즘 비교에 부적합

V3 검증을 위해 백테스트를 돌리고 싶어지지만, 다음 이유로 거의 도움 안 됨:

1. **이중 점수 스케일**: 백테스트는 40점 만점 `_calc_buy_score(bars)` 만 사용, 라이브 screener 의 100점 (`screen_candidates()`) 과 별도. V3 변경은 100점 쪽이라 백테스트에 미반영
2. **펀더/리서치는 과거 시점 재현 불가**: 분기 데이터 (PER/ROE/EPS) 가 시점별 스냅샷으로 저장 안 됨. V3 변경의 절반 (`earnings_yield` 신규 / 안정성 4점 / EPS 절대값 폐기) 이 모두 펀더 영역
3. **수집 시점 스크리너 결과 미보존**: 매일 라이브 스크리너가 어떤 30종목을 뽑았는지의 스냅샷이 없으면 V2/V3 의 가상 매수 시뮬레이션 불가

→ 진짜 알파 비교는 **forward test** 외 방법 없음:

| 방법 | 가능? | 무엇을 알 수 있나 |
|------|-------|-------------------|
| run_backtest.py 그대로 | ✅ 즉시 | V2 vs V3 비교 불가 (40점 경로) |
| validate_score_ic.py 확장해 V2/V3 기술 점수만 IC 비교 | ✅ 1~2h | 기술 알파 변화는 보임. 펀더 효과 불가 |
| **Forward test**: 매일 점수 + N일 후 수익률 로깅 | ✅ 2~4주 | V3 전체 평가, 가장 신뢰 |
| 100점 스크리너 전체 과거 재현 | ❌ | (수집 스냅샷 부재) |

## 컷오프 임시 캘리브레이션 패턴

V3 분포에서 "과거 ≥62 통과 빈도 (6~21건/일)" 와 동등한 빈도를 회복하는 컷오프 계산:

1. V3 분포의 일별 점수 히스토그램 산출 (5/10~5/11 등 첫 며칠)
2. 일평균 5~10건 정도 통과하는 컷오프 찾기 → V3 에서는 ≥48
3. yaml 두 파일 (`config/strategies/scoring.yaml`: `buy_min_score_full`, `config/trading.yaml`: `min_score`) 모두 수정
4. 커밋 메시지에 명시: **"forward return 으로 검증된 컷오프가 아닌 임시 캘리브레이션. 2~4주 라이브 데이터 누적 후 V3 점수 vs 실제 수익률 평가 필요"**

## 일반 원칙

1. **알고리즘 교체 후 컷오프 미조정 = 매수 정지 사고** — 분포 평균이 바뀌면 그 자체로 컷오프 통과 빈도가 0 이 될 수 있음. 점수 변경 PR 에는 반드시 분포 시뮬레이션을 첨부
2. **Spearman ρ + Top-N 교집합** 두 지표가 비교의 핵심 — Pearson 만으로는 절대값 평행이동을 reordering 으로 오인 가능
3. **백테스트 ≠ 알고리즘 비교** — 백테스트가 작동하지 않는 점수 경로는 시작 전에 식별. forward test 인프라가 진짜 정답
4. **forward 로깅을 인프라로 구축** — 매일 매수 후보 + 점수 + entry price + N일 후 종가 를 단일 로그에 누적. 알고리즘 변경 의사결정의 표준 비교 자료
5. **컷오프 캘리브레이션은 임시 조치** — "분포 정상화" 만으로 끝내지 말 것. 정상화 후 forward return 으로 V3 가 V2 보다 진짜 나은지 검증

## 안티 패턴

- **점수 알고리즘 교체 후 매수 정지 → "잘못된 캘리브레이션이니 컷오프만 낮춤" 으로 종결** — V3 의 진짜 알파 효과를 검증 안 한 채 알고리즘만 갈아입힘. 컷오프 조정과 동등한지가 사실상 미확인
- **순위 비교 없이 컴포넌트 평균만 본다** — "기술 평균이 18 → 13 이 됐다" 만으로 "V3 가 더 보수적" 결론. 순위가 거의 같으면 의미 변화 없음
- **Pearson 만으로 비교** — 평행이동에 대한 분별력 없음. 반드시 Spearman 병행
- **단일 일자 비교** — 30 종목 × 1일 = 30 표본은 부족. 최소 며칠 누적
