# 스코어 시스템의 IC 검증 방법론

룰베이스 트레이딩 시스템의 종목 선별 점수 (스크리너 점수, 시그널 점수) 가 실제로 알파를 가지는지 정량 검증하는 절차. 직관·관습으로 짜인 가중치 조합이 한국 시장에서 어떻게 무너지는지 보여 주고, 검증 후 재설계의 표준 형식을 정리한다.

## IC (Information Coefficient) 정의

> IC = 점수와 N일 후 forward return 사이의 상관계수.

- **Pearson IC** — 선형 상관 (0 부근이면 알파 없음)
- **Spearman IC** — 순위 상관 (스코어 절대값보다 *순위* 가 의미 있을 때 사용)
- **컴포넌트별 IC** — 합산 점수가 아닌 *가중 전 sub-score* 별로 측정. 어느 인자가 알파를 만드는지 분해
- **구간별 평균 수익률·승률** — 점수 구간을 5/10/15 단위로 끊어, 각 bucket 의 forward return 평균과 승률 분포 (cutoff 캘리브레이션 근거)
- **Sharpe (간이)** — 평균 / 표준편차 (연환산 미적용, 비교용 지표)

## 표본 설계의 5가지 결정

| 항목 | 결정 |
|------|------|
| Universe 크기 | 단일 종목으로는 표본 부족 → 25종목 × 2년 = 약 2,300 표본 권장 |
| Forward horizon | 5/10/20일 다중 측정 — horizon 이 길수록 노이즈 ↓ 알파 ↓ |
| Sample 주기 | sample_every (예: 5일) — 일별 표본은 자기 상관이 강함 |
| 점수 재현성 | 일봉 OHLCV 만으로 재현 가능한 인자만 IC 측정 가능 (펀더는 분기 데이터, 과거 시점 재현 어려움) |
| 벤치마크 | 상대 수익률 비교 시 KODEX200 (`069500`) 일봉 페어링 |

## 한국 시장에서 검증된 IC 분포 (2026-05-10, n=2,362)

샘플 케이스. 직관적 인자 일부가 *음수 IC* 를 보였다.

### 5일 forward return 기준 컴포넌트별 IC

| 컴포넌트 | IC | 평가 |
|----------|-----|------|
| 양봉 (candle, ≥2% body) | **+0.0387** | 가장 강한 알파 |
| 거래량 비율 (sweet 2~4배) | **+0.0287** | 두 번째 알파 |
| 신고가 근접 (95~99%) | +0.0144 | 약함 (예상보다 낮음) |
| MA 정렬 (sma5/20/60) | -0.0031 | **사실상 0** |
| 단기 모멘텀 5일 상대수익률 | **-0.0383** | **음수! mean reversion 압력** |
| RSI 안정구간 45~65 | -0.0201 | 음수 |

### 핵심 발견 (한국 시장 특이성)

- **단기 모멘텀이 mean reversion 으로 작동**: 5일 상대수익률이 높은 종목은 다음 5일에 평균 손실. 미국 모멘텀 팩터의 한국 역작용
- **RSI 안정구간 진입은 역효과**: "과매수도 과매도도 아닌 안전 구간" 직관이 실제로는 통하지 않음
- **신고가 95~99% 만점도 약함**: "신고가 직전 진입" 휴리스틱이 IC 검증 결과에서는 미미한 알파만 제공
- **단일 룰 OR (`강양봉 ≥2% AND 거래량 2~4배`)** 이 합산 점수보다 우월: n=64, 승률 67.2%, 평균 +3.38% — *복합 점수보다 단순 강신호 OR* 이 실전에서 더 강한 경우가 있다

## cutoff 시뮬레이션 표 형식

각 cutoff 임계값 위 표본의 forward return 분포로 cutoff 의 통계적 근거를 산출.

| cutoff | n | 평균 | 승률 | Sharpe* |
|--------|---|------|------|---------|
| 25 (현 라이브) | 310 | +1.60% | 55.2% | 0.192 |
| 30 | 91 | +2.71% | 62.6% | 0.287 |

→ cutoff 5점 상향만으로 평균 +69%, 승률 +7%p 개선 가능. 단, 표본 1/3 로 감소 (진입 기회 ↓).

## AND vs OR 게이팅

여러 점수 체계 (Legacy, V3 등) 가 공존할 때 어떻게 결합할지의 비교:

```
=== AND 게이팅: Legacy>=L AND V3>=V (5일 수익률) ===
 L cut  V cut   n     평균    승률   Sharpe*
   30     25    88   +2.50%  61.4%  +0.264   ← 둘 다 통과 시 최강
   30     30    77   +2.39%  58.4%  +0.242

=== OR 게이팅 ===
 L>=30 OR V>=30: n=147 avg=+2.01% win=60.5%   ← 표본은 늘지만 알파 ↓
```

> **AND 게이팅** 은 보수적 (둘 다 통과해야 매수) → 알파 ↑ 빈도 ↓
> **OR 게이팅** 은 적극적 → 빈도 ↑ 알파 ↓
> 자본/거래 빈도 제약에 따라 선택

## Regime 안정성 검증

분기별로 같은 cutoff 의 알파를 측정하여 시장 regime 에 따른 안정성 확인.

```
=== 분기별 Legacy>=30 의 5일 평균수익률 ===
  2025-Q4: n= 17/300 알파=+5.62% 승률 82.4%   ← 강세장에서 강함
  2025-Q3: n= 16/325 알파=-1.87% 승률 50.0%   ← 약세장에서 약함
  2026-Q1: n= 28/300 알파=+1.38% 승률 67.9%
```

표본 충분 (≥10) 한 분기가 지속적 양의 알파를 보이면 regime-robust. 한 분기라도 음수면 cutoff 단독으로는 충분치 않음 → 시장 regime 필터 (KODEX200 200일선, VIX 등) 추가 검토.

## 검증 → 재설계의 표준 절차

1. **검증 스크립트 작성** — 점수 알고리즘과 분리된 *사본* 으로 IC 측정. 운영 코드 미변경 상태에서 안전하게 다양한 가중치 실험
2. **컴포넌트별 IC 측정** — 음수 IC 또는 0 IC 인 인자 식별
3. **단일 룰 OR 시그널 탐색** — 합산 점수보다 강한 단순 룰이 있는지 확인
4. **재설계안의 IC 측정** — 운영 적용 전 동일 스크립트로 재검증
5. **cutoff 재캘리브레이션** — 점수 분포가 바뀌었으므로 임계값도 다시 산출
6. **forward 로깅 인프라** — 운영 코드에 점수 + 종목 + N일 후 수익률 누적 로깅 → 1~2개월 후 in-vivo IC 측정

## 검증 못 한 인자 — 펀더멘탈

분기 보고서 데이터 (PER/ROE/EPS) 는 과거 시점 재현이 어려워 IC 측정 불가. 따라서 펀더 컴포넌트는 *학술적 결함* (예: EPS 절대값 = 고가/대형주 보너스) 만 즉시 수정하고, 가중치 재배치는 forward 로깅 후 결정하는 보수적 경로가 맞다.

## 적용 우선순위 (검증 결과 → 운영)

| 우선순위 | 조치 | 효과 추정 | 적용 비용 |
|----------|------|-----------|-----------|
| 1 | yaml `min_score` 1줄 상향 (cutoff 25→30) | 평균 +1.1%p, 승률 +7.4%p | 1줄 |
| 2 | 골든시그널 OR 채널 (`강양봉 AND 거래량 sweet`) | 별도 진입 채널, 승률 67.2% | strategy 1 함수 |
| 3 | 컴포넌트 가중치 재배치 (V3) | 점수 분포 정상화 | 점수 함수 + 회귀 테스트 |

## 안티 패턴

- **검증 없이 가중치 직관 조정** — 5/8 분석에서 "RSI 45~65 안정구간" / "5일 단기 모멘텀" 같은 신호 제안이 IC 검증으로 모두 폐기됨. 검증 없이 적용했으면 운영 알파 -3.8% / -2.0% 손실
- **합산 점수만 측정** — 컴포넌트별 IC 분해를 안 하면 어느 인자가 알파를 만드는지 모르고 그대로 가중치를 키움
- **single-asset 검증** — 1~3종목 dry-run 결과를 신뢰. 표본 부족 + survivorship bias. 25종목 × 2년 정도가 최소 권장
- **forward horizon 단일 측정** — 5일만 측정하고 끝. 10일/20일에서 부호가 뒤집히면 알파가 아니라 일중 변동성
