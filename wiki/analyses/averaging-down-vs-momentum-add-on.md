# 물타기 vs 모멘텀 추가매수

분할 매수(3분할 DCA)에서 **남은 분할을 채우는 게이트**를 어떤 신호로 막을지는, 그 전략의 **진입 철학과 일관**돼야 한다. ht_trading 의 분할 폴백 게이트가 진입 결정과 정반대 철학으로 짜여 있어 "펀더로 산 종목이 조정받으면 분할이 무음 정체"되던 사례에서 도출한 일반 교훈.

## 두 가지 추가매수 철학

| | 물타기 (averaging-down / DCA) | 모멘텀 추가매수 (pyramiding) |
|---|---|---|
| 추가 시점 | 가격이 **빠졌을 때** (평단 낮추기) | 가격이 **오를 때** (강세 확인 후 증액) |
| 신호 | 펀더멘털 유지 / 드로다운 한도 내 | 추세·신고가·거래량·양봉 |
| 위험 | "떨어지는 칼날" 물타기 → 손실 확대 | 고점 추격 → 평단 상승 |
| 가드 | 드로다운 상한 + 반등 확인 | 모멘텀 식으면 중단 |

둘은 **정반대 방향의 행위**다. "분할 매수가 왜 안 채워지나"라는 기대는 보통 물타기(평단 낮추기)를 가정하는데, 게이트가 모멘텀 기준이면 *조정 구간마다 분할이 멈춰* 기대와 어긋난다.

## 안티패턴 — 진입과 추가매수의 철학 불일치

ht_trading 의 실제 사례:

- **진입**은 n_stock_info **0/80/20** 점수로 결정 — 기술(모멘텀)을 총점에서 **완전히 제거**(예측력 없다고 판단).
- 그런데 보유 종목이 당일 스크리너 후보에 없을 때 **남은 분할은 순수 40점 기술 점수(≥25)로 게이트**.
- 결과: *살 때는 기술 무시, 추가매수는 기술만 봄.* 펀더로 산 종목(SK스퀘어)이 모멘텀 식자 기술 22/40 < 25 로 2·3분할이 **무음 정체**.

추가로 드러난 결합 문제:

1. **컷오프 25 가 "백테스트용"으로 라벨된 값**인데 라이브 폴백 게이트로 전용 — 라이브 진입컷(62)과 별개의 문서화 안 된 숫자.
2. **모멘텀 게이트 중복** — 메인 경로엔 이미 `_passes_trend_gate` 가 있는데 폴백이 40점 점수로 비슷한 걸 또 함(파라미터만 다른 이중 게이트).
3. **무음** — 분할이 막혀도 알림이 없어 사용자가 "왜 안 채워지나"를 모름.

## 선택지와 결정

의도(물타기냐 모멘텀이냐)에 따라 셋 중 하나:

- **(A) 물타기/소신 averaging 이 목적이면:** 보유 + 리스크 한도 내라면 남은 분할은 간격 throttle 로 그냥 채우고 **기술 재게이트 제거**(드로다운 가드만 유지).
- **(B) 모멘텀 확인 후 추가가 목적이면(기존 동작 유지):** 게이트를 라이브 철학과 정렬(40점 기술 대신 같은 100점/펀더 기준 재채점) + 컷을 명시적 라이브 파라미터로 승격 + 정체 시 알림 1회.
- **(C) 하이브리드:** 재선정(≥62) **또는** 펀더 유지면 허용, 기술은 tie-breaker.

ht_trading 은 0/80/20(기술 폐기) 결정과의 일관성을 위해 **(A)** 채택 — 폴백 기술 게이트를 제거하고 분할 간격·드로다운 가드(`_can_add_split`: 18시간 + 평단 -5% 초과 시 반등+3%·회복 요구)와 수량>0 만 통과 조건으로(commit `dd3d1b7`). 단 **독립 40점 모드**(스크리너 미사용·백테스트)와 드로다운 회복 판정의 점수 사용은 그대로 둬 `buy_min_score` 가 거기선 여전히 유효.

## 일반 교훈

- **진입 결정과 추가매수 결정은 같은 철학을 공유해야 한다.** "기술을 버린다"고 정했으면 추가매수 게이트도 기술을 버려야지, 한쪽만 다르면 의도치 않은 정체/왜곡이 생긴다.
- **백테스트 라벨이 붙은 파라미터를 라이브 경로에 전용하지 말라.** 의미가 다른 두 컷을 한 값으로 묶으면 추적이 어렵다.
- **같은 위험을 두 게이트가 중복으로 막고 있지 않은지 본다.** 추세 게이트 + 점수 게이트처럼 비슷한 걸 파라미터만 달리해 두 번 하면 한쪽은 보통 잉여이거나 모순.
- **조용한 보류는 가시화한다.** 분할/주문이 게이트로 막히면 사후에라도 1회 알린다([[notification-dedup-throttle]]).
