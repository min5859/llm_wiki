## 개요

LLM 에 뉴스 코퍼스를 주고 시장 전망을 묻는 워크플로 ([[news-driven-market-signal-framework]]) 의 결과물을 의사결정 입력으로 쓰기 전에 인지해야 할 6가지 함정. 결과는 그럴듯하게 구조화돼 보이지만 다수가 검증 불가능한 직관에 기반한다. 시장 예측 결과는 wiki 에 영속화하지 않고, 본 함정 목록과 방법론만 영속화한다.

## 함정 1 — 데이터 검증 결여

LLM 은 뉴스 텍스트만 보고 응답한다. 동일 시점의 *가격·거래대금·옵션·VIX 실시간 데이터로 cross-check 할 수단 없음*. 「외국인 6조 순매도」 같은 사실은 뉴스가 그렇게 보도했기 때문에 그렇다고 답변할 뿐, 실제 거래소 데이터로 확인하지 않는다.

→ 답변에 **「수치 검증 안 됨」 면책 강제**. tool 사용 가능한 환경이라면 핵심 수치는 외부 API 로 cross-check 후 제시.

## 함정 2 — 확률 가중의 위험

「베이스 50% / 중립 30% / 리스크 20%」 같은 확률 추정은 **실제 통계 모델이 아닌 LLM 의 직관**. 동일 입력으로 다시 물어도 ±10%p 변동 가능. 의사결정 입력으로 쓸 때 정량 모델의 산출물로 오해하기 쉽다.

→ 답변에 「**비통계적 추정** — 정량적 검증 없음」 라벨 부착. 또는 확률 표기 자체를 제거하고 「가능성 높음/중간/낮음」 정성 라벨로 다운그레이드.

## 함정 3 — 확증 편향 위험 (Selection Bias)

사용자가 수집한 뉴스 코퍼스 자체가 *수집 채널·키워드·언어*에 의존하는 selection bias 를 가진다. 예:

- 한국어 뉴스만 → 해외 카운터 시그널 누락
- 「AI 반도체」 키워드 중심 → 해당 섹터 비중 과대 표상
- 특정 매체 (예: 투자 전문지) 중심 → 톤 편향

LLM 은 입력 외 시장 정보의 *부재* 를 인지하지 못함. **「본 코퍼스에 누락됐을 정보」를 명시적으로 묻는 단계** 추가 필요.

## 함정 4 — Stale 속도

시장 예측 결과물은 발표 직후부터 stale 되기 시작 — 새 헤드라인 1건이 나오면 이전 분석의 전제가 무너진다. **결론 (전망값) 을 wiki 에 영속화하는 것은 부적절**.

영속화 가치 있음:

- 방법론 / 프레임워크 ([[news-driven-market-signal-framework]])
- 함정 목록 (본 페이지)
- 뉴스 → 영향 매핑 휴리스틱 (수년 단위 안정)

영속화 부적절:

- 「코스피 7,500 ~ 8,000 박스권 예상」
- 「반도체 비중 확대 권장」
- 「외국인 6조 순매도 = 차익실현」
- 특정 종목 / 섹터의 매수/매도 시점 권고

## 함정 5 — 추천 행동 회피의 회색지대

「분할 매수 + 헷지 병용」 같은 전략 제안은 **투자 자문 영역에 인접**. 한국에서는 자본시장법상 무인가 투자자문은 위반. LLM 출력의 책임 한계 명시 필요:

- 「개인 학습 / 분석 보조」 명시
- 「투자 결정·손실은 사용자 책임」 면책
- 종목 단위 권고 회피 (섹터/시각만)

## 함정 6 — 단정적 인과관계 추론

「외국인 6조 순매도 = 차익실현」 같은 단일 해석은 **다른 가설을 배제**. 실제 가능한 원인:

- 차익실현 (단기 차익 확보)
- 포트폴리오 리밸런싱 (분기말 / 월말)
- 환율 헷지 (달러 강세 시 한국 비중 축소)
- 특정 종목 이벤트 (대형주 단일 매도)
- 매크로 펀드의 거시 베팅 변경

LLM 은 가장 그럴듯한 단일 가설로 수렴하는 경향. **대안 가설 N개 + 각 가설의 검증 데이터 요구사항** 형태로 답변하도록 prompt 강제 가능.

## 결론

- LLM 은 뉴스 코퍼스에 대한 **그럴듯한 구조화** 에 강하지만 **시장 예측 자체는 약함**
- 결과물의 stale 속도가 빨라 wiki 에 결과를 영속화하면 오히려 오정보 축적
- 6가지 함정 (검증 결여 / 확률 직관 / selection bias / stale / 자문 회색지대 / 단일 인과 추론) 을 답변마다 명시적으로 짚어 사용자가 결과물의 신뢰도를 보정하게 할 것
- 검증 가능한 수치 비교 (시계열 가격 / 외인 수급 / 옵션 변동성) 는 별도 데이터 파이프라인으로 검증한 뒤 LLM 에 사실로 주입하는 것이 정공법
