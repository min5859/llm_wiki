# 전략 탐색의 4가지 선결 조건 — "최적을 찾았다"를 판정하기 전에

"최적의 자동매매 알고리즘을 찾는다"는 목표를 실행 항목으로 쪼개기 전에 갖춰야 할 **구조적 선결 조건 4가지**. ht_dde 페이퍼 트레이딩 운영(약 14거래일 누적) 위에서 목표 지향 검토로 정리 (2026-07-04). 개별 실행보다 상위의 방법론이며, 이것이 없으면 어떤 실험 결과로도 "찾았다/못 찾았다"를 판정할 수 없다.

## 1. 목적함수(objective function)를 먼저 정의한다

절대 손익만 보면 판정 불가. 최소 구성:

- **벤치마크 대비 초과수익** (시장이 -0.4% 인 날의 -0.2% 는 이긴 것)
- **비용 차감 후** (수수료·세금·슬리피지 — 회전율이 높으면 비용이 지배항)
- **리스크 조정** (MDD / 샤프 — 같은 수익이라도 경로가 다르면 다른 전략)

## 2. 단일 레짐 함정 — 검증 구간의 시장 성격을 명시한다

검증 데이터 14거래일이 전부 하락장이면, 거기서 나온 모든 결론("장중 롱 엣지 전무" 포함)은 **하락장 한정**이다. 상승/횡보 구간 누적 또는 분봉 정식 백테스트로 레짐을 확장하기 전에는 어떤 전략도 채택·기각할 수 없다. (봉 간격에 따른 신호 역전은 [[backtest-timeframe-sensitivity]] 참조.)

## 3. counterfactual 평가 하네스 — 라벨과 실현손익은 다르다

포워드 라벨(예: "5일 보유 시 -8%")과 **실제 청산 규칙을 통과한 실현손익**은 부호까지 다를 수 있다 (트레일링·손절이 중간에 자르므로). 신호 평가와 전략 평가를 잇는 counterfactual 하네스("이 규칙이었다면 어디서 잘렸나") 없이는 검증 루프 전체의 판정이 흔들린다. (같은 진입신호가 보유기간에 따라 손실↔초과수익으로 뒤집히는 실측은 [[holding-period-signal-mismatch]].)

## 4. 탐색 방향 — gross 알파 ≈ 0 공간에서는 비용·회전 통제가 우선

이 프로젝트에서 검증된 최대 개선(전량 청산 후 재진입 금지 → 손실 68% 축소)은 **신호 개선이 아니라 비용·회전 통제**였다. 단일 지표 예측력이 사실상 0인 공간에서는 새 진입 신호 발굴보다 회전 억제·악성 신호 회피(급등 추격 차단)·보유기간 연장이 기대값이 높다. "손실의 91~92%가 거래비용"이라는 회계가 이 방향을 가리킨다 ([[ht-dde]] 거래비용 드래그 판단).

## 사용법 — 체크리스트

- [ ] 초과수익·비용 차감·리스크 조정이 포함된 목적함수를 문서로 못박았는가?
- [ ] 검증 구간의 레짐(상승/하락/횡보)을 명시했고, 단일 레짐 결론임을 표시했는가?
- [ ] 라벨 기반 신호 평가와 청산 규칙 통과 후 평가를 구분할 하네스가 있는가?
- [ ] 신호 탐색 전에 비용·회전 통제 여지를 먼저 소진했는가?
