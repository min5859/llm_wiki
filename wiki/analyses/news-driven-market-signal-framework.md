## 개요

수집된 다일자 뉴스 코퍼스를 LLM 으로 분석해 시장 시그널을 추출할 때, 일관된 출력 구조를 강제하고 데이터 분량을 분할하는 7-축 + 3-시나리오 프레임워크. 일자별 병렬 서브에이전트로 분해 후 종합 단계에서 통합. 메타 지식 / 방법론 자체가 본 페이지의 가치이며, 시장 예측 결과물은 빠르게 stale 되므로 결과는 영속화하지 않는다.

## 데이터 분해 전략

수집 대상 (예: 3일 × 20분 단위 ≈ 216 샘플, 합계 약 500KB) 이 단일 LLM 컨텍스트로 처리하기엔 큰 경우:

```
[뉴스 코퍼스 N일치]
       ↓
   분해: 일자별 병렬 서브에이전트 N개 spawn
       ↓
   각 에이전트가 동일한 7-축 템플릿으로 출력
       ↓
   종합: 추세 표 + 3-시나리오 + 섹터별 비중 + 체크포인트
```

장점:

- 누락 방지 (단일 컨텍스트는 후반 데이터를 압축 / 누락 경향)
- 재현성 (각 에이전트 출력이 같은 schema → diff 비교 가능)
- 병렬 시간 단축 (3 일치 = ~80초/agent × 3 병렬 ≈ 90초)

## 7-축 분류 템플릿

각 일자 분석 에이전트가 동일하게 출력해야 할 7개 카테고리:

| 축 | 포함 항목 |
|---|---|
| 1. 매크로 환경 | 금리·고용·유가·물가·중앙은행 결정 |
| 2. 지정학·정책 리스크 | 분쟁·제재·대선·통상·규제 |
| 3. 시장 흐름 | 지수 (KOSPI, S&P500, NASDAQ)·외국인 수급·환율 |
| 4. 섹터 동향 | AI·반도체·금융·헬스케어·에너지·해운 등 |
| 5. 주요 기업 이벤트 | 어닝·M&A·IPO·생산 차질 |
| 6. 수급·심리 | 공매도 잔고·VIX·외인 순매수·옵션 |
| 7. 시장 톤 | 낙관·중립·비관 종합 라벨 |

이 7축은 시장 영향 시그널을 빠짐없이 매핑하기 위한 점검표 역할. 카테고리 비어 있어도 **명시적으로 「해당 일자에 매크로 신규 사항 없음」 으로 적어 누락 vs 부재를 구분**.

## 종합 산출물 형식

### 1. 일자별 추세 표

| 축 | Day 1 | Day 2 | Day 3 | 추세 |
|---|---|---|---|---|
| 매크로 | … | … | … | ↑/→/↓ |
| 시장 톤 | 낙관 | 중립+ | 중립- | ↓ |
| 외국인 수급 | … | … | … | … |

### 2. 확률 가중 3-시나리오

| 시나리오 | 확률 | 트리거 조건 | 예상 결과 |
|---|---|---|---|
| Base (현 추세 지속) | 50% | … | … |
| Neutral | 30% | … | … |
| Risk (하방) | 20% | … | … |

⚠️ 확률 추정은 **LLM 의 직관** — 통계 모델이 아님. 정량 근거 없이 의사결정 입력으로 쓰지 말 것 (→ [[llm-news-prediction-pitfalls]]).

### 3. 섹터별 비중 시각

| 섹터 | 시각 (비중 확대 / 중립 / 축소) | 근거 |
|---|---|---|
| 반도체 | 확대 | AI 칩 어닝 강세, HBM 수요 |
| 정유·에너지 | 중립 | 호르무즈 변동성 양방향 |
| 금융 | 축소 | 금리 동결 시그널 |

### 4. 단기 체크포인트

| D-day | 이벤트 | 영향 가능성 |
|---|---|---|
| D+1 | FOMC | 금리민감주 변동 |
| D+3 | 주요 기업 어닝 | 단일 종목 갭 |

## 뉴스 카테고리 → 시장 영향 매핑 (관찰된 일반 패턴)

| 뉴스 종류 | 자주 매핑되는 시장 영향 |
|---|---|
| 중동 지정학 (호르무즈 봉쇄 등) | 유가 상승 → 정유 강세 / 해운·항공 약세 (양방향) |
| AI 칩 어닝·IPO | 반도체·HBM·AI 인프라 강세, 관련 ETF |
| 외국인 대규모 순매도 + 공매도 잔고 사상 최대 | 과열 차익실현 신호 (단기 조정 가능) |
| 중앙은행 금리 결정 | 금리민감주 (금융·유틸리티)·달러 인덱스·채권 수익률 |
| 고용지표 (NFP, 실업률) | 인플레 기대치 → 금리 경로 → 성장주/가치주 회전 |
| 지수 사상 최고 + 시총 글로벌 순위 상승 | 단기 과열 경계, 차익실현 트리거 |
| 대형 IT 어닝 (어닝 서프라이즈) | 동일 섹터 동조, 협력사 파급 |

이 매핑은 **휴리스틱이며 보장 없음**. 같은 뉴스가 시장 컨텍스트에 따라 반대 방향으로 작동 가능 (예: 금리 인하 = 일반적으론 호재이나, 경기 침체 우려가 더 크면 악재).

## 트레이딩 시스템 통합 시 권장사항

[[ht-trading]] 같은 알고리즘 트레이딩 시스템에 뉴스 시그널을 통합한다면:

- **검증 레이어 필수**: 뉴스 → 시그널 변환 후 동일 시점의 가격·거래량·옵션 데이터로 cross-validation. 미검증 시 false signal 다발
- **Intraday 활용**: 20분 단위 수집의 가치는 단순 일별 합산이 아니라 *특정 헤드라인 직후 N분 가격 반응* 추출에 있음
- **정량 임계값 코드화**: 외국인 순매도 규모, 공매도 잔고, 시총 순위 변화 등 정성적 지표를 정량 임계값 + 이벤트 트리거로 자동화
- **체크포인트 캘린더 모듈화**: D-day 이벤트 캘린더는 트레이딩 시스템의 별도 모듈로 자동화 가치 있음 (사용자 알람 + 포지션 사이즈 조정 트리거)

## 결론

- 다일자 뉴스 분석은 단일 LLM call 보다 **일자별 병렬 서브에이전트 + 동일 템플릿** 이 누락 방지·재현성에서 우수
- 7-축 템플릿 + 3-시나리오 + 섹터 비중 + 체크포인트의 4-블록 형식으로 출력 강제
- 매핑 휴리스틱은 *시작점* 일 뿐, 시장 컨텍스트로 항상 재해석 필요
- LLM 의 시장 예측 결과 자체는 빠르게 stale → wiki 에는 **방법론만** 영속화 (→ [[llm-news-prediction-pitfalls]])
