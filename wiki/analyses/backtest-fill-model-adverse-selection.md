# 백테스트 체결 모델과 역선택 — 즉시체결 가정이 은폐하는 것

백테스트의 "지정가 주문은 무조건 즉시 체결" 가정은 단순한 낙관 편향이 아니라, **진입 규칙에 내재한 역선택(adverse selection)을 통째로 은폐**한다. ht_trading 백테스트 신뢰도 검토(2026-07-04)에서 낙관 편향 2개를 순차 제거하며 실측한 결과.

## 1. 슬리피지 dead code (commit ce9724a)

`_calc_fill_price` 가 `order.price` 유무로 슬리피지 적용을 분기하는데, `Strategy.buy/sell` 이 항상 종가로 price 를 채워 **슬리피지 경로가 도달 불가** → 모든 체결이 종가·슬리피지 0. 라이브의 MARKET 매도(ord_dvsn=01)는 `order.price` 를 무시하므로 백테스트가 청산에서 체계적으로 낙관적이었다. 분기 기준을 price 유무 → `order_type` 으로 수정, 기준가도 open → close (신호가 종가에 생성되므로).

## 2. 즉시체결 → 대기 큐 모델이 드러낸 역선택 (commit bbf8fff, 890d860)

지정가를 대기 큐로 변경 (매수금 예약 → 다음 봉 시가가 유리하면 시가, 저가가 닿으면 지정가 체결, 미도달 시 만료 환불) 하자:

- 수익률 **+18.13% → -2.54%**, MDD 24.7% → 43.2%.
- 메커니즘 — **지정가 = 현재가(ratio 1.0)는 "가격이 내려와야만" 체결**된다. 하락 진입(신호 붕괴)은 다 잡히고, 갭업 승자는 놓친다. 모멘텀 전략에서 현재가 미만 대기 호가는 **"공짜 옵션 매도"** 와 같다.
- `limit_price_ratio` 스윕(일봉): `1.0 = -2.5% / 1.005 = +14.7% / 0.99 = +28%` (V자). **+0.5% 프리미엄은 "가격 상한 있는 시장가"처럼 동작**해 역선택을 해소한다.
- 부수 버그: 신규 포지션이 `current_price=0` 으로 생성돼 체결 직후 -100% pnl 로 같은 봉에서 절대손절 오발동.

## 3. 일봉 fill-window 아티팩트 — 스윕의 0.99=+28% 는 가짜 (commit 314082b)

일봉 스윕에서 0.99 (현재가 -1% 대기) 가 최고로 보이지만, 이는 **주문이 하루 종일 살아있다는 일봉 가정** 때문이다. 라이브는 ~26분 창에서 만료된다. 장중 스냅샷 리플레이(11거래일·128체인)로 검증:

```
체결률:  0.99 = 55.5%  /  0.995 = 77.3%  /  1.0·1.005 = 100%
미체결 체인의 당일 종가: +1.1~1.7%   ← 놓친 것이 승자 (역선택 실증)
```

**결론: 일봉 백테스트로는 "짧은 창에서 현재가 미만 지정가가 체결되는가"라는 질문에 답할 수 없다.** 봉 간격 민감도([[backtest-timeframe-sensitivity]])와 다른 축의, 체결 모델 자체의 해상도 한계.

## 일반 원칙

1. **즉시체결/100% 체결 가정은 낙관 편향을 넘어 역선택을 은폐한다** — 어떤 주문이 체결되고 어떤 주문이 만료되는지가 곧 표본 선택이다.
2. **지정가 프리미엄/디스카운트는 체결률과 함께 스윕**해야 한다 — 수익률만 보면 "하루 종일 살아있는 주문" 아티팩트에 속는다.
3. **체결 모델을 바꾸면 과거 파라미터 결정의 근거가 무효**가 된다 — ht_trading 의 `limit_price_ratio` 반전 이력(1.0→1.005→0.995→1.0)은 전부 즉시체결 구모델 하의 결정이라, 신모델 통찰로 1.005 재채택.
