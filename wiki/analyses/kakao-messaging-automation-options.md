# 카카오톡 자동화 3 옵션: Kakao Developers API / LOCO / 하이브리드

개인이 카카오톡을 프로그램과 직접 통신시키려 할 때 선택지는 (1) 정식 Kakao Developers API, (2) 비공식 LOCO 프로토콜 클라이언트, (3) 둘을 섞은 하이브리드의 세 가지로 갈린다. 각 방식은 가능 범위·리스크·유지보수 비용이 크게 다르다. 결론부터: **개인용 메모리·요약 시나리오에는 옵션 3 (하이브리드) 이 비용/리스크 대비 가장 합리적**이다.

## 옵션 1. 공식 — Kakao Developers API

`developers.kakao.com` 에서 제공하는 정식 API.

**할 수 있는 것**
- 카카오톡 채널 (구 플러스친구) 메시지 발송
- "나에게 보내기" (`/v2/api/talk/memo/default/send`) — 본인 토큰으로 본인에게만
- 친구 목록 조회 + 친구에게 발송 (사전 동의·스코프 필요, 실제로는 카카오싱크/비즈 채널 맥락에서만 의미 있음)
- 알림톡 / 친구톡 (카카오비즈니스 + 발신 프로필 등록 필요)

**할 수 없는 것**
- **개인 1:1·단톡방 메시지 읽기는 어떤 공식 API 로도 불가능**. "내 카톡방의 대화를 가져와 요약" 같은 시나리오는 정식 경로로는 닫혀 있다.

**리스크**: 없음. 정식 사업자/개인 개발자 등록만 하면 안전.

→ 이 도구를 **"내가 나에게 요약을 카톡으로 받기"** 정도로 확장하는 데에는 충분. 그 이상은 사실상 불가.

## 옵션 2. 비공식 — LOCO 프로토콜 클라이언트

KakaoTalk 클라이언트가 자체 서버와 통신할 때 쓰는 바이너리 프로토콜 (LOCO) 을 역공학해서 "또 하나의 카톡 클라이언트" 가 되는 방식. 비슷한 구현체:

- `node-kakao` (TypeScript)
- `PyKakao` / `loco-protocol` (Python)
- `agent-messenger` (Node)

**할 수 있는 것**
- 실시간 수신 (PC 카톡 앱 켤 필요 없음)
- 메시지 송신, 방 입장/퇴장, 답장, 이미지 전송 등 거의 모든 클라이언트 동작

**리스크 — 진지하게 고려할 것**
1. **Kakao 서비스 약관 위반.** "비공식 클라이언트로 접속" 은 명시적 금지 항목.
2. **계정 정지·영구 차단 가능성.** 비정상 세션·이중 로그인·메시지 패턴이 감지되면 계정이 막힐 수 있고, 카카오 계정에는 카카오페이/뱅크/T 등이 묶여 있을 수 있음. 잃을 게 큰 사람일수록 비추.
3. **인증 키 관리가 매우 민감.** LOCO 는 디바이스 등록 → e2e 키 → 세션 키 흐름인데, 이 키가 새면 타인이 내 카톡으로 로그인 가능.
4. **프로토콜이 수시로 깨짐.** 카톡이 클라이언트 버전을 올리거나 핸드셰이크를 바꾸면 라이브러리가 따라잡을 때까지 동작 멈춤. 유지보수가 사실상 별개의 부업.
5. **법적 회색지대.** 개인용 자동화 수준이면 형사·민사 사례가 드물긴 하지만, "타인 메시지 수집·재배포" 로 가면 정통망법·개인정보보호법 이슈가 즉시 생김.

**관행**: 진짜로 옵션 2 를 진행한다면 **별도 카카오 계정** 을 만들어 그 계정으로만 LOCO 클라이언트를 돌리고, 메인 계정은 절대 건드리지 않는 게 업계에서 굳어진 관행.

## 옵션 3. 하이브리드 — Mac 로컬 DB + Kakao Developers 송신

가장 현실적인 절충안:

- **읽기**: 로컬 Mac 의 KakaoTalk sqlcipher DB 를 직접 읽음 ([[kakaotalk-mac-data-locations]]). PC 카톡 앱이 켜져 있는 동안 자체 동기화한 결과를 활용.
- **쓰기**: "나에게 보내기" 공식 API 로 매일 23:50 같은 시간에 요약을 본인 카톡 "나와의 채팅" 에 발송.

**장점**:
- 약관 위반 없이 "PC 켜진 동안 수집 → 모바일에서 요약 받아보기" 동선 완성.
- LOCO 통합 결정을 미룰 수 있음.
- 계정 정지 위험 0.

**한계**:
- PC (또는 Mac mini) 가 항상 켜져 있고 KakaoTalk 앱이 백그라운드 로그인 상태여야 함.
- "친구에게 보내기" 가 아닌 "나에게만 보내기" 시나리오에 한정.

## 시나리오별 추천

- **개인 메모리·요약 (=내가 내 단톡 요약을 자동으로 받기)** → 옵션 3.
- **봇이 친구·단톡에 자동 응답** → 옵션 2 외에는 길이 없음. 별도 계정 필수.
- **사업자 발신 (마케팅·알림)** → 옵션 1 (알림톡/친구톡).
- **AI 에이전트와 카톡으로 대화** → 옵션 1 (채널) + 본인은 옵션 3 으로 보강.
