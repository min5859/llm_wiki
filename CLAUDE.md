# LLM Wiki v2 — Claude Code 지침

개인 지식 축적 시스템. v1(llm_wiki)이 주제 산개로 write-only 창고가 된 교훈을 반영해,
**범위를 좁히고 재조회율을 성공 기준으로 삼는다.**

## 수집 기준 (가장 중요한 규칙)

다음 두 도메인에 해당하고, "3개월 뒤 다시 검색할 것"이라고 답할 수 있는 지식만 수집한다.

1. **AI 에이전트 활용** — Claude Code 설정·패턴, MCP, 토큰 절약, 에이전트 자동화 파이프라인
2. **트레이딩·투자 시스템** — 증권 API, 백테스트, 스코어링, 운영 버그·규칙

### 스킵 대상 (명시적 제외)

- 뉴스성 논문·신모델 리뷰 (다시 읽히지 않음)
- 일회성 도구 사용법, 단발 트러블슈팅 (검색엔진이 더 빠름)
- 위 두 도메인 밖의 모든 주제 — 아무리 흥미로워도 기록하지 않는다
- 억지로 만들지 말 것: 애매하면 스킵이 기본값

### 승격 규칙

- 같은 지식이 **두 번째 등장했을 때** wiki로 승격하는 것을 기본으로 한다.
- 첫 등장 지식은 세션에서 소비하고 버린다. 단, 도메인 핵심 버그·결정은 즉시 기록 가능.

## 읽기 경로 (wiki의 존재 이유)

- 질문에 답하기 전 반드시 `wiki/` 를 검색해 기존 지식을 확인한다.
- 답변에 wiki 문서를 활용했으면 해당 문서를 인용한다.
- **성공 기준은 문서 수가 아니라 답변에 wiki가 인용되는 빈도다.**

## 디렉토리 구조

```
llm_wiki/
├── wiki/
│   ├── concepts/          # 개념 정의
│   ├── patterns/          # 실무 패턴, AI 활용법
│   ├── analyses/          # 방법론·기술 분석 (뉴스성 리뷰 금지)
│   ├── projects/          # 프로젝트 기록
│   ├── decisions/         # 기술 결정
│   └── bugs/              # 버그·트러블슈팅
├── raw-sources/           # 개인 원본 자료 (git 포함)
├── schema.md              # frontmatter 규칙
└── GUIDE.md               # 작성 가이드
```

## Wiki 파일 형식

- 파일명: `kebab-case.md`
- frontmatter 필수: `domain` (`ai-agent | trading`), `sensitivity` (`public | internal | confidential`), `sources`, `updated`
- 추가/수정 시 `## 변경 이력` 섹션에 한 줄 기록

## 절대 하지 말아야 할 것

- `raw-sources/` 파일 수정 금지
- 사용자 확인 없이 wiki 삭제 금지
- 출처 없는 정보를 기정사실로 기록 금지
- `sensitivity: confidential` 문서 git commit 금지
- **수집 기준 밖 주제의 문서 생성 금지** — 제안도 하지 않는다
