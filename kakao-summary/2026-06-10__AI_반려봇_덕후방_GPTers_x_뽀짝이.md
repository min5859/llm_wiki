## 인사이트·노하우

### Claude Fable 5 (Mythos) 특성 및 사용 팁

- **정체**: "미토스(Mythos)" 중 사이버 공격·생화학 관련 능력을 제거한 버전. 공식 포지셔닝은 소프트웨어 엔지니어링·지식 기반 작업·컴퓨터 비전·과학 연구.
- **토큰 소비**: Opus 대비 체감 5~20배 빠르게 소진. "1시간 만에 5시간 사용량 소진" 사례 다수. Effort가 high 설정인 것이 원인 중 하나로 추정.
- **캐싱 개선**: Opus는 프롬프트 캐시를 1024 토큰부터 적용했으나 Fable은 512 토큰부터 적용 → 반복 주입 프롬프트에 캐싱 할인이 더 자주 들어가 실질 비용은 2배보다 덜 비쌈.
- **컨텍스트 창**: Claude Code에서 Fable 선택 시 기본 1M 컨텍스트로 붙음 (Opus는 200K 기본, 수동으로 1M 강제 가능).
- **보안 필터 + 폴백**: 보안 이유로 거부된 요청은 자동으로 Opus가 폴백 처리하나, 사용자가 어느 모델이 응답했는지 알 방법이 없음.
- **22일 이후**: 별도 usage credits(API 토큰 과금) 방식으로 전환 예정. 구독 플랜 포함 여부는 미결.
- **적합 용도**: "아주아주 해결 안 되는 문제" 원포인트 해결사 또는 그동안 잘 안 되던 까다로운 작업에 집중 투입하는 전략 권장.

### 모델 선택 실무 판단 (커뮤니티 합의)

- Sonnet: 하트비트 등 가벼운 반복 태스크 전용. "확실히 멍청하다"는 평가 다수.
- Opus 4.6: 만족도 피크. 코딩·복잡 추론 메인 모델.
- Fable: 토큰 부족해 Opus로 회귀한 사례 다수. "소넷↔Opus 슬쩍 교체는 분노, Fable↔Opus 교체는 무방"이라는 뉘앙스.
- API 챗봇 비용: "클로드 API로 대화 몇 번에 1달러씩 빠져나감 → DeepSeek v4로 교체". API 용도는 현재 DeepSeek이 비용 대비 답이라는 공감대.

### Hermes 에이전트 소켓 누수 디버깅 (실 사례)

- **원인 1층 (트리거)**: 맥미니 DNS가 Tailscale MagicDNS(100.100.100.100)에 의존 → Tailscale 재연결 시 `api.telegram.org` 조회 일시 실패.
- **원인 2층 (진짜 범인)**: 실패한 소켓을 닫지 않고 계속 쌓는 코드 버그 → 소켓 1,268개 누적 → 한도 초과 → 영구 먹통. DNS 회복 후에도 소켓이 막혀 자가 복구 불가.
- **해결**: 프로세스 재시작 필요. 근본 수정은 실패 소켓 명시적 close 처리.

### Hermes 다중 계정 운용 (OAuth 방식)

- Claude 계정 2개 + Codex를 폴백 체인으로 구성: 1번 계정 한도 소진 시 2번 계정 자동 전환.
- 한도 감지 및 폴백이 에이전트마다 다르게 동작하는 문제 존재. OAuth 방식 사용 시 특히 불안정 사례 있음.
- 오픈클로·헤르메스에 구독 계정을 붙이려면 현재 Codex 외 선택지 없음.

### VS Code 페르소나 훅 → Codex 이식 실패

- VS Code에서 잘 작동하던 Claude 페르소나 훅을 Codex로 가져오면 "읽지 않은 듯 하지 말란 것을 하는" 현상 발생. 훅 구조 차이로 인한 미적용 문제.

### 카카오톡 CLI 트러블슈팅

- `kakaocli` → `kmsg` → `openkakao-cli` 순으로 이동 시도, 결국 kmsg의 다른 포크 버전에서 해결.
- 터미널 권한을 전부 넘겼음에도 안 되는 경우 → 다른 포크 버전 시도가 유효한 우회책.

### LLM 봇 테스트 인프라 설계 (Fable 실 작업 사례)

- **문제**: 게시판 AI 봇이 댓글 맥락을 잘못 파악하거나, 자신의 이전 댓글을 인지하지 못해 엉뚱한 응답.
- **해결 구조**:
  1. 과거 실패 사례 32건을 회귀 테스트 케이스로 변환.
  2. LLM 응답의 정답 비교 불가 문제 → 저렴한 LLM 판정자가 "의도에 맞는 반응인가"를 채점.
  3. 실패 댓글 URL 하나만 주면 DB에서 상황을 통째로 캡처해 테스트 케이스로 만드는 도구 제작.
  4. 코드·케이스 미변경 시 API 호출 0회 캐시 처리 → 36개 재실행 비용 0원.
- **근본 수정**: 봇 맥락에 "이 이름의 댓글은 내가 쓴 것"이라는 자아 인식 표시를 심어 자기 발화 추적 가능하게 함.
- **레거시 함정**: 20년 묵은 PHP에서 테스트용 `include` 시 봇 큐 처리 전체 실행 버그 발견 → 가드 추가. 같은 이름 함수 중복 정의 + 에러 표시 off 설정으로 에러 없이 조용히 죽는 현상은 이분 탐색으로 추적.

## 자동화·시스템 적용 후보

- **LLM 봇 회귀 테스트 패턴**: 과거 실패 URL → DB 캡처 → LLM 판정자 채점 → 캐시로 무비용 재실행. LLM 행동 테스트의 범용 골격으로 재사용 가능.
- **봇 자아 인식 패턴**: 봇 맥락에 "이 발화자명 = 나" 표시를 심어 자기 이전 발화를 추적하게 하는 구조. 댓글 스레드 기반 봇 모두에 적용 가능.
- **뽀짝이 서재 자동화 흐름**: 링크 하나를 주면 → 요약 칼럼 작성 → SNS 배포까지 자동화. "소재 선정·검토는 인간, 작성·배포는 AI" 역할 분리 구조.
- **다중 Claude 계정 폴백 체인**: 계정 1 한도 소진 → 계정 2 자동 전환. Hermes/OpenClaw 운용 시 비용 연속성 확보 패턴.
- **매일 오류 리스트 작성 루틴**: AI 프로젝트 운용 중 "매일 마지막에 오늘 오류 난 부분을 리스트로 작성하고 개선점을 말하게 했더니 3일째부터 안정화" — 지속적 자가 감사(self-audit) 루프 패턴.
