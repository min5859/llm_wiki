## 인사이트·노하우

- **공급망 공격(주의)**: npm·PyPI 경유로 TanStack, UiPath, Mistral AI, OpenSearch, Guardrails AI 등 관련 패키지에 악성 코드 삽입 사례가 언급됨(당일 뉴스 링크로 전달).
- **요금·한도 감각**: Claude Code에서 “코드 분석” 비중이 크면 Pro 한도가 빨리 소진된다는 체감 공유(GPT 쪽은 상대적으로 덜 든다는 식의 비교가 나옴. 수치·정책은 원문이 추정 수준이라 **검증 필요**).
- **요금 전략**: 종량제는 버그·예측 불가 사용량에 대한 불안 → **한도 설정이 있어도** 심리적 부담. 비용 여유가 없으면 **Pro 계정 2개**나 **GPT 1 + Claude 1**처럼 역할 분리가 언급됨.
- **Hermes + Android/Termux**: 공식 문서상 Termux 경로는 있으나, Linux/macOS 원라이너급 완전 자동이라기보다 **문서화된 수동 설치**에 가깝다는 정리. `.[all]` 대신 **`.[termux]` extra**(음성 등 일부 의존성이 Android와 안 맞음).
- **Termux 설치처**: 플레이스토어 Termux는 **업데이트가 안 된다**는 경험담도 나옴 → **F-Droid** 쪽 권장.
- **OpenClaw(Android)**: [openclaw-android](https://github.com/AidanPark/openclaw-android)로 설치 언급. 업데이트 시 **`oa --update`** 권장(봇에게도 “메모리”로 남기라는 농담성 팁).
- **OpenClaw 게이트웨이**: “게이트웨이 스타트”가 안 되고 **게이트웨이 모드로 켜야** 한다는 식의 설명(정확한 UI/명령어 명은 대화만으로는 **불명**).
- **Slack에서 OpenClaw 그룹·봇 대화**: `~/.openclaw/openclaw.json`의 `channels.slack.accounts.<accountId>`에 예시로  
  `dmPolicy: "pairing"`, `groupPolicy: "open"`, `allowBots: true`, `replyToMode: "all"`  
  병합. **채널 스레드에 답하려면 reply 모드**, **봇↔봇은 allowBots**가 필요하다는 **6시간 시행착오** 언급.
- **Anthropic “서드파티”·OAuth 경고**: 공개적으로는 제한적인 이야기가 있으나, **Claude CLI 방식으로 연결하면 동작**한다는 경험 공유(정책 해석은 **불명/변동 가능**).
- **Hermes `computer_use` + macOS 권한 반복** (원인→절차 요약):
  1. `hermes update` → `hermes computer-use install --upgrade`
  2. `open -n -g -a CuaDriver --args serve` 후 `cua-driver check_permissions`
  3. **접근성(Accessibility)·화면 기록(Screen Recording)** 허용 후 다시 `check_permissions`로 granted 확인
  4. Hermes 자체 안전 팝업은 **`hermes --yolo -t computer_use`** 또는 세션에서 **`/yolo`** (신뢰할 수 있는 로컬에서만)
  5. 반복 시: `cua-driver diagnose`, `which cua-driver`, 버전 확인 — **IDE 터미널 vs 실제 권한 주체(bundle id) 불일치**가 흔한 원인으로 설명됨.
- **OpenClaw + Claude CLI 느림/인증 문제** (한 참가자의 분석·프롬프트 처방 형태):
  - OAuth 토큰 교체 타이밍에 키체인 읽기 race → `readClaudeCliKeychainCredentials()`에 **200ms 간격 최대 3회 재시도**, 단 `"could not be found"`·**exit 44**는 재시도 없이 null (구현은 `Atomics.wait`로 blocking sleep 제안).
  - `~/.claude/projects/` 비대 → `pi-trajectory-flush` 타임아웃 가능 → `du -sh ~/.claude/projects/*/ | sort -rh`로 정리 후 **백업·삭제**(활성 `openclaw-workspace-*` vs 일회성 구분).
- **브라우저 자동화(로컬·도커 없이)**: “보통 그냥 한다”는 취지 + **`agents.md`에 로그인·결제·포스팅·삭제 금지**, 사용자 명시 허용 시 예외 같은 **가드레일** 사례.
- **문서/논문 작업 추천 경향(합의에 가까운 느낌)**: 장기 맥락·스타일 반영은 **OpenClaw/Hermes**, 논문 본문 톤은 **Claude 웹**, 자동화는 **OpenClaw 쪽**에 기울일 수 있다는 의견 다수.
- **크론·일회 작업**: 크론은 **새 세션**으로 열린다는 설명. 컨텍스트 이어주기는 OpenClaw **helper**(“마지막 세션 이어주는” 옵션 언급) 또는 **“리줌 스킬”**을 봇에 만들어 쓰는 패턴.
- **Hermes 컨텍스트 압축**: 설정에서 압축 비율 등 조절 가능, **칸반** 활용 제안.
- **로컬 LLM(Hermes)**: Qwen 3.6 35B는 **타임아웃** → 타임아웃 연장 시도. Gemma 4 31b Q8_0 등 **양자화 모델은 비추** 의견. Beelink GTR9 Pro + 128GB RAM 사례에서 **BIOS UMA 버퍼** 조정 제안(VRAM/UMA 이슈로 추정; 세부는 **불명**).
- **한도·결제 후에도 OpenClaw만 안 됨**: “**Codex로 인증 정보 지우고** 다시 온보딩” 시도 제안.
- **인증 자주 풀림**: “`claude` CLI refresh” 또는 **TTY 방식 재인증**이 잘 맞는다는 팁.

이건 보통 두 가지가 섞여서 생깁니다.

## 1) macOS 권한 팝업이 계속 뜨는 경우

Hermes `computer_use`는 내부적으로 `cua-driver / CuaDriver.app`를 쓰는데, macOS TCC 권한은 “내가 권한을 준 앱/번들 ID” 기준으로 매칭됩니다.

그래서 Terminal / Cursor / VS Code / Claude Code 같은 IDE 터미널에서 실행하면, 권한을 준 대상과 실제 권한을 요청하는 프로세스가 달라져서 매번 다시 묻는 것처럼 보일 수 있어요.

아래 순서로 해보세요.

bash
hermes update
hermes computer-use install --upgrade

만약 `hermes computer-use` 명령이 없으면 Hermes 버전이 오래된 거라 업데이트가 먼저 필요합니다.

그다음 CuaDriver를 앱 컨텍스트로 먼저 띄웁니다.

bash
open -n -g -a CuaDriver --args serve
cua-driver check_permissions

이때 macOS 팝업이 뜨면 둘 다 허용하세요.

- System Settings → Privacy & Security → Accessibility
- System Settings → Privacy & Security → Screen Recording

그리고 다시 확인:

bash
cua-driver check_permissions

정상이라면 Accessibility / Screen Recording 둘 다 granted로 나와야 합니다.

## 2) “Allow computer_use to capture/click/type?” 같은 Hermes 승인 팝업인 경우

이건 macOS 권한 문제가 아니라 Hermes 자체의 안전 승인 팝업입니다.  
신뢰할 수 있는 로컬 작업에서 승인 팝업을 줄이고 싶으면:

bash
hermes --yolo -t computer_use

또는 세션 안에서:

text
/yolo

다만 `--yolo`는 승인 없이 도구 실행을 허용하는 모드라, 믿을 수 있는 로컬 작업 범위에서만 쓰는 게 좋습니다.

## 계속 반복되면 진단 출력 확인

bash
cua-driver diagnose
which cua-driver
cua-driver --version
