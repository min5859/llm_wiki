# AI 요약

제공된 대화 원문만을 바탕으로, 요청한 카테고리에 해당하는 내용만 정리했습니다.

---

## 인사이트·노하우

- **회의 STT·다화자 상황**  
  - 위스퍼만으로 화자가 많을 때 인식이 잘 안 되는 경우가 있다는 경험이 공유됨.  
  - **다글로(Daglo)**, 구글 계열 서비스, **리턴제로** 등이 언급됐으나, 구체 제품명·비교 결과는 사람마다 체감이 다름 — 위스퍼가 가장 나았다고도 함.  

- **Deepgram (유료 성격, 무료 티어·크레딧)**  
  - **화자 분리(dirization 등)** 된다는 경험.  
  - **실시간 전사 속도**는 Deepgram급으로 빠른 것이 많지 않았다고 함 — 녹음 **후처리**에는 다른 선택을 검토했다는 의견.  
  - 가입 시 **$200 크레딧**, STT 분당 대략 **$0.0043** 근처로 인용됐고, 두 달 개인 사용에 약 **$3.5** 수준이었다는 사례. 개인 규모로는 크레딧이 한창 남는다는 인상. 서비스 목적 아니면 회원 재가입 등의 언급이 있었으나 실제 정책·약관은 채팅 밖 확인 필요 (**불명: 최신 이용약관**).  

- **슈퍼 그록 체험·연동**  
  - **iOS 그록 앱**에서 "**슈퍼그록**" **3일 체험** 가능했다는 정보; 구독 갱신은 중지했다는 참고 패턴.  
  - 참가자에 따라 **안드로이드**에서도 같은 체험 UI가 떴다는 언급.  
  - **오픈클로·헤르메스**에서 OAuth로 연결하면 **메모리 서치**, **auxiliary model**, **이미지·영상 생성** 등에 활용 가능하다고 함 (**불명: 정확한 메뉴·제한사항은 공식 문서 확인**).  
  - 오픈클로 **5.18 업데이트**로 그록 OAuth 지원 갸념 됨이라는 회신.  

- **X(포스트)·스킬**  
  - 슈퍼 그록 연결 시 **`xurl` 스킬**로 **X 포스트 검색 및 자동 등록**도 가능하다고 함.  

- **슬랙 봇 스킬 구성 예**  
  - 이미지: **Gemini 키**만 주면 되는 형태로 **「나노바나나(이미지) 스킬」**을 연결했다는 실사례 (**불명: 정확한 패키지/리포 이름**).  
  - 영상: **Hedra API 키**를 제공하는 방식.  

- **무료 LLM과 로컬**  
  - 헤르메스에서 **무료 중 가장 나은 선택** 요청에 대해 **OpenRouter 무료 모델**은 대화 가능 횟수가 많지 않음, **Nous Research 딥시크 v4 플래시**는 불안정, **네모트론 30B**(오픈라우터 무료 등)은 빠른 편인데 데이터 수집 우려처럼 “**꽁짜에는 대가가 있다**”는 톤.  
  - **큐웬(Qwen)** 은 로컬에 **컴퓨팅 자원이 극히 클 때만** 현실적인 “무료에 가까움”; 어설픈 사양에서는 느리고 소음·스트레스만 커져 **결국 구독으로 옮겼다**는 경험.  
  - **올라마 클라우드 약 $30 요금대**에서 **GLM 5.1**, **딥시크**, **큐웬** 등을 조합해 써보는 제안이 있음 (**불명: 현 요금표**).  

- **GPU·로컬 vs API 구독**  
  - “**웬만하면 GPU 살 필요 없고 API·구독이 성능 대비 저렴**하다”는 귀납형 조언 다수 — **3090 구매 후회** 같은 사례.  
  - **큰 장비**(예: 채팅에서 **DGX Spark** 이름이 나옴)여도 클라우드 대형 서비스 수준 속도와 큰 모델을 함께 기대하기 어렵다는 논점.  
  - 로컬은 **보안·연구·튜닝**에 장점이라는 한줄 귀결. 장비 목적 불명확 시 “**몇 천 들여 자랑만**”이라는 농반진반 경고. 같은 작업이라도 고성능 GPU는 **소비 전력 증가** 이슈.  
  - “**목적 없이 진입하면 후회**” — 장비 최대 성능 라인 타령까지 나왔으며, **약 3년치 클맥 금액(대략 1300만 원대로 환산)** 을 레퍼런스로 던져 투자 감각을 비교했다는 논점. (**정확한 월 과금 단가는 채팅에서 확정 불가 → 불명**)

- **헤르메스 vs 오픈클로 멀티 역할 운영**  
  - **디스코드 다 계정**(예: 헤르메스 메인 1 + 서브 4 등)에서는 **경로별 스킬 설치**(본인 스킬·루트 스킬)가 꼬여 **중복처럼 보이거나 기대 경로 미사용** 문제가 발생한다고 함.  
  - 완화: 봇과 대화해서 **운영 규칙으로 정리**, **헤르메스 운용용 LLM 위키**에 적어 놓고 필요 시 검색 참조.  
  - 여러 역할(**소울** 등 표현)·워크스페이스 분리가 중요하면 **오픈클로가 유리**, 헤르메스는 초기에 **공용 스킬 vs 개별 스킬** 나누기부터 삽질이 있었다는 비교 (**불명: 최신 버전에서 개선 여부**).  

- **같은 GPT라도 헤르메스 경유 시 품질 저하 체감**  
  - “**헤르메스 거치면 더 멍청해지는 것 같은 함정**” 같은 표현과, 무한 반복 같은 경우는 작성자 스스로 **잘못 처리한 문제**였다는 수정 답변도 함께 있음 (**원인 불명 또는 복합**).  

- **녹취·녹음 파이프라인 아이디어**  
  - 업로드 → STT → **“싫어요”, “이미 했어요”, “필요없어요” 등 키 발화 검출** → **A/S 타당 / 재배정 사유 / 검토 필요**로 자동 분류 → 관리자는 **검토 필요분만 실청**.  
  - 답변: “에이전트”보다 **스크립트로도 충분**할 가능성 언급.  
  - 유사 패턴 예: 유튜브 검색→스크립트 저장→**원하는 문구 + 타임스탬프** 추출→후속은 LLM 판단 일부 포함.  

- **Claude/OpenClaro·Notion MCP**  
  - **GPT 컴퓨터 유즈**(불명: 정식 명칭)·**오픈클로** 활용 조합과 **노션 MCP**로 그룹웨어 마이그레이션에서 막히는 부분 처리가 유용했다는 회고 (**불명: MCP 스펙 디테일**).

## 자동화·시스템 적용 후보

- **고객·녹취 VOC 파이프**: STT → 부정·거부 키워드 검출 → 케이스 라벨링 → **관리자 청음 범위 축소**(스크립트 중심으로도 가능하다는 제안 포함).  
- **유튜브 대량 조사 후 보고 파이프**: 검색 → 자막/스크립트 확보 → **키워드+타임스탬프** 추출 → LLM 후처리.  
- **노션 MCP + 오픈클로**(및 채팅에서 언급된 다른 도구 조합): **레거시 그룹웨어 → 노션 마이그레이션** 시 막히는 구간 처리용 자동화·보조 패턴 (**재현에는 MCP·환경 변수·권한 설계 필요**).

## 질문·미해결

- **에이전트 많을 때 CPU 부하** 가능성 물어보았으나 “**람만 이야기**”라 불안이 컸지만, **CPU 관점의 구체적인 답·수치 없음**(대화 속 후속 명확한 해소 없음).  
- **iOS/OS 인증이 임의로 풀린 것처럼 보였을 때**(원격에 맥 미니 두지 않았다는 배경)·**별도 에이전트 복구 봇** 사용 사례를 물었지만 **직접적 해결책 회신 없음**.  
- **HTML에 애니메이션 넣은 발표 형식**(어떤 툴 쓰는지) 요청 후 **후속 답변 없음**.  
- **SAM3 API 비용**이 적정한지 물어본 질문 — **금액·체감에 대한 회신 없음**.  
- **지피터스 청강 절차**는 “**별도 신청 없이 줌만**”이라는 회신으로 **해결됐음**.
