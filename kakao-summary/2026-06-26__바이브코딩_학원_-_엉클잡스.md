# AI 요약

## 인사이트·노하우

- **AI 업무 위임의 핵심 역량 전환**: OpenAI Codex 사용 데이터 기준, 표본 이용자의 80.6%가 30분 이상, 70.2%가 1시간 이상, 25.6%가 8시간 이상 걸릴 일을 AI에 위임. "좋은 질문"만큼이나 "맡길 일의 범위·자료·완료 기준을 정리하는 능력"이 중요해지고 있음.

- **AI 자동화 설계 5줄 템플릿**: 반복 업무를 AI에 맡기기 전 선행 설계로 제안된 포맷 — `목표 / 입력 자료 / 처리 순서 / 완료 기준 / 사람이 꼭 확인할 부분` 5가지를 명시. Codex든 Copilot이든 동일하게 적용 가능.

- **모델 종속 방지 원칙**: 특정 AI 모델 하나에만 묶이지 않도록 문서·프롬프트·스킬을 "옮기기 쉬운 형태"로 남겨두는 것이 권장됨. AI 인력 이동 및 모델 경쟁이 잦은 환경에서 실용적 원칙.

- **AI 서비스 보안 관리 필요성**: 앤트로픽-알리바바 모델 증류 의혹 사례 기준, 자동화 도구 연결이 많을수록 사용량 알림·API 키 분리·비정상 호출 감지가 필수 요소로 부상.

- **AI 보정·복원 흐름 대비**: 어도비의 Topaz Labs 인수로 AI 생성뿐 아니라 후반 보정·복원이 기본 기능화되는 추세. "원본 사진·영상 파일을 잘 보관해두면, AI 보정 품질이 올라갈수록 다시 쓸 여지가 커진다"는 실용 팁.

- **엑셀 Copilot Skill 파일 방식**: Microsoft 365 Copilot에서 DCF·월간 보고·결산·차이 분석 같은 반복 업무 절차를 SKILL.md 파일로 정의하면 Copilot이 해당 방식대로 실행. "한 번 물어보기"에서 "우리 회사 방식으로 반복 실행"으로 패러다임 전환.
