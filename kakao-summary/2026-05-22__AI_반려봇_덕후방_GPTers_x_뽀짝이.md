# AI 요약

## 인사이트·노하우

- **카카오톡 봇 폴링 주기**: 뽀짝이(카톡 AI봇)의 폴링 간격은 **기본 2초**, 급할 경우 더 짧게 반응. 단, 일부 사용자는 2초에서도 인증이 풀리는 현상 경험 → 환경에 따라 차이 있음.
- **카톡 자동화 스택 (뽀짝이 사례)**:
  - 내부 작업/응답 처리: **OpenClaw**
  - 메시지 전송: 별도 "안전 wrapper" 사용
- **AI 서비스 장애 대비 이중화 전략**: Claude 장애 발생 시 대응을 위해 Claude / Codex 2개를 상시 구비 권장. 이미지 생성은 GPT → Grok까지 3개 폴백 구성하는 사례도 있음.
- **Grok 에이전트 모드 실사용 경험**: 잘 동작하나 이미지·영상 약 **20개 생성 시 리밋** 도달. 한도 소진이 빠른 편.
- **GPT Computer Use 사용량 소모**: 몇 시간 사용으로 월 한도 50% 소진 사례 보고. 159,000원 플랜 기준 12시간 연속 실행 시 약 30% 소모.
- **Spark 모델 현황**: 어느 시점부터 OpenClaw(Hermes) 내 Spark 모델 선택 불가 상태로 확인됨. "스파크 불러보라고 하면" 간접 호출은 가능하나 소모가 빨라 수 시간 이상 운용 어려움.
- **젠스파크·힉스필드 등 크레딧 기반 호스팅**: 크레딧 소모가 "엄청 심하다"는 실사용 평가.
- **호스팅어(Hostinger) VPS 실사용 팁**:
  - 해외 유튜버 초대 코드 사용 시 **10% 할인**
  - **연간 결제** 시 혜택 적용
  - 맥미니(RAM CTO 사양)보다 나은 퍼포먼스라는 사용자 평가

## 추천·자료

- **OpenClaw**: 카카오톡 봇 자동화에 사용되는 도구. 내부 작업 처리 및 응답 생성에 활용; 안전 전송 wrapper와 조합해 사용.
- **호스팅어(Hostinger) VPS**: 맥미니 대체 로컬 AI 실행 환경으로 추천됨. 월정액 플랜, 연간 결제 + 유튜버 초대코드 조합으로 비용 절감 가능.

## 반응이 컸던 화제

- **Claude 장애 발생**: 회의 준비 중 갑자기 막혔다는 호소와 함께 여러 명이 동시 반응. 상태 페이지 링크 공유, "AI가 전기같이 될 거라는 말이 실감난다"는 발언으로 이어짐 → AI 인프라 의존도에 대한 공감대 형성.
- **이중·삼중 AI 서비스 폴백 구성**: Claude 장애를 계기로 "클로드/코덱스 2개 상시 구비" 권장 발언이 나오고, GPT+Grok 3개 구성 사례까지 공유되며 공감 반응 다수.
- **호스팅어 VPS vs 맥미니 비교**: 로컬 AI 실행 환경에 대한 실사용 경험 공유 요청이 이어지며 스펙·비용·체감 성능까지 구체적인 답변으로 발전.

## 자동화·시스템 적용 후보

- **카톡 봇 자동화 패턴**: OpenClaw로 내부 로직 처리 + 별도 안전 wrapper로 전송을 분리하는 2-레이어 구조. 카톡 자동화 봇 구축 시 참고 가능.
- **AI 서비스 폴백 구성 패턴**: 주력 모델(Claude) → 보조 모델(Codex) → 이미지 전용 대체(Grok) 순으로 폴백을 명시적으로 설정해두는 운용 전략.
- **폴링 기반 챗봇 설계**: 2초 폴링 주기 + 긴급 시 단축 반응 로직. 단, 인증 세션 유지 문제를 별도로 핸들링해야 함.
