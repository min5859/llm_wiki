## 인사이트·노하우

### Pi 에이전트 특성 및 활용법
- **기본 아키텍처**: read/write/edit/bash 4가지 도구만 제공하는 극단적 미니멀리즘 설계. 메모리.md, 에이전트.md, 빌트인 스킬 일절 없음
- **장기 메모리 구성**: 기본 탑재 아님. 패키지로 받거나 직접 구현 필요. 메모리.md를 직접 만들고 "이게 너의 일기장이야"처럼 가스라이팅식 프롬프트로 학습시켜도 잘 동작하는 사례 보고
- **세션 분기 명령어**: `/tree`(같은 세션 파일 내 대안 탐색), `/fork`(이전 프롬프트 기준 새 세션 파일 생성), `/clone`(현재 작업 복제 후 계속) — 세 가지 분기 방식의 용도가 구분됨
- **자가개선 루프 가능성**: 설정 변경 후 즉시 리로드 가능 → "자기 코드를 고치고 리로드하고 반복"하는 자가개선 루프를 태울 수 있다고 언급 (실제 검증 여부 불명)
- **무게**: 클로드 코드·Codex보다 훨씬 가벼워 라즈베리파이 구동 가능. 추가 패키지 설치 후에도 클코·Codex보다 가볍다는 사용자 증언
- **CLI 한글 입력 고질**: CLI에서 한글 첫 글자가 밀리는 현상 존재. 실용적 우회책은 "포기하고 쓰기"(강제로 입력하면 결국 들어감) 또는 텔레그램/슬랙 등 외부 인터페이스 경유

### Fable 실사용 평가
- Claude Max 포함 토큰 순삭 심각. Opus 4.6과 체감 차이가 크지 않다는 의견. "설계·논의할 때 조금 쓰고 웬만한 건 Opus로 했다"는 실용적 사용 패턴

### Oracle Cloud 프리티어 인스턴스 획득 (춘천 리전)
- 원인: 프리티어 인스턴스(1 CPU / 6 GB 기준)는 경쟁이 극심해 무료로는 끝없이 대기
- 시도: 30초 간격 자동 재시도
- 해결 팁: **유료 결제 수단 등록 후 예산 제한 설정**하면 실질 과금 없이 잡힐 확률이 높아짐. 4 OCPU / 24 GB / 200 GB 기준 30초 간격으로 약 60시간 소요된 사례 있음

### Claude 결제 오류
- 결제일이 아닌데 더 높은 플랜(Max)으로 자동 결제되는 오류 보고. **계정당 1회 환불 가능**

## 반응이 컸던 화제

- **Pi 에이전트 탐구**: 다수 참여자가 기능·무게·라즈베리파이 적합성·메모리 구성법 등을 활발히 토론. "다마고치처럼 키우는" 비유가 공감을 얻으며 Pi의 철학(미니멀 + 사용자 커스터마이징)에 대한 궁금증이 촉발
- **Fable 토큰 소모 문제**: Fable을 실전에서 쓰기 어렵다는 의견에 여러 명이 공감. Claude Max에서도 동일 현상이라는 증언으로 확산

---

## 자동화·시스템 적용 후보

- **Pi + 텔레그램 + 라즈베리파이 + gpiozero 조합**: 텔레그램으로 Pi 에이전트에 명령 → 라즈베리파이 GPIO 제어 → 모션 미니카 구동 아이디어. 구체 구현 사례는 미완
- **Pi 자가개선 루프 패턴**: 에이전트가 자신의 설정 파일을 수정 → 리로드 → 반복하는 자율 개선 루프. 처음에는 사용자 개입 빈도가 높지만 차차 줄어드는 구조로 언급됨
- **Pi 장기 메모리 가스라이팅 패턴**: `memory.md` 파일을 만들고 "이게 너의 일기장이야"로 지정 + `learning.md` + `agent.md` 조합으로 사실상 메모리 시스템 대체
