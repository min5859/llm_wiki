# AI 요약

## 인사이트·노하우

### Codex 초기화권(rate-limit reset credit) 유효기간 확인
- 유효기간이 UI에서 사라졌고, 프롬프트나 소스로 확인해야 함
- **확인 프롬프트** (실제 공유된 내용):
  > `~/.codex/auth.json` 파일에서 `tokens.access_token`을 읽어 `chatgpt.com/backend-api/wham/rate-limit-reset-credits`에 요청. `available_count`, `status/title/granted_at/expires_at`만 요약하고 UTC→로컬 시간 변환. 401 반환 시 자격증명 만료 또는 Authorization 헤더 누락.
- 초기화권은 발급 후 **30일 유효**, 최대 3개까지 누적 가능한 것으로 보임
- OpenAI가 기분 내킬 때 주는 방식이며 월 1개 정도로 추정

### Hermes/OpenClaw 크론잡 토큰 절약 패턴
- AI가 판단·요약하는 작업만 OpenClaw 크론잡으로 → 나머지(크롤링, 파싱, 엑셀 처리, 구글시트 연동 등)는 **리눅스/macOS 내장 crontab으로 이전** → 토큰 0 소비
- OpenClaw에게 "크론 중에 리눅스 크론잡으로 옮길 수 있는 거 물어보면 잘 옮겨준다"
- 대시보드로 크론 관리 추천. 실제 사례: 토큰 드는 OpenClaw 크론 6개 / 토큰 0인 리눅스 크론 31개로 운영 중
- 크론잡은 아무리 잦아도 **1일 1회 이상 잡지 말 것**

### Claude API Opus 과금 함정
- Hermes에서 메인 GPT + writer Claude Opus API 구성으로 운영 시 **3개월에 $10,000** 소진 사례 발생
- 근본 원인: 매 메모리 로드(2만 자)·규칙 적용·병렬/연속 글 작성이 Opus API에 모두 청구됨
- **해결 방향**: Claude API → GPT OAuth(구독)로 전환. Hermes는 Claude 종량제만 직접 지원하므로 GPT OAuth가 가성비 최선

### 로컬 LLM 실행 현실적 조언
- Windows: VRAM이 전부. 오프로드 무관하게 **깡 VRAM** 기준으로 판단해야 함
- 12B 모델은 RTX 5070(VRAM 16GB)에서 돌아가지만 **성능(속도·퀄리티)이 처참**
- Gemma 4 12B: 인코더 없음, GPT 대비 파라미터 100배 이상 차이, 추론 성능 현격히 낮음
- 글 작성 목적이라면 최소 **31B 이상** 필요 → 5090급 VRAM 아니면 불가
- 로컬 추천 기준: **맥 스튜디오 풀옵 또는 DGX급** 아니면 권장 안 함
- DGX Spark는 리눅스 전용, RTX Spark은 윈도우 지원
- "클라우드 모델 쓰다가 로컬 쓰면 화딱지나서 못 씀. 특히 Claude 쓴 사람은."

### Hermes에서 Claude Code를 서브에이전트로 연결하는 방법
- GPT 메인 + Claude Code `-p` 옵션 연결로 **OAuth Claude Code**를 글쓰기 서브에이전트로 활용 가능
- 실제 프롬프트 예시:
  > "Hermes Agent에서 백엔드 코딩을 직접 수행할 하위 에이전트인 Claude Code를 -p(프로젝트 경로 또는 프롬프트 지정) 옵션과 함께 연결하여 ChatGPT 메인 인터페이스처럼 제어하게 설정해줘"
- Claude Code `/rc` 명령으로 폰에서도 Claude Code에게 글쓰기 지시 가능 (OAuth)
- Discord 채널 연결도 가능 (세션별 연결 방식)

### AI 봇 말투 다양화 기법
- **사용자 ID 기반 말투 분기**: 텔레그램 유저 info의 고유 ID를 인식시켜 상대방별로 말투 다르게 설정 가능 → "저한테는 친근하게, 타인에게는 업무적으로"
- **퍼스널리티 프리셋**: 커뮤니티별 말투를 프리셋으로 만들고, "프리셋 40% + 60% 섞기"로 새 프리셋 파생 가능
- **시간/게시물 고유값으로 랜덤 seed** 부여해 글 다양성 확보 가능
- 말투 깎는 절차: 마음에 안 드는 부분을 찬찬히 설명 → AI와 대화해 규칙 추출 → 스킬로 절차화 → 검증방법 체크 → 증거·출처 남기기

### LLM 글쓰기 품질 개선 패턴 (최종 정리)
잘 써진 글 몇 개 선별 → GPT에게 "이 글들의 전개 방식·문단 구성 형식을 분석하라" → 반복해서 포맷 정착 → 그 포맷으로 글 작성 지시 → 작성 후 검토·수정 절차 추가
