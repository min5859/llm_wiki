# AI 요약

## 인사이트·노하우

- **AI 에이전트 권한 분리 원칙**: 웹을 읽는 에이전트에게 결제·파일 삭제·코드 실행·외부 전송 권한을 한꺼번에 부여하면 위험. "읽기 권한"과 "실행 권한"을 분리하는 습관이 필요.
- **에이전트 공격 사례**: JadePuffer가 Langflow 취약점(CVE-2025-3248)을 악용해 1,342개 Nacos 설정 기록 암호화. Zscaler는 숨겨진 웹 프롬프트가 에이전트를 속여 결제·가짜 사이트 신뢰 유도 사례 공개. → 에이전트 워크플로에는 인간 승인 단계를 반드시 포함해야 함.
- **Claude Code로 대규모 코드 점검**: 알버타 주정부가 ~50개 에이전트를 병렬로 돌려 4억 6,600만 줄 코드를 20시간 만에 취약점 점검. 사람이 했으면 약 6.5년 소요. 활용 패턴: 취약점 탐지 → 수정안 작성 → 테스트 → 사람 승인 루프.
- **Fable 모델 종료 일정**: 2026-07-08 15:59:59(한국시간)까지 사용 가능. 2026-07-13까지 +50% 사용량 제공, 이후 종료.
- **Fable → Opus 4.8 대체 경험**: Fable로 제작한 콘텐츠가 셧다운으로 중단됐을 때 Opus 4.8로 재촬영 후 공개. Fable이 일시 부활했다가 7월 8일 재종료 예정.
- **삿갓(Ssat) 라이브러리**: HTML 개선용 라이브러리. Claude Code + Fable 작업 시 터미널 작업 최소화를 위해 스킬을 사전에 구현해두는 방식 활용.
- **콘텐츠 제작 워크플로**: 설계는 GPT Pro + Fable, 슬라이드는 브랜드 디자인 시스템 녹여낸 Claude 디자인 + Fable, 터미널 작업 최소화를 위해 스킬 사전 구현.

👉 볼 포인트:
웹을 읽는 에이전트에게 결제, 파일 삭제, 코드 실행, 외부 전송 권한을 한꺼번에 주면 위험해요. 자동화할수록 "읽기 권한"과 "실행 권한"을 분리하는 습관이 필요해요.
