## 인사이트·노하우

- **Codex 사용량 리셋 타이밍**: 계정 3개 기준 사용량이 16% / 21% / 0% 상태에서 남은 기간 2일이라는 사례 공유. 복수 계정 분산 운용 시 잔여량 모니터링이 중요.

- **로봇암 픽앤플레이스 접근 전략** (P14 정리):
  - 제조사마다 SDK·통신 방식이 다름 (ROS, TCP/IP, Modbus 등). 두산·뉴로메카, 레인보우, UR, Franka, xArm 각각 상이.
  - **고정 좌표 단순 반복** → 제조사 펜던트 or RoboDK로 충분
  - **무작위 부품·비정형 환경** → 상용 빈피킹 솔루션
  - **완전 커스텀·연구·통합 개발** → ROS 2 + MoveIt 직접 구현

- **농업용 로봇 사업성 평가 프레임** (P14):
  1. 1등 기업의 해당 로봇 매출 규모 확인 (=시장 사이즈 직결)
  2. 기술적 차별성 검토
  3. 원가·마진 구조 분석
  4. 국내 한정 vs 글로벌 사업 방향 결정
  - 국내는 지원사업 외 구매 수요가 약하므로 해외 사례 우선 참고. 해외 경쟁 제품 구입 후 리버스엔지니어링이 빠른 진입 경로.

- **신규 서비스 런칭 전 구매의향 검증** (P14):
  - 진성 고객군과 직접 맞닿아 있지 않으면: 커뮤니티에 무료로 풀고 → 피드백 수집 → 업그레이드 → 프로(유료) 버전 추가
  - MVP 먼저 찐 고객에게 배포 후 피드백 반영 후 시장 출시

- **결제 모듈 구현 시 멱등성 필수** (P13): "결제는 멱등성을 꼭 잘 지켜야 합니다"라고 현업 개발자가 강조. 중복 결제·재시도 처리에서 멱등성이 가장 자주 무너지는 포인트.

- **AI에게 맡기기 어려운 파트**: 운영 서버 셋팅은 AI에게 쉽게 위임되지 않는다는 현업 10년 개발자의 실감. 현재도 직접 서버 공부 중.

- **agents.md 한계**: 각 분야 전문가급 agents.md를 만들어도 "에이전트가 한 일을 내가 판단할 수 없다"는 문제 제기(P39). AI 출력 결과의 품질 판단력이 사용자에게 필요함.
