## 자동화·시스템 적용 후보

- **텔레그램 장애 대비 이중 채널 패턴**: 에이전트(OpenClaw·Hermes 등)를 텔레그램 + 슬랙(또는 디스코드) 동시 연동. 텔레그램 connect 실패 시 슬랙을 fallback 입구로 자동 전환, 텔레그램은 백오프 재시도로 자동 복구 대기. 로그에 `✓ telegram connected` 뜨면 재연결 완료 확인.
- **봇투봇(Bot-to-Bot) 멀티에이전트**: 슬랙·디스코드에서 봇투봇 대화가 가능하다는 언급 → "goal 설정 → 칸반·업무 위임으로 에이전트 분담"이 단순 봇투봇 채팅보다 퀄리티가 높다는 현장 경험. 단순 봇투봇은 "토큰 낭비, 별 효과 없음"이라는 반례도 있어 설계 주의 필요.
- **로컬 LLM 폴백 체인**: `claude → codex → minimax → local LLM` 순서로 폴백하는 백업 체인 구성. 툴 콜 가능 최소 기준은 9B 이상.
- **슬랙에서 HTML 파일 공유 우회**: 슬랙은 HTML 인라인 렌더링 불가 → "PNG로 달라고 하면 캡처해서 준다"거나 파일 다운로드 후 로컬에서 열기. 또는 로컬 서버로 열고 임시 터널 주소 전달하는 방법 사용 사례 있음.
