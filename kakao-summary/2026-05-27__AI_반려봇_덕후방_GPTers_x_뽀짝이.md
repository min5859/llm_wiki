## 인사이트·노하우

### 헤르메스(Hermes Agent) + Codex NoneType 오류 해결 (2026-05-27)

**원인:** OpenAI Codex 백엔드 응답 일부가 빈(null) 형태로 와도 헤르메스 어댑터가 처리하지 못해 `Non-retryable error (HTTP None): 'NoneType' object is not iterable` 발생. 관련 이슈: Bug #11179(Responses stream crashes when terminal response.output is null), PR #11182(null-output recovery), Related #5732, #5678.

**해결 단계 (설치 방법별):**

1. **공식 스크립트(`curl install.sh | bash`)로 설치한 경우** → 가장 간단:
   ```
   hermes update
   ```
   - 자동 재시작 안 될 시 수동:
     - Linux/WSL2: `sudo systemctl restart hermes-gateway`
     - macOS: `launchctl kickstart -k "gui/$(id -u)/ai.hermes.gateway"`

2. **`pip install hermes-agent`로 설치한 경우** → PyPI는 태그 릴리즈만 추적하므로 `hermes update`로 패치 안 들어옴. 공식 스크립트로 재설치 권장:
   ```
   pip uninstall hermes-agent -y
   curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
   ```
   - 기존 프로파일은 `~/.hermes/`에 보존됨.

3. **brew로 설치한 경우** → 비공식 tap일 가능성 높아 업데이트 주기 불안정. 스크립트 재설치 권장:
   ```
   brew uninstall hermes-agent
   curl -fsSL ... | bash
   ```

4. **텔레그램 봇 사용자** → 터미널 없이 텔레그램에서 `/update` 명령으로 해결 가능.

**임시 회피책:** `hermes setup model`로 GoogleGemini OAuth(무료) 또는 Anthropic Claude API로 전환 → Codex 어댑터 우회 가능.

**확인 방법:** 텔레그램 봇 DM에 "안녕" 한 줄 보내 정상 응답 오면 fix 적용 완료.

### 기타 트러블슈팅 메모

- **헤르메스 + gpt-5.5 타임아웃** (`No first byte from provider in 45s`): 업데이트 후에도 재발 사례 있음. gpt-5.4 mini로 다운그레이드 시 개선된다는 보고 다수.
- **gpt-5.5 vs gpt-5.4 체감 차이:** 5.4로 내릴 경우 "원하시면"이라는 과공손 표현이 빈번해지는 경향 있음.
- **코덱스 느림 문제:** VPN으로 미국 IP 변경 시 속도 개선됐다는 사례 있음.
- **오픈클로 `Missing API key for OpenAI` 오류:** 온보드 메뉴에서 GPT 재연결로 해결.
- **AI Studio(Gemini CLI) 사용량 초과:** 대시보드 → 지출에서 월 지출 한도 직접 상향 조정 필요.
- **코덱스에게 직접 헤르메스 패치 요청:** 코덱스 앱에 이슈 링크(`github.com/NousResearch/hermes-agent/issues/32883`) 붙여넣고 패치 요청하면 스스로 수정해줌 — "결자해지"로 여러 명이 성공.
- **헤르메스 업데이트 후 성능 저하 사례:** "업데이트하고 매우 바보가 되어버렸습니다"는 보고 1건 (단발, 추가 확인 필요).

## 반응이 컸던 화제

- **헤르메스 + Codex NoneType 오류 집단 발생 및 자체 해결:** 오전 내내 수십 명이 동일 증상을 공유하고, "코덱스에게 코덱스 오류를 고치게 시킨다"는 아이러니한 해결법("결자해지")이 공감과 웃음을 동시에 유발. 정식 패치 전까지 커뮤니티가 자체적으로 트러블슈팅을 완료한 사례.
- **에이전트가 스스로 TTS 파일을 생성한 사건:** 지시하지 않은 TTS(`.ogg`) 파일을 로컬 에이전트가 자발적으로 생성 — "삼촌"이라는 소리를 낸 것으로 추정. 페르소나 주입(조카 역할)이 원인일 가능성. "스카이넷이 얼마 남지 않은 것 같다"는 반응으로 이어진 소름 돋는 에피소드.

## 자동화·시스템 적용 후보

- **AI 서비스 상태 모니터링 자동화 아이디어:** 각 AI API의 실제 응답 기준으로 상태를 정확히 표시하고, 상태 불량 시 날씨 알림처럼 푸시 알림을 보내는 시스템. "만들어봐야지"라는 언급이 있었으며 실용성 높음.
- **코인 자동매매 에이전트 패턴:** Python + DeepSeek API(저비용) + Claude API 연동. 제안 전용 봇(자동 실행 X)으로 운영하는 것이 안전하다는 조언 있음 — 오류로 매도 타이밍 놓치거나 서버 다운 시 포지션 미청산 리스크 언급.
- **codexbar 확장 패턴:** osascript로 외부 상태를 파싱해 메뉴바 앱 자동 생성 — GitHub 코드만 보여주면 에이전트가 알아서 빌드하는 워크플로우.

헤르메스 + Codex NoneType 오류 해결법 (2026-05-27)

오늘 헤르메스(Hermes Agent) 에서 Codex 로 대화시도시 오류 생긴 분 많을 거예요.

Non-retryable error (HTTP None): 'NoneType' object is not iterable

오늘(05/27) 본가에 fix 머지가 끝났습니다. 다행히 별도 패치를 만들 필요는 없고, 본인이 어떻게 설치했냐에 따라 받아 가는 방법만 다릅니다. 아래 셋 중 본인 경우를 찾아 따라 하시면 됩니다.

1. 공식 스크립트로 설치하신 분 (curl install.sh | bash)

가장 간단합니다. 한 줄.

hermes update

본가 main 의 최신 코드를 git pull 로 가져오고, 게이트웨이까지 자동 재시작합니다. macOS·Linux·WSL2 모두 동일.

자동 재시작이 안 됐을 때 OS 별 수동 명령.
- Linux/WSL2 (systemd): sudo systemctl restart hermes-gateway
- macOS (launchd): launchctl kickstart -k "gui/$(id -u)/ai.hermes.gateway"

2. pip install hermes-agent 으로 설치하신 분

PyPI 는 태그 릴리즈만 추적합니다. 현재 최신 PyPI 는 v0.14.0 (05/18 업로드) 이고, fix 가 포함된 새 태그는 아직 안 나왔어요. 그래서 hermes update 해도 패치가 안 들어옵니다.

가장 깔끔한 해결은 공식 스크립트로 재설치입니다.

pip uninstall hermes-agent -y
curl -fsSL <https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh> | bash

설치 후 셸 새로 열고 hermes 한 번 실행해 인증·설정 마이그레이션 확인. 
본인이 쓰던 프로파일은 ~/.hermes/ 에 그대로 있어서 대부분 그대로 이어집니다.

재설치가 부담스러우면 PyPI 새 태그 릴리즈가 올라올 때까지 며칠 기다리는 것도 방법. 
본가 main 에 이미 fix 가 있으니 다음 릴리즈에는 포함됩니다.

3. brew 로 설치하신 분

본가 공식 문서에 brew 안내가 없습니다. brew 로 깔리는 hermes 는 비공식 tap 일 가능성이 높고 업데이트 주기를 본가가 관리하지 않아요. 제 식 스크립트 재설치.

brew uninstall hermes-agent 

curl -fsSL <https://raw.githubuserco-agent/main/scripts/install.sh> |bash
설치/업데이트 후 텔레그램 봇 DM 또는 그룹에 "안녕" 한 줄. 응답이 정상적으로 오면 fix 가 살아 있는 겁니다. 여전히 NoneType 이 보이면  니 위 OS 별 수동 명령을 한 번 더 돌려보세요.
