## 인사이트·노하우

- **Claude Fable 성능 저하 체감**: 출시 초기 대비 성능이 낮아진 느낌이라는 의견 다수. "이전엔 와 진짜 신세계 다른 모델이다 했는데 지금은 오푸스 5.n 느낌"이라는 표현이 핵심. Opus로의 위임이 잦아졌다는 부수 증거도 언급됨.
- **Fable 비용 실측**: Claude Code 데스크탑 앱에서 5시간 세션 소진 후 크레딧으로 Fable을 돌린 경우, "생각보다 비용이 많이 안 나갔다"는 실사용 보고.
- **Claude 가격 인상 예정**: 구체적 금액·시점은 불명이나, 구독 플랜 개편 가능성과 함께 언급됨. "이전 버전에 비해 성능 저하가 있다는데 구독 플랜에 흡수시키려고 테스트 중일 수 있다"는 추측.
- **Apple 스토어 구독 시 가격 차이**: Claude를 모바일 App Store로 구독할 경우 일반 구독보다 비쌀 수 있음. "모바일에서 애플스토어로 구독해서 그러실 수도"라는 언급.

## 추천·자료

- **Reddit (서브레딧)**: Claude/Anthropic 관련 서브레딧이 Discord와 함께 "대화가 활발한" 커뮤니티로 언급됨. 사용 팁, 트러블슈팅 공유에 적합.

## 반응이 컸던 화제

- **Fable 성능 저하 논쟁**: 여러 명이 체감 저하를 공유하며, "걍 처음부터 위임할 건 위임하는 게 낫다", "Fable은 기획·설계자 고정, 실행은 GPT 5.5" 같은 실무 대응 전략으로 이어진 토론. 초보 사용자가 "성능 저하를 어떻게 판단하나요?"라고 질문하며 폭넓게 참여함.
- **AI 업무 권한 설계**: 영끌맨 뉴스 브리핑의 "AI가 혼자 해도 되는 일 vs 반드시 내 승인이 필요한 일을 구분하라"는 실천 제안이 공유됨. 발송·결제·삭제·고객정보 접근은 승인 대상으로 두라는 구체적 지침 포함.

## 자동화·시스템 적용 후보

- **Fable + Composer 역할 분리 패턴**: Cursor에서 Fable을 설계·검사자로, Composer 서브에이전트들을 실행자로 명시적으로 역할을 분리하는 구성. CLAUDE.md(또는 프로젝트 설정 파일)에 역할을 명시하면 자동 분배가 더 안정적이라는 실사용 팁.
- **멀티모델 오케스트레이션 패턴**: "Fable 기획 → Opus 설계 → Codex 실행" 역할 체인 구성 가능성 논의. Droid + Missions 조합으로 각 역할을 분리 배정하는 방식이 대안으로 언급됨.
- **AI 자동화 권한 문서화**: 현재 운영 중인 AI 자동화 하나를 골라 "AI 단독 실행 가능 작업"과 "인간 승인 필수 작업"을 한 문서에 나눠 정리하는 워크플로 제안. 특히 발송·결제·삭제·고객정보 접근은 승인 대상으로 명시.
- **GitHub Issue Fields + MCP 연동**: GitHub Issue의 Priority, Effort, Start/Target date 필드를 GitHub MCP 서버와 연결해 AI 에이전트가 이슈 우선순위·일정을 자동 읽고 설정하는 패턴. 프로젝트 관리 자동화에 적용 가능.
