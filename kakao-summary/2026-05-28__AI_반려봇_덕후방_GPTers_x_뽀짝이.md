# AI 요약

## 인사이트·노하우

- **OpenClaw 클린설치 시 npm 차단 문제** (5.26 버전)
  - 증상: 클린설치 시 npm 차단 발생
  - 해결: 업데이트 경로로 설치하면 정상 작동 (`openclaw doctor` 자동 실행)

- **Hermes + GPT-5.5 `No response from provider` 에러**
  - 증상: `gpt-5.5` 모델이 `chatgpt.com/backend-api/codex`에서 silent reject, non-streaming 모드 전환
  - 임시 우회: model을 `gpt-5.4-codex`로 변경
  - 근본 해결 사례 1: Hermes 최신버전 업데이트 후 Codex에게 "non-streaming 모드 전환 문제 해결"을 직접 명령 → 3일째 재발 없음
  - 근본 해결 사례 2: `hermes update` 공식 패치 적용으로 해결

- **GPT-5.5 vs 5.4 체감 차이**
  - 5.5 → 5.4 다운그레이드 시 답변 품질 체감 저하 다수 보고
  - 5.4 mini는 더 큰 품질 저하, Sonnet도 "답답하다"는 평가

- **Hermes에서 "원하시면 릴레이" 반복 문제**
  - 원인: 모델이 5.4로 낮춰지면서 작업 방식 변화
  - 해결 시도: "원하시면 릴레이 금지, 이후 작업을 모아서 한 번에 알려달라"고 명시적 지시 → 일부 개선

- **클로드 세션 끊김 원인**
  - 단일 HTML 파일이 4,000자를 넘어가면서 세션 반복 종료됨

- **Codex CLI 버전 요구사항**
  - OpenClaw 5.26 업데이트 사용 시 **Codex CLI 0.134.0 이상**으로 업데이트 필요

- **스프레드시트 조작 작업 대안**
  - Codex/Hermes에서 API 응답 지연 발생 시 Gemini 활용 권장 (구글 계열 작업)

- **클로드를 Codex 헤드로 두고 CLI로 호출하는 방식**
  - OAuth 없이 Claude를 쓰는 방법으로 "Codex를 헤드로 두고 Claude CLI로 호출" 패턴 언급 (상세 불명)

## 반응이 컸던 화제

- **Hermes + GPT-5.5 에러 및 모델 다운그레이드 고민**: 다수 사용자가 동일 증상을 보고하고, 해결 방법을 공유하며 활발히 토론. 요금제 상향(Plus→Pro) 여부까지 논의로 이어짐. 코덱스 불안정 → 클로드 복귀 움직임이 여러 명에게서 동시 발생.

- **Gemini 무료/할인 코드 요청**: 여러 명이 연달아 4개월 무료 코드를 요청하는 흐름 발생. GPT 할인 방법 부재에 대한 공감도 이어짐.

## 자동화·시스템 적용 후보

- **Hermes 불안정 시 자동 모델 폴백 체인 구성**: `gpt-5.5` 실패 시 `gpt-5.4-codex`로 자동 전환하는 fallback chain 설정 패턴 (hermes-agent#21444 참고)
- **"릴레이 금지, 작업 일괄 보고" 프롬프트 지시**: Hermes/에이전트가 불필요한 중간 릴레이를 반복할 때 "이후 진행할 작업이 있으면 모아서 한 번에 알려달라"는 지시로 토큰 낭비 방지
- **비개발자 업무 자동화 진입점**: "내 일상에서 무엇을 자동화할 수 있을지 먼저 고민하는 것"을 시작점으로 권장 (클로드 활용 방향 모색 시)
