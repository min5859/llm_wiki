# AI 요약

## 인사이트·노하우

- **Claude Code 승인 팝업 제거 방법** (3가지)
  1. `claude --dangerously-skip-permissions` 플래그로 실행
  2. `Shift+Tab`으로 Automode 활성화
  3. `CLAUDE.md`의 Do/Do not/Avoid 항목에 명시 → "90%는 문제없고 나머지 10%는 노코멘트"

- **직접 하네스 구성 vs AI Agent 경유 비교** (P32 경험담)
  - Hermes/OpenClaw 같은 에이전트를 거치지 않고 직접 하네스를 짜서 Claude API에 연결하면:
    1. **비용**: 에이전트가 활성 도구 때문에 불필요하게 소모하는 토큰이 없어 압도적으로 저렴
    2. **속도**: 동일 질문·동일 모델 기준 응답 속도 빠름
    3. **개인화**: 트리거별 하네스 개별 세팅 가능 ("AI비서" / "AI팀장" 등 트리거마다 별도 하네스)
  - "카톡봇으로만 쓰겠다"는 용도에 특히 적합

- **OpenClaw-Slack 봇 좀비 세션 문제**
  - 증상: 한 스레드 내에서 연달아 메시지를 보내다 어느 순간부터 이모지 반응만 하고 텍스트 응답 없음. 새 포스팅을 올리면 즉시 응답하나 해당 스레드는 영영 응답 안 함
  - 원인: 좀비 세션으로 고착
  - 해결: Claude Code를 열어 진단 후 "좀비 세션 죽여달라" 요청

- **Slack `replayToMode = all` 설정**
  - 이 값이 설정되면 봇이 매번 스레드 답글 형식으로만 응답
  - 스레드 단위 세션 관리를 원하면 켜두고, 원하지 않으면 해당 설정 변경

- **Slack 대화 세션 로컬 저장**
  - 봇과의 대화는 '세션' 단위로 로컬 Mac에 JSONL 파일로 저장됨 → 90일 제한 무관하게 후속 검색 가능
  - 인간끼리만 나눈 대화는 세션에 포함 안 됨

- **한국 vs 체코 월드컵 조별리그 첫 경기** — 뽀짝이(봇) 스코어 예측 vs 오픈클로 예측 비교, 실시간 경기 중 단체 응원, 예측 적중 여부 화제. "뽀짝이 점집 개업"이라는 말이 나올 만큼 봇 예측의 정확성에 여러 명이 반응. 다음 경기 멕시코전 예측 예고까지 이어짐

## 자동화·시스템 적용 후보

- **트리거별 경량 하네스 패턴**: "AI비서" / "AI팀장" 같은 서로 다른 트리거 키워드마다 별도 하네스 파일을 세팅하는 방식. 에이전트 레이어 없이 API 직접 연결로 토큰·속도 최적화. 봇 전용 서비스에 적합
- **봇 대화 세션 JSONL 로컬 저장 + 검색**: Slack 세션이 로컬 JSONL로 저장되는 점을 활용해 "기가막힌 문장이 책에 안 들어갈 때" 대화 아카이브를 뒤지는 워크플로. 전자책·글쓰기 작업에서 LLM과의 과거 대화를 재료로 재활용하는 패턴
- **Fable 모델로 분류 설계 + 저렴한 모델로 실행**: 옵시디언 고아 노트 정리 등 분류 작업에서 고성능 모델(Fable)로 방법론 설계, 실행은 저렴한 모델로 위임하는 2단계 패턴
