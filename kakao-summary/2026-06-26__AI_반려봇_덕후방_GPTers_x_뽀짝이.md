## 인사이트·노하우

- **Claude(AI)에게 파일 삭제 권한 위임 시 주의**: WSL 환경에서 Claude에게 "다른 드라이브로 이동해줘" 지시 후 파일이 삭제됨. `rm` 같은 비가역 명령은 bypass 훅을 써도 허락 없이 실행될 수 있음.
  - 원인: 이전 세션에서 해당 명령을 허락해둔 이력이 남아 있었을 가능성
  - 해결: P27 조언 — "bypass 쓰실 거면 훅으로 `rm`이나 `sudo` 같은 명령어는 항상 허락받고 쓰게 하세요"
  - 다행히 AI가 별도 브랜치에 커밋해둬서 완전 손실은 면함 → **AI 작업 중에도 커밋 상태를 주기적으로 확인할 것**

- **Hermes 동작 불량 시 엔진(운영 두뇌)을 먼저 확인**: Hermes 자체는 로컬 서비스이므로 "잘 되고 아니고"의 문제가 아니라, 연결된 LLM 엔진(GPT OAuth 등) 상태가 원인일 가능성이 높음. GPT OAuth로 연결 시 정상 동작한다는 사례 있음.
