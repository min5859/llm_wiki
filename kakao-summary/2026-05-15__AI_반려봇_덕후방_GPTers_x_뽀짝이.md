# AI 요약

제공하신 대화 정리본만을 근거로, 장기 보존 가치가 있는 내용만 카테고리별로 정리했습니다.

---

## 인사이트·노하우

- **Slack 장애 확인**: 2026-05-14경 다운 보도([gvwire 기사](https://gvwire.com/2026/05/14/slack-down-for-thousands-of-users-downdetector-shows/))와 사용자 체감(이후 복구)이 맞물림.
- **Hermes × Codex 런타임**: 공식 문서상 OpenAI 모델 사용 시 Codex를 핵심 런타임으로 켤 수 있다고 하나, 실사용자 경험으로는 **켜면 `delegate_task`(데리게이트 태스크)가 안 된다**는 보고가 있음 → 개발 전용 프로필 등으로 분리해 검토하는 언급.
- **Codex를 Hermes 런타임으로 둘 때 트레이드오프(발언 요약)**  
  - 손해: 즉시 메모리 저장(`이 사실 기억해` 등)이 그 턴에 바로 안 먹고 백그라운드 대기, `session_search`로 과거 Dev 세션 회상 불가, Hermes todo 대신 Codex `update_plan`으로 기록 형태가 달라짐, **subagent 분기(`delegate_task`) 불가**.  
  - 보존: 브레인·메모리 자동 주입·백그라운드 memory/skill 루프·스킬 로딩·웹/브라우저/비전/이미지/TTS 등.  
  - **운용 제안**: 코드·디버깅·테스트 중심 Dev에는 Codex 강점(shell, apply_patch, 샌드박스, GitHub 플러그인 등)이 맞을 수 있음. 과거 세션 회상·즉시 메모리·대규모 병렬 서브에이전트가 필요하면 **`/codex-runtime auto`로 일시 복귀**하거나 PKM/Main으로 라우팅하라는 식의 분기 언급.
- **클로드/클코 장애 시 대응 논의**: 당일 다수가 동시에 불안정 체감(모델·구간별 상이). **장애 대비 폴백**(예: 클코 메인 사용자는 GPT Plus 등), **로컬 준비** 언급.
- **맥 로컬 추론 속도**: 스레드에서 “맥 유저면 올라마보다 빠르다”는 취지의 글이 공유됨(구체 스택은 링크 설명에 따름).
- **오케스트레이션용 로컬 모델 기대치**: 참여자 의견으로 **Opus 급·불명의 상위 GPT 계열 급이 아니면 오케 역할은 어렵다**, 로컬은 목적에 따라 쓰되 개인이 무리해 맞출 필요는 적다는 쪽의 공감대.
- **실측·체감 혼재**: 예) `qwen3.6 35b`를 특정 맥 환경에서 높은 토큰/초라고 기대하는 발언 vs 고사양 PC에서도 버벅임 체감 등 — **동일 모델이라도 환경·양자화·컨텍스트에 따라 천차만별**.
- **Hermes에서 OpenClaw 게이트웨이 복구 스킬** 존재 언급 + 안정성은 **5.4 패치 전후부터 좋아졌다**는 체감 여러 건.
- **게이트웨이 워치독**: 끊김 시 복구해 주는 도구로 [jkf87/clawdog](https://github.com/jkf87/clawdog) 소개 — “며칠간 게이트웨이 껐다 켠 기억이 없다”는 사용 후기.
- **병렬 에이전트 호출 시**: 바쁜 일정에 직렬 규칙을 안 지켜 병렬로 급히 돌렸다가 **무한 루프·사색** 수준으로 꼬인 사례 공유. **“첫 호출 때는 답변 올 때까지 가만히 지켜보기”**를 강조.
- **출력/추론 유출 버그**: 사용자에게 **영어 `<think>`류 내부 계획과 한국어 최종 답변이 함께 노출**되는 현상이 보고됨. 실행 플랫폼(문맥상 OpenClaw 쪽) **버그로 규정**하고 버그 리포트 필요하다는 설명 흐름.
- **OpenClaw + Hermes(WSL) 충돌 후 에르메스 종료**: 윈도우에 OpenClaw, WSL에 Hermes 조합에서 업데이트 후 Hermes가 자주 죽는다는 제보. 대안으로 **도커 설치** 제안이 나옴(원인·재현 단계는 대화에 없음).
- **클로드 OAuth/API 과금 오류**: `API provider returned a billing error` — **OAuth로 로그인해도 실사용이 API 경로면 그렇게 나올 수 있다**는 해석과, **다른 모델로 바꾼 뒤 OAuth 사용 방식으로 설정해 달라고 요청**하라는 조언.
- **Anthropic 정책 변경 요지(대화 중 전달 내용)**: **2026-06-15부터** Agent SDK 및 **`claude -p` 등 프로그래밍/에이전트성 사용량이 구독 한도에 포함되지 않고**, 별도 **크레딧(Pro $20 / Max 5x $100 / Max 20x $200)** 으로 **API 정상 요율** 청구로 이동한다는 요약. 인터랙티브 클로드 코드만 쓰면 변화 없을 수 있으나, CI·GitHub Actions·스케줄 에이전트 등은 **실질 대폭 인상**으로 읽는 논조.

## 질문·미해결

- **이미지 템플릿 + 대화 텍스트 → 페이드 인/아웃 영상 느낌 자동화**: 유사 사례를 물었으나 **구체 구현 답변 없음**.
- **카카오톡 Loco WebSocket으로 메시지 송신 불가**: 제한·변경 여부 **대화 내 미해결**.
- **Slack 앱 매니페스트 `display_name` 한글 불가**: 입력 위치 확인만 있고 **해결책 없음**.
- **Codex 웹의 시간 제한(5시간 등) 표시 사라짐**: 구독 종료와 연관인지 **오답/무관 응답만 있고 실제 원인 불명**.
- **Nous 포털 DeepSeek 무료 유지 기간**: **불명**.
- **윈도우 OpenClaw + WSL Hermes 충돌**: 재현 조건·로그·공식 이슈 **대화에 없음**.
- **클로드 코드 구독 ‘액세스 비활성화’·관리자 허용**: 조직 정책으로 추정되나 **확정 절차 없음**.

핵심 변경점 요약

2026.5.9 주요 변경

• Core Tool Search 추가
큰 OpenClaw/MCP/client tool catalog를 검색·호출하는 compact bridge 추가.

• Skills symlink 허용 옵션 추가
skills.load.allowSymlinkTargets로 신뢰한 sibling repo의 symlink skill 폴더 사용 가능.

• 모델/추론 상태 명령 개선
/think default, /fast default 추가. 세션 override를 지우고 기본값 상속 가능.

• 에이전트 system prompt에 실제 provider/model identity 주입
“지금 무슨 모델이냐” 같은 질문에 런타임 기준으로 답하기 쉬워짐.

• Subagent orchestration 강화
delegationMode, sessions_yield guidance, active child context 등 추가.

• Plugin SDK / model catalog 확장
text/image/video/music provider catalog 통합 등록.

• Active Memory 개선
recall 실패/없음/비활성 상태 구분 개선.

• CLI 에러 메시지 개선
실패 시 “무슨 일인지 + 다음 recovery command”를 더 잘 안내.

• Breaking change 있음
BlueBubbles 기반 iMessage surface 제거/폐기. 기존 channels.bluebubbles는 channels.imessage + Mac imsg/SSH wrapper 방식으로 마이그레이션 필요.

2026.5.12 주요 변경

• 설치 가벼워짐
Bedrock, Bedrock Mantle, Slack, OpenShell sandbox, Anthropic Vertex 등 일부 provider/plugin dependency가 core에서 외부화됨. 안 쓰는 기능의 의존성을 덜 끌고 옴.

• Control UI / WebChat 개선
streaming auto-scroll 모드 저장 가능: near-bottom / always-follow / off.

• ACP fallback 추가
acp.fallbacks로 primary backend 실패 전 backup runtime 시도 가능.

• Telegram 안정성 크게 개선
polling을 isolated worker + durable local spool로 옮겨 event-loop stall에도 덜 죽게 함. Markdown/HTML formatting, group media mention gate도 개선.

• Subagent model 적용 순서 수정
agents.defaults.subagents.model이 target agent primary보다 먼저 적용되도록 수정.

• 보안/샌드박스 강화
Windows USERPROFILE도 sandbox blocked home root에 포함. .codex, .openclaw, .ssh 같은 credential-bearing path 노출 방지.

• Provider auth/config 더 엄격화
provider apiKey를 아무 대문자 문자열에서 env var로 추정하던 동작을 줄이고, structured SecretRef 중심으로 정리.

• Codex/OpenAI 경로 개선
Codex OAuth/profile, media tools, MCP projection, app-server thread rotation, source-reply delivery 쪽 다수 수정.

• Plugin install/update/doctor 안정화
pnpm 11, peer dependency 보존, plugin runtime scan, managed install 복구 등 개선.

• Gateway/session/history 개선
Gateway protocol v4, explicit delta/replace frames, session history sequence 처리 개선.

• Rich replies / WebChat / TUI 개선
text 없는 card/button-only reply가 빈 응답으로 드랍되는 문제들 수정.

주의할 점

• BlueBubbles iMessage 쓰면 2026.5.9가 breaking.
• provider key를 예전 방식으로 넣어둔 config는 2026.5.12 이후 SecretRef 쪽 점검 필요할 수 있음.
• plugin 외부화 때문에 업데이트 뒤 doctor --fix가 필요할 가능성 있음.
• 안정판 기준으로는 2026.5.12까지 올리는 게 맞고, 2026.5.14-beta.1은 테스트 목적 아니면 아직 보류 추천.

ANTHROPIC이 조용히 모든 CLAUDE 구독을 너프했습니다

6월 15일부터 Agent SDK와 claude -p 사용량이 더 이상 구독 한도에 포함되지 않습니다.

무료 업그레이드처럼 들리죠?

아닙니다.

6월 15일 이전에는 모든 프로그래밍 사용량이 인터랙티브 Claude Code 사용과 동일한 보조금 풀에서 차감되었습니다.

$200 Max 플랜은 구독 요율이 원시 API 가격의 약 25배 저렴했기 때문에 $1,000 이상의 토큰을 소모할 수 있었습니다.

이제 Anthropic은 이를 두 개의 별도 풀을로 분리합니다.

인터랙티브 사용량은 그대로 유지됩니다.

하지만 모든 에이전트 및 프로그래밍 사용량은 전체 API 요율로 측정되는 새로운 "크레딧"으로 이동합니다.

크레딧:

Pro — $20 
Max 5x — $100 
Max 20x — $200

전체 API 요율로 $200는 무거운 에이전트 사용 시 며칠 정도밖에 지속되지 않습니다.

기존 보조금 풀에서 동일한 작업 부하는 한 달 내내 지속되었을 것입니다.

Anthropic은 이를 "무료 크레딧을 제공합니다"라고 포장하고 있습니다.

실제로는 25x 보조금을 제거하고 1x를 돌려준 것입니다.

이것은 6주 만에 세 번째 청구 정책 변경입니다.

4월에 그들은 구독에서 타사 에이전트를 완전히 금지했습니다.

이제 그 결정을 철회했지만 유용하게 만들었던 경제성을 죽였습니다.

터미널에서 Claude Code를 인터랙티브하게만 사용한다면 당신에게는 아무런 변화가 없습니다.

CI 파이프라인, GitHub Actions, 스케줄된 에이전트, 또는 claude -p를 통해 아무거나 실행한다면 방금 대규모 실질 가격 인상을 받았습니다.
