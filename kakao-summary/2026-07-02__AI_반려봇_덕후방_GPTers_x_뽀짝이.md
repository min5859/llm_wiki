# AI 요약

## 인사이트·노하우

- **OpenClaw 버전별 Fable 5 지원**: 6.11 이상 패치가 적용되어야 Fable 5 카탈로그가 노출됨. 구버전(예: 6.5)에서 Fable 5를 억지로 지정하면 카탈로그 없음 → Opus 폴백 시도 → 같은 맥에 설치된 로컬 QWEN으로 폴백되어 "중국어 모드"가 됨. 해결: `claude update`로 최신 버전 업데이트 후 새 세션 재시작.

- **Fable 5 토큰 소모량**: 코드를 많이 읽는 작업(예: 스킬 레이지로드 분석) 1회 세션에 오픈라우터 잔액이 84.92 → 37.91 로 감소, Fable 20회 호출. Max×5 플랜도 짧은 시간(약 1시간) 안에 한도의 50% 이상 소진 가능.

- **Fable 절충 운용 패턴**: Fable은 스펙·계획서 작성·설계·검토에만 사용하고, 구현 서브에이전트는 Opus로 지시하는 방식이 토큰 절약에 효과적. "전부 Fable로 돌리면 너무 녹는다"는 실사용자 경험.

- **GPT를 Fable 모드처럼 활용**: Fable 한도 소진 후 GPT에 Fable 스타일 프롬프트를 주입하는 방식으로 대체 사용. Fable mode 특성으로 정리된 내용:
  - 3개 이상 파일·다단계 추론 작업에서 추론 깊이 향상
  - 모호한 요구사항을 스스로 좁히려는 시도 증가
  - verification 서브에이전트 호출 전후 자체 점검에 유리

- **Codex `--yolo` 플래그**: Codex CLI의 auto mode에 해당. `codex --yolo` 로 실행.

- **Sonnet 5 토큰 소모**: effort 높게 설정 시 Opus보다 비용이 더 나올 수 있음. OpenClaw 연결 기준으로도 "묘하게 빨리 탄다"는 체감.

- **AI 도입 전 선행 정의 프레임** (P57 정리):
  1. 어떤 작업을 할 것인지
  2. 그 작업에 필요한 정보 접근 범위
  3. 액션이 자유로울 때 문제가 생길 지점
  4. 허용하려면 필요한 절차
  → "업무가 정의되면 참조 방식도, 가이드라인도, 파일 구조도 자연스럽게 정의된다"
