# AI 요약

## 인사이트·노하우

- **AI 에이전트 보안 설계 원칙**: 에이전트가 로그·이슈·에러 리포트(Sentry, GitHub Issues, Linear 등 외부 도구)를 읽고 터미널 명령을 자동 실행하게 두면 "Agentjacking" 공격에 노출됨. 외부 도구에서 온 내용은 "자료"로만 취급하고, 실행은 사람이 승인하는 구조로 전환 필요.
- **에이전트 권한 설계 원칙**: 메일 발송·결제·파일 삭제 같은 비가역 작업은 승인 단계를 남기는 것이 기본값이 되어야 함. "무조건 연결"보다 "필요한 도구만 허용"하는 설계가 권장됨.
- **AI 모델 다중화 필요성**: AI 도구 접근성이 국가 정책·기업 정책에 따라 갑자기 바뀔 수 있으므로, 특정 모델 하나에만 의존하지 말고 ChatGPT·Claude·Gemini 대체 경로를 병행 마련해두는 것이 안전.
- **GLM 5.2 OpenRouter 안정성 문제**: [P31] 실사용 경험 — GLM 5.2를 OpenRouter에서 사용했을 때 "자꾸 멈추고 제대로 못하는 것 같다"는 보고. Pro 모델 사용 시 차이가 있는지 미확인.
- **사카나 AI Fugu 모델**: 자체 학습 모델이 아니라 라우팅된 모델이라는 평이 있음 (확인되지 않은 루머 수준).

## 자동화·시스템 적용 후보

- **에이전트 승인 게이트 패턴**: Cursor, Claude Code, Codex 등에서 자동 명령 실행·무승인 실행 모드를 끄고, 외부 도구(Sentry, GitHub Issues, Linear)에서 온 내용은 실행 전 사람 승인을 거치는 워크플로로 전환. 특히 MCP로 연결된 외부 데이터가 에이전트 명령으로 오해되는 프롬프트 인젝션 경로 차단.
- **ARD(Agentic Resource Discovery) 패턴**: 에이전트에게 모든 도구를 미리 붙여두는 방식 대신, 필요한 순간에 안전하게 찾아 붙이는 방식. 구글·MS·GitHub Copilot이 공동 추진 중인 표준으로, 개인 자동화 설계에도 적용 가능한 방향.
