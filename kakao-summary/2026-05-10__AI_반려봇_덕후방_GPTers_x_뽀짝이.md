# AI 요약

## 인사이트·노하우

- **Tailscale 맥미니 원격 끊김 사례 (P6)**: 듀얼모니터 중 하나로 맥미니2를 VNC 띄워 쓰던 중, 카톡 에이전트 연결 작업 도중 "이 컴퓨터는 원격 제어 중입니다 → ESC 눌러주세요" TTS 가 나오며 연결 끊김 반복. **해결**: 맥미니2 재부팅 + 맥미니1 에서 VNC 연결 삭제 후 재연결. (P3 의 일반 팁: 입력 멈춤 시 연결된 모니터를 모두 뺐다 다시 끼우면 풀릴 때도 있음.)
- **봇 프롬프트가 응답을 폭주시키는 패턴 (P55 진단)**: 설이 봇이 트리거 없이 끼어들거나 동일 응답("막힌 지점 한 줄만 더 줘") 을 반복 → 원인은 "예시(few-shot)" 로 줘야 할 문구를 봇이 "요청값(실제 입력 처리 규칙)" 으로 해석하도록 프롬프트가 짜인 것. 카톡 본문이 자기 응답 트리거가 되어 무한 끼어듦 발생.
- **GPT vs Claude × Outlook 연동 (P5)**: 클로드 쪽에 MS/아웃룩 커넥터가 새로 붙음. GPT 는 Copilot 과 충돌해서 추가가 안 된 듯하고, Copilot 도 구독 없으면 몇 번 후 멈춤 → 아웃룩 자동화는 Claude 쪽으로 가는 게 현실적.
- **seedance 2.0 그리드 스토리보드 노하우 (P55·P17·P136)**:
  - 9분할은 한 컷 15 초 한계 안에 9 장면을 욱여넣는 꼴 → 의미 있는 영상 잘 안 나옴.
  - **6 장짜리 그리드**가 일관성 잡기 좋음.
  - **스토리보드 컷마다 글로 설명을 덧붙여야** 시드댄스가 잘 읽음.
  - 함정: 스토리보드 컷에 숫자를 적어두면 그대로 영상에 숫자가 박혀 나옴(크레딧 날린 사례).
  - 비용 이슈로 image2 는 무한 뽑고 seedance 는 아껴쓰는 방식 권장.
- **맥미니 vs 미니PC 가성비 (P14·P15·P25)**: 로컬 LLM 안 돌릴 거면 맥미니 다대 구매보다 AMD 미니PC(예: `ser8`, Ryzen 7 8745HS) 가 가성비 우위. 굳이 맥이 필요한 게 아니면 윈도우 미니PC 로 충분.
- **맥미니 여러 대 활용법 (P14·P139)**: 썬더볼트 병렬 컴퓨팅은 사실상 불가. 현실 패턴은 "하나의 메인 머신에서 여러 대를 **원격(VNC/SSH)** 으로 동시에 사용". 도커 안 쓸 거면 16/256 맥미니 m4 4 대 같은 구성도 후보.
- **카톡 답변용 로컬 모델은 컨텍스트 길이가 1차 병목 (P2·P15)**: M4 Mac 128GB 에서 `gemma-4-26b` 로 카톡 봇 돌리니 `n_keep 19455 >= n_ctx 13824` 에러. 텔레그램 DM 1:1 은 26B 로도 잘 되는데, 카톡 봇은 사전 프롬프트가 길어서 컨텍스트 폭주. 해법으로 `--ctx-size 131072` 옵션 제안.
- **봇 모델 자기 보고를 믿지 마라 (P2·홀든 사례)**: 봇이 시스템 프롬프트의 모델 ID(`claude-opus-4-7`) 만 보고 자기를 클로드라고 우김. 실제로는 GPT 5.5 로 갈아끼웠는데 시스템 프롬프트가 안 바뀌어 거짓 보고. → 모델 자기인식을 신뢰하지 말고 외부 `/status`·런타임에서 직접 확인해야. (P14: Minimax 도 자기를 클로드라 우김.)
- **에르메스 / 오픈클로 둘 다 쓰는 사람들의 결론 (P2·P3)**: "둘 다 잘하는 게 다르니 그냥 취향". 에르메스로 넘어간 사람 다수는 만족도가 높다는 보고.
- **클로드코드 CLI 가 에르메스에서 거부될 때 (P14)**: 도커 세팅이 잘못된 경우가 흔함 → 도커 설정 점검.

## 추천·자료

- **RTK (Rust Token Killer) (P10)**: caveman 후속 격 토큰 효율화 CLI. 영어 입력일 때 효과가 더 크고, 한국어는 의미 누락 가능성 있음. 사용 시 메시지에 따라 검토 필요.
- **honcho 메모리 / qmd**: 에이전트 장기기억 후보 두 종(P39·P2). honcho 는 영상까지 검토 중이지만 P2 의 경험상 "자기 발전·메모리 정리" 류는 세팅만 거창하고 실제 동작은 비어 있을 때가 잦았음 → 도입 시 실제로 동작하는지 뒷단 확인 필수.
- **Hermes Achievements (P39)**: 칸반보드 옆에 도전과제 시스템이 새로 붙은 듯. 게이미피케이션 요소.
- **져키리얼리즘 페퍼 육포 (P55)**: 밋업에서 받은 그 육포. 네이버에서 "져키리얼리즘 육포 페퍼" 검색.
- **ser8 (Beelink/미니PC, AMD Ryzen 7 8745HS)**: 맥미니 대체 후보로 P15 가 실사용 중. 스피커 옆에 둘 정도로 조용.

## 반응이 컸던 화제

- **봇 운영 시행착오 (설이·홀든)**: 트리거 없이 끼어들거나 모델 자기보고가 틀어지는 사례가 종일 이어졌고, 거의 모든 참가자가 호출·교정에 동참. 비개발자/개발자 모두에게 "내 봇도 비슷한 증상" 의 공감대 → 프롬프트 짜기·토글·관리자 인식 같은 운영 패턴 토론으로 발전.
- **헤르메스 vs 오픈클로**: "에르메스 나오면 오픈클로에서 다 넘어가나?" (P134) 질문에서 시작해 다수가 양립 의견. 입문자에게는 헤르메스가 만족도 높다는 정성평가 다수.
- **맥미니/미니PC 사양 토론**: 맥미니 m5 대기조 + AMD 미니PC(ser8) 추천 + 다대 활용법(원격 전제) 까지 길게 이어짐. 로컬 LLM 을 안 돌리면 굳이 맥일 필요 없다는 합의가 컸음.
- **시드댄스 그리드 스토리보드 영상**: 토옵이(P55) 가 실제 만든 망한/성공 영상을 시연하면서 노하우 공유. "망한 영상도 퀄리티는 좋다" 는 반응 + 9 분할은 상술이라는 P136 의 비판이 의미 있었음.
- **Tokenmaxxing**: 사내 토큰 대시보드로 경쟁 분위기를 만든다는 P55 의 경험담에 "그런 분위기에서 일해보고 싶다"(P2) 는 반응. 단톡 흐름과 직접 연결되는 트렌드라 호응이 컸음.

## 자동화·시스템 적용 후보

- **봇 토글 + 관리자 인식 패턴 (P2·P6·P17)**: 
  - 봇 응답을 채팅 중에 켜고 끌 수 있는 토글.
  - **관리자 인식 정보를 먼저 박아두고**, 관리자가 "조용히 해/들어가" 같은 키워드를 치면 자동응답 중단, "나와" 같은 키워드로 복구.
  - 특정 인물(예: 스터디장) 호명에는 응답하지 않도록 룰 추가.
  - **민감정보 관련 지침은 시스템 프롬프트 최상단에 배치** (P6).
- **매시간 단톡 요약 봇 (P10)**: 이미 운영 중이며 "AI 도구/에이전트 운영 팁과 봇 동작 개선 방향에 대한 실용 대화가 많음" 식의 한 줄 요약을 받고 있다고 함. → 다른 단톡 인사이트 추출에 도입 가능.
- **여행 숙박 가격 모니터링 (P17)**: 오픈클로로 여행 직전까지 호텔 가격 폴링 → 더 싼 가격 뜨면 자동으로 갈아타기. 단순하지만 효용 큰 자동화 레시피.
- **칸반 + 에이전트 스웜 (P2 의 도전 과제)**: 칸반 보드에 에이전트들을 작업자로 붙이는 구성. 단, 모든 추천을 profile 로 분리하면 오히려 복잡 → 코드로 처리할 것 / profile 로 분리할 것을 사전에 구분하라는 자체 가이드.
- **로컬 모델 카톡 봇 튜닝 출발점**: `--ctx-size 131072` 부터 시도, 카톡용 사전 프롬프트는 슬림화. M4 128GB 정도면 26B 까지는 사용권에 있음 (텔레그램 DM 한정으로는 이미 실용 수준).

## 질문·미해결

- **honcho vs qmd 선택**: P39·P2 모두 결정 못함. 실제 효용 검증 안 됨.
- **헤르메스에서 웹 제어 방법 (P16)**: 오픈클로의 chrome relay 등가물이 헤르메스에 있는지 / 별도 스킬이 필요한지 답 안 나옴.
- **맥미니 m5 출시 시기 (P25)**: 단순 궁금증이지만 사양 결정의 분기점.
- **시드댄스 그리드 한계**: 일관성 있는 6 장 그리드까지는 가능하다는 합의는 있었으나, "어디까지 진지한 영상화가 가능한가" 는 아이디어 싸움으로 미루어짐.
- **클로드코드 CLI 가 에르메스에서 거부되는 진짜 원인 (P25)**: 도커 세팅 의심 외 구체 원인 미확정.
- **토큰 효율화 ↔ 성능 손실 트레이드오프 (P55)**: 한국어 입력에서 RTK 가 의미 누락을 일으킬 가능성 — 정량 데이터 없음.
