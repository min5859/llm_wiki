## 인사이트·노하우

- **Fable 서비스 종료 대응**: Fable 플러그인 기능을 오픈소스로 재현한 `RIP-Fable`이 공유됨. Fable 종료로 진행 중이던 프로젝트가 중단된 사례 다수.
- **RAM 증설 vs 발열 문제**: Dell Precision 7760(RTX A4000, 32GB RAM)에서 AI 툴 구동 시 발열·성능 저하 발생. RAM을 128GB로 늘려도 발열은 근본 해결 안 됨. 추천된 접근: ① 먼저 API로 테스트 후 GPU 구매 결정 ② 발열 해결은 수랭 쿨링.
- **LLM Wiki → Google OKF 표준화**: Andrej Karpathy가 4월 제안한 "LLM Wiki(텍스트 파일 폴더 구조로 AI 지식 축적)" 개념을 Google Cloud가 2개월 만에 OKF(Open Knowledge Format)라는 이름으로 공식 표준화. 실체는 단순 텍스트 파일 집합.
- **AX(AI 전환) 화두**: Microsoft CEO 발언 — "AI 시대 진짜 기회는 가장 좋은 모델 선택이 아니다". AX(AI Transformation)가 새로운 키워드로 부상.

## 반응이 컸던 화제

- **Fable 서비스 종료**: 여러 참여자가 "진행 중인 프로젝트가 중단됐다", "돌아와달라"며 공감. 대안(RIP-Fable)이 즉각 공유될 만큼 의존도가 높았던 서비스.
- **Ponytail 도구**: "AI가 코드 짜기 전에 먼저 필요성을 검토" → 토큰 절감이라는 실용적 가치로 화제. 게시 직후 GitHub 스타 8천 돌파 언급.

## 자동화·시스템 적용 후보

- **Remotion + Hyperframes 영상 워크플로우**: "코드는 AI가 다 짜니까 워크플로우 설계만 잘 해두면 영상이 뚝딱" — 영상 자동 생성 파이프라인 구성 패턴. Remotion(코드 기반 영상)과 Hyperframes 조합이 게임체인저라는 평.
- **OKF(Open Knowledge Format) 패턴**: 텍스트 파일 폴더 구조로 AI 지식 베이스 관리 → Google 공식 표준. 현재 gieok 구조와 유사하나 표준 포맷 명칭 참고 가능.

## 질문·미해결

- **TTS 로컬 구동 관련**: TTS를 잘 아는 사람을 찾는 질문이 있었으나 구체적 답변 없이 종료됨. 용도는 회사 마케팅 홍보물(영상) 추정.
