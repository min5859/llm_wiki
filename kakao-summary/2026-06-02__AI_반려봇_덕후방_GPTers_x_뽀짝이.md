# AI 요약

## 인사이트·노하우

- **LLM 컨텍스트 정제 파이프라인**: LLM 프로덕트 개발 시 정보를 모델에 던지기 전 반드시 정제하는 파이프라인을 구축하면 비용 절감보다 **성능 향상 효과가 더 크다**. 특히 제미나이 계열은 불필요한 컨텍스트 로딩에 따른 성능 저하가 더 심하게 체감됨.

- **Claude 모델 버전별 체감 비교**:
  - 4.6: 안정적 기준점, 일부 사용자가 고정 사용 중
  - 4.7: "멍청해진 게 체감이 많이 됨" — 여러 사용자가 성능 저하 인지
  - 4.8: 4.6 대비 프로세스 단위 작업 처리·보고 품질 향상. 단, 작업 흐름이 길어지고 컴팩트 처리가 잦아짐. 지침을 약간 추가해주면 토큰 소모량도 줄어듦.

- **Codex 자동 개발(Goal) 토큰 소모 주의**: 자동 개발 기능을 켜놓으면 12시간 만에 주간 할당량의 30%가 소진됨. Codex는 Hermes를 경유하며 토큰이 소모되는 구조로 추정됨. 6월 1일부터 Codex의 X2 토큰 보너스가 폐지되어 소모 속도 증가.

- **Codex Goal로 인한 계정 정지 사례**: Codex Goal을 켜놓고 자는 사이 ChatGPT 계정이 아동 성착취 규정 위반으로 정지됨. 실제로는 개발 관련 작업만 수행했으나 에이전트가 독자적으로 무언가를 한 것으로 추정. **대응책**: 이의 신청 메일에 로그를 첨부해 Codex 측에 해명 메일 발송 권장.

- **Hermes + Discord 연동 주의**: Hermes Codex를 Discord에 연결해 두었다가 스팸 게시글 집중 게시로 판단되어 Discord 계정이 정지된 사례. 1주일간 이의 제기 끝에 복구했으나 이상 활동 경고는 미해결 상태.

- **AI 에이전트 오타 실수**: Tailscale을 Tailwind로 잘못 지시했다가 에이전트가 그대로 실행한 사례. 에이전트에 지시할 때 용어 오타에 유의 필요.

- **MacBook i9 발열 관리**: 악명 높은 i9 MacBook의 경우 AI에게 상태를 보여주고 조치 방법을 물어본 후 지시에 따라 실행하는 방식으로 상태 개선 효과 확인.

- **Claude Max 플랜 한도**: "1일 한도"는 없고 **5시간 한도 + 주간 한도** 구조. 상위 플랜(x20)으로 업그레이드 시 해당 한도가 상향됨.

- **유튜브 자막 외부 접근 차단**: 5월 말까지는 Hermes에서 유튜브 자동 자막 텍스트를 가져와 요약이 가능했으나, 이후 유튜브가 외부 자막 접근을 일부 차단. 우회 방법은 약관 위반 소지가 있어 계정 정지 위험 있음.
