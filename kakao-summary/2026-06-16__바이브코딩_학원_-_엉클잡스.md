# AI 요약

## 인사이트·노하우

- **Codex vs Claude 선택 기준**: Codex는 터미널 기반 에이전틱 코딩에 강점. Claude는 유저 친화적이라 진입 장벽이 낮음. 어느 쪽이 "좋다"는 맥락에 따라 다르므로, 먼저 무엇을 만들지 목적을 명확히 한 뒤 도구를 선택해야 함.

- **AI 작업 분해 원칙 (P8 경험담)**: AI가 작업을 잘 해내지 못할 때는 작업 단위를 더 세부적으로 분해할 것. 실제 사례: 자동화 업무를 8단계로 나누고, 1단계 완료 전에 2단계로 절대 넘어가지 않았더니 나중에 어디서 무엇이 잘못됐는지 예상이 가능해짐.

- **로컬 LLM 구동 조건**:
  - Mac Mini (Apple Silicon): 통합 메모리 구조라 VRAM과 RAM이 공유 → 로컬 모델 구동에 유리
  - Windows/갤럭시북 등 PC: 그래픽카드(VRAM)가 핵심. VRAM 부족 시 속도가 극단적으로 느려짐
  - 갤북5프로 (메모리 32GB, 5코어): VRAM 별도 확인 필요, 구동 가능하나 체감 속도 주의

- **LM Studio + Gemma 4 12B 로컬 셋업 (스레드 인용)**:
  - 모델 크기 7.04GB (QAT 버전 6.66GB로 더 가벼움)
  - 16GB RAM 환경에서 12GB를 모델에 할당 시 약 67,000 토큰 컨텍스트
  - Claude Code(Opus 4.8) 터미널로 설치 자동화 후 Hermes Agent 연결 → 프라이빗 로컬 AI 구성 가능
  - 단, 터미널·에이전틱 코딩 미숙자에게는 설치·연동 난이도가 높아 스트레스 유발 가능

- **초보자 권장 워크플로 (P2 조언)**:
  1. 먼저 목적·구현할 것을 명확히 구상
  2. 외부 API 키 활용 (예: DeepSeek v4 Pro via OpenRouter) → 비용·난이도 낮음
  3. 터미널에 친숙해진 뒤 Hermes Agent 연동
  4. 남이 한다고 따라서 구축하지 말 것 — "목적이 불분명한데 과한 시간·비용 투입은 피할 것"

- **DeepSeek**: 중국 모델이라 보안 우려가 있지만 가성비가 매우 뛰어나다는 평가. 민감 데이터 취급 시 주의.

## 자동화·시스템 적용 후보

- **단계별 잠금(Gate) 방식 작업 분해 패턴**: AI 자동화 작업을 N단계로 분해하고, 이전 단계 완료 전에 다음 단계로 이행 금지. 오류 발생 위치 추적이 용이해짐. 복잡한 자동화 파이프라인 설계 시 적용 가능.

- **OpenRouter → Hermes Agent 연동 시작 템플릿**: 터미널 초보자도 로컬 환경 없이 외부 API 키로 에이전트 연동을 시작할 수 있는 진입 경로로 언급됨. 온보딩 가이드 소재로 활용 가능.
