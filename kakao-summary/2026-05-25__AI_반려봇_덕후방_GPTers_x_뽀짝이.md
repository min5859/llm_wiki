## 자동화·시스템 적용 후보

### soul.md 기반 VRChat 자율 봇 구현 로드맵
단계적 구현 순서 (클로드 제안):

1. **OSC 헬로월드** — `python-osc`로 VRChat 캐릭터를 1초간 전진시키기 (핵심 검증)
2. **텔레그램 → OSC 브릿지** — 텔레그램 명령 → OpenClaw 해석 → 제스처 OSC 전송 (원격 조종 모드)
3. **챗박스 발화** — soul.md 톤으로 문장 생성 후 `/chatbox/input` OSC로 말풍선 출력
4. **자율 루프** — 백엔드 타이머로 soul.md 기반 자율 행동 결정 (자율 모드)
5. *(선택)* **환경 인식** — VRChat 로그 파싱 또는 UdonSharp 커스텀 월드로 주변 정보 수신
6. *(선택)* **음성 출력** — TTS + VB-Cable(가상 오디오 케이블) → VRChat 마이크 입력 라우팅

**VRChat OSC로 제어 가능한 것**: 이동/회전, 점프/달리기/앉기, 아바타 표정·제스처 파라미터, 챗박스 텍스트, 마이크 토글  
**불가한 것**: 시스템 UI 조작, 다른 플레이어 위치/이름 안정적 읽기

세 도구의 역할 재배치
당신이 보낸 3개 중에서 실제로 필요한 건 사실상 한 개, 나머지 둘은 참고용이에요.

핵심: WebVRChatOSC (또는 직접 OSC 구현) 이게 핵심 통로예요. VRChat은 OSC(Open Sound Control) 프로토콜로 외부에서 아바타 파라미터를 제어할 수 있게 열어놨어요. WebVRChatOSC는 "버튼 누르면 OSC 명령 보냄"인데, 당신은 버튼 대신 에이전트가 직접 OSC 명령을 쏘면 됩니다. 그래서 WebVRChatOSC 자체를 쓰기보단, 그 안의 CoreOSC 로직이나 Python python-osc 라이브러리로 OpenClaw가 직접 OSC 패킷을 보내는 게 깔끔해요.

aiavatarkit — 부분적으로 참고 이건 음성 대화(STT→LLM→TTS) 중심이에요. VRChat 캐릭터에 "말하기" 기능을 붙이고 싶다면 유용한데, soul.md + OpenClaw가 이미 LLM 두뇌 역할을 하니까 TTS 출력 + 립싱크 OSC 부분만 따오면 됩니다. 전체를 쓸 필요는 없어요.

agent-skills-vrc-udon — 이 프로젝트엔 거의 불필요 이건 VRChat 월드를 만들 때 AI가 UdonSharp 코드를 잘 짜게 돕는 도구예요. 당신의 봇이 월드 내부 오브젝트(예: 자기 전용 공간에 있는 의자, 버튼)랑 상호작용하게 만들려고 커스텀 월드를 직접 짤 거면 그때 유용해요. 단순히 아바타 움직임만 원하면 안 써도 됩니다.

VRChat OSC로 실제 뭘 할 수 있나
이게 가장 중요해요. OSC로 제어 가능한 범위가 봇의 "행동 반경"을 결정합니다.

VRChat OSC가 제공하는 것:

이동/회전 — /input/Vertical, /input/Horizontal, /input/LookHorizontal 같은 입력으로 캐릭터를 걷고 돌리게 할 수 있어요. 즉, 자율 보행 가능.
점프, 달리기, 앉기 — /input/Jump, /input/Run 등.
아바타 파라미터 — 표정, 제스처, 의상 변경, 댄스 토글 등 아바타에 노출된 모든 expression 파라미터.
챗박스에 텍스트 띄우기 — /chatbox/input. soul.md 인격의 발화를 머리 위 말풍선으로 출력 가능.
음성 입력 시뮬레이션 — 마이크 토글 등.
OSC가 못 하는 것:

메뉴 클릭, 친구 추가, 월드 이동 같은 시스템 UI 조작.
다른 플레이어의 위치/이름을 안정적으로 읽기 (보안상 제한).
즉, 소셜 월드에서 자율적으로 돌아다니며 표정 짓고 챗박스로 말하는 봇까지는 충분히 가능해요.

텔레그램 ↔ VRChat 시나리오
당신이 말한 두 가지 모드 다 됩니다:

모드 1: 텔레그램 명령 → VRChat 행동 텔레그램에서 "춤춰봐"라고 보내면 → OpenClaw가 받아서 soul.md 톤으로 해석 ("아 귀찮은데 시키니까 해줌") → "댄스 제스처 OSC 명령 + 챗박스에 멘트" 결정 → OSC로 VRChat에 전송. 이건 사실상 원격 조종이에요.

모드 2: soul.md 기반 자율 행동 (더 흥미로움) 백엔드에 일정 주기로 도는 루프를 만들어요. 예: 30초~수 분 간격으로 OpenClaw가 "지금 내(soul.md 인격) 기분은? 주변에 누가 있지? 뭘 하고 싶지?"를 스스로 판단 → 행동 결정 → OSC 실행. 이게 진짜 "스스로 움직이는" 그림이에요.

두 모드는 동시에 굴릴 수 있어요. 평소엔 자율로 돌고, 텔레그램 명령이 오면 인터럽트 걸어서 처리하는 식.

현실적으로 부딪힐 문제들
이 부분이 더 중요할 수도 있어요. 미리 알고 가야 덜 고생합니다.

1. VRChat 클라이언트가 켜져 있어야 함 OSC는 실행 중인 VRChat 인스턴스에만 명령을 보내요. 즉, 봇 전용 PC(또는 VM)에 VRChat 24/7 켜놓아야 자율 봇이 됩니다. 일부 사람들은 Quest 스탠드얼론이나 저사양 PC에 띄워놓고 운영해요.

2. 계정 정책 VRChat은 봇/자동화 계정에 명시적으로 우호적이진 않아요. ToS상 OSC 자체는 공식 기능이라 OK인데, AFK 우회용 자동 입력이나 다른 유저 추적/괴롭힘 류는 밴 사유가 됩니다. 자율 봇을 돌릴 거면 본인 전용 월드나 친한 사람들 있는 곳에서만 노는 게 안전해요.

3. 주변 인식 한계 OSC로는 "누가 내 옆에 왔다", "누가 나한테 말 걸었다"를 직접 받기 어려워요. 이걸 풀려면:

화면 캡처 + 비전 모델 (느리고 비쌈)
VRChat 로그 파일 파싱 (입장/퇴장 이벤트는 잡힘)
커스텀 월드에 UdonSharp로 감지기 만들고 OSC로 백엔드에 알리기 ← 여기서 agent-skills-vrc-udon이 비로소 쓸모 있어짐
즉, 봇이 "환경을 인식하는" 수준까지 가고 싶으면 결국 커스텀 월드를 짜야 하고, 그때 UdonSharp 스킬이 들어와요.

4. 음성 출력 챗박스 텍스트만 쓸 거면 간단한데, 진짜 말하게 하려면 TTS → 가상 오디오 케이블(VB-Cable 등) → VRChat 마이크 입력으로 라우팅해야 해요. aiavatarkit이 이 파이프라인을 이미 갖고 있어서 여기서 참고할 가치가 있어요.

추천 시작 순서

OSC 헬로월드: OpenClaw에서 Python으로 python-osc 써서 VRChat 캐릭터를 1초간 앞으로 걷게 만들기. 이게 되면 80%는 된 거예요.
텔레그램 → OSC 브릿지: 텔레그램 메시지 "손 흔들어" → OpenClaw가 해석 → 해당 제스처 OSC. 모드 1 완성.
