## 인사이트·노하우

- **Claude Code / Codex 사용 시 디스크 용량 주의**
  - 원인: Claude Code·Codex가 작업 시 자동으로 git 워크트리를 생성함
  - 문제: 대부분 사용자가 이를 모르다가 용량이 불어난 뒤에야 인지
  - 해결: Mole + 에이전트 툴로 워크트리 잔여 파일 주기적 삭제 (1일 1 Mole 루틴 권장)

- **Claude Code용 실시간 보안 플러그인 (Anthropic 공식 공개)**
  - 코드 작성·수정 중 취약점 실시간 검사
  - PR 단계 전 보안 문제 사전 차단
  - 민감 파일 수정 시도 차단 및 AI 에이전트 위험 행동 제한
  - 효과: 최종 코드 리뷰 단계 보안 문제 최대 40% 감소

- **Cursor Composer 2.5 vs 키미 비교**
  - "커서 안에서는 컴포저가 키미보다 나은 것 같다"
  - 키미의 강점은 서치 능력과 에이전트 스웜; 세션이 길어지면 키미도 컨텍스트 유지가 불안정해짐
  - Cursor 2.5 fast는 2.5 대비 **7배 비쌈** → 용도에 따라 on/off 구분 필수

- **에이전트를 며칠씩 장시간 실행하는 방법**
  - 핵심: PRD(제품 요구사항 문서)와 ADR(아키텍처 결정 기록)을 상세히 작성시키고, 체크리스트로 진행 상황 확인하며 "완료까지 멈추지 마"라고 지시
  - 실제 2일 동안 지속 실행된 사례 보고됨
  - `/goal`, `$ultragoal` 명령어 활용 중이라는 사례도 있음
  - 작업 규모 자체가 작으면 2시간 내 종료는 자연스러운 결과

- **클로드를 활용한 자기 학습 점검 방법**
  - "클로드에게 역질문하게 만들어서 내가 부족한 부분을 검토하는 시간을 가짐" — 자기주도 학습 루프로 활용

## 자동화·시스템 적용 후보

- **장기 에이전트 실행 패턴**: PRD + ADR 상세 작성 → 체크리스트 기반 진행 확인 → "완료까지 멈추지 마" 지시. 복잡한 앱 개발 자동화에 적용 가능

- **1일 1 Mole 루틴**: 에이전트 작업이 많은 환경에서 디스크 정리를 습관화하는 워크플로. CI/CD 환경이나 개발 서버에서 정기 스케줄로 자동화 가능
