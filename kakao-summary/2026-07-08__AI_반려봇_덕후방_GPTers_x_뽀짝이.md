# AI 요약

## 인사이트·노하우

- **Gemini 코딩 작업 비추천** — 코드를 엉망으로 꼬아놓고 되돌리기 어렵게 만드는 사례 다수. 빠르지만 할루시네이션이 심하고 신뢰성이 낮음. 한 참가자는 Gemini로 일주일 삽질 후 GLM로 전환해 하루 만에 해결했다고 언급.
- **Gemini 적합 용도** — 번역, 이미지/영상 분석, 커밋 메시지 생성(토큰 아끼는 용도), 단순 심심풀이. 코딩 오퍼레이션에는 부적합.
- **GLM 5.2 디자인 성능** — 비전 모델이 아닌 순수 LLM임에도 UI 디자인 생성에서 높은 평가. "마크다운 주고 Vercel로 만들어줘만 했다"는 수준에서 만족스러운 결과. 오픈라우터를 통해 Hermes/OpenClaw와 연동 가능. Claude Code 내에서도 API 키 설정으로 사용 가능.
  - 주의: GLM 자체에는 비전 없음 — 디자인이 잘 나오는 경우는 프레임워크/프록시를 통해 다른 컴포넌트가 엮여 처리되는 것일 가능성 있음.
- **GLM 5.2 로컬 구동 조건** — 통합 메모리 238GB 이상 필요. M5 Ultra Mac Studio 최대 768GB 구성이 언급됨(가격 미정). 현재 512GB 모델 중고가 약 2,800만원 수준.
- **Claude Fable 5 프로모션 연장** — 원래 7월 8일 종료 예정이었으나 7월 13일(월) KST 15:59까지 연장. 이후부터는 Usage Credits 차감. 주간 사용량 한도의 최대 50%까지만 무료 사용 가능. Opus/Sonnet보다 사용량 소모 속도가 빠름.
- **모델 역할 분담 패턴** — 실사용자들이 실제로 운영 중인 조합:
  - "Fable 설계 → GPT 실행 → Fable/Opus 검증 → 핑퐁 리페어"
  - "Claude가 메인, Hermes에서는 Codex만 사용"
  - "Codex 메인, 오픈라우터 충전 후 MoA(Mixture of Agents) 운영"
  - Gemini는 커밋 메시지 등 토큰 아끼는 보조 용도로만
- **Fable이 Opus 성능을 끌어올리는 효과** — Fable로 설계 지침을 만들어두면 Opus가 더 빠릿하게 알아듣는 현상 경험담. 컨텍스트/세션 이력이 영향을 주는 것으로 추정.
- **스마트홈 자동화 구성 예시** — SmartThings/ThinQ → Home Assistant 연결 → Hermes 자동화. 온도 센서 기반으로 "방 온도 X도 이상이면 에어컨 ON" 트리거 구성 계획. 스마트 온도계 네트워크 이탈 문제(샤오미, 국산 제품 모두 경험)가 주요 장애물로 언급됨.
- **에어컨 에너지 팁** — "껐다 켰다 하면 오히려 돈이 더 든다 — 요즘 에어컨은 켜두고 나가는 게 전기료가 덜 나온다."

## 추천·자료

- **GLM(중국 오픈소스 LLM)** — 코딩 보조 및 UI 디자인 생성에서 예상 외의 고성능. 예산 제약이 있는 경우 DeepSeek과 함께 대안으로 언급됨. 오픈라우터를 통해 접근 가능.
- **Voicebox** — 오픈소스 TTS 도구. 본인 목소리 녹음 기반으로 TTS를 만들고 싶을 때 추천됨. STT 용도로도 사용되는 사례 언급.
- **안티그래비티 CLI** — Gemini 기반 CLI. 최근 업데이트 이후 루프 엔지니어링이나 MoA 흉내 내기에 쓸 만해졌다는 평. 단독으로 메인 코딩 도구로는 여전히 비추.
- **공공법령정보 MCP** — @chris_gomdori 계정이 공개한 Threads 포스트에서 언급. 공공 법령 데이터를 MCP로 연결하는 구현으로 채팅방 내 긍정 반응.

## 반응이 컸던 화제

- **GPT 5.6 출시 임박** — 트럼프 행정부 규제 해제 뉴스로 여러 참가자가 연이어 반응. "얼른 나와", "이번 주 금요일" 등 기대감 폭발. Fable 한도 소진 후 대안 모델로 5.6을 기다리던 참가자들이 많았기 때문.
- **Gemini 코딩 비추 공감대** — "원수에게 권하는 모델"이라는 표현을 시작으로 다수가 경험담을 쏟아냄. 삽질 경험 → GLM/GPT 전환 성공 사례가 공감을 얻음.
- **GLM 5.2 디자인 성능 + 로컬 구동 가능성** — "Claude 부럽지 않다"는 주장과 함께 Mac Studio 고용량 메모리 모델 가격 이슈로 연결됨. 예산 제약 사용자들의 대안 모색 맥락에서 주목.

## 자동화·시스템 적용 후보

- **모델 역할 분담 파이프라인** — `설계(Fable/고성능 모델) → 실행(GPT/Codex) → 검증(Opus/Fable) → 핑퐁 리페어` 구조. 비용 효율과 품질을 동시에 추구하는 패턴으로 여러 참가자가 실운영 중.
- **스마트홈 자동화 스택** — SmartThings/ThinQ → Home Assistant → Hermes/OpenClaw 연동으로 온도 기반 에어컨 자동 제어. 스마트 온도계 네트워크 안정성이 핵심 변수.
- **GLM 오픈라우터 연동** — 오픈라우터를 통해 GLM 5.2를 Hermes/OpenClaw/Claude Code에 연결하여 디자인 생성 보조로 활용. 비용 절감이 주 목적.
