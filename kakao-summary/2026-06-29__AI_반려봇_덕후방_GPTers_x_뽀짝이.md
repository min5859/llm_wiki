# AI 요약

## 인사이트·노하우

- **OpenClaw Rate-limit ~1630분 문제**
  - 증상: 사용량이 남아 있는데도 "Rate-limited — ready in ~1630 min" 메시지 발생
  - 원인 후보: 짧은 시간 내 요청 집중, 특정 모델/provider 제한, Codex/OpenAI 계정 단위 쿨다운, 세션 단위 제한, 인증 만료 또는 fallback 라우팅 문제, 서버 retry-after 대기
  - 핵심: 5h·Week 잔여량과 rate limit은 **별도 지표**. 1630분(≒27시간)은 단순 한도 초과가 아니라 provider 쿨다운 또는 인증 문제를 먼저 의심해야 함
  - 확인 명령: `openclaw status --usage`, `openclaw doctor --non-interactive`, `openclaw models auth status`

- **Nous Research 구독 비효율**
  - 구독해도 토큰 단가가 API 직구매 대비 약 10% 추가 제공 수준에 불과 → 구독 실익 없음
  - Ollama Cloud는 thinking effort 변경 불가 (GLM 5.2 기준)

- **Claude(Anthropic) 구독 우회 경로 제한**
  - OpenClaw·Hermes 같은 third-party 하네스를 통한 Claude 사용이 최근 정책으로 막힘
  - 페이블(Fable) 도 차단된 사례 언급
  - 대안으로 OpenRouter 종량제 전환 권장

- **Codex 사용량 비정상 소모 이슈 (2026-06-29 기준)**
  - 이미지 10장 생성만으로 주간 한도 소진된 사례 보고
  - OpenAI 측에서 비정상 소모 버그 인지 및 수정 중이라고 공지
  - 당일 리셋 이벤트 진행됨 (이슈 인정 차원)

- **Codex 요금제 가성비 비교**
  - $200/월 플랜 = x20 배율 → 가성비 최상
  - x5 플랜 2배 이벤트(x10)는 종료됨; x20은 x25로 상향된 적 있음
  - 비즈니스 플랜: 토큰 수 최악, 개인 용도엔 실익 없음 (팀 quota 관리·부가기능이 필요한 기업용)

- **Claude 컨텍스트 토큰 효율화 팁**
  - 스킬 다이어트(불필요한 스킬 제거)
  - auto-compact 후에도 컨텍스트가 가득 찬 상태 방치 → 비효율
  - 하네스 품질 향상으로 세션이 길어져 토큰 소모 증가하는 구조적 원인 존재

- **비트코인 자동매매 시스템 중간 결과 (6/22~6/29)**
  - 장투 -2.69% vs 시스템 -0.3% (수수료 100% 반영 확인)
  - 5개 엔진(E1~E5) 병렬 운용: E1 손절 과다로 조건값·코드 수정
  - "매매 횟수 ≠ 수익 증가", 안정적 수익을 위해 매수 빈도보다 조건 품질 우선
  - 최장 28시간 무매수 케이스 존재 → 정상 동작으로 수용
  - 슬리피지·수수료 미반영 시 성과가 과장됨 → **보수적으로 전비용 포함 계산이 선결**
  - 단타~4시간봉 타임프레임 봇은 "정신건강에 안 좋다"는 경험담 (조건 설정 난이도 높음)

- **Hermes MoA(Mixture of Agents) 기능 패치**
  - 헤르메스에 여러 모델을 혼합 사용하는 MoA 기능 추가됨
  - 설정 예시: GPT-5.5 × Opus 4.8 (OpenRouter 경유), `/moa` 명령으로 호출
  - Opus를 OpenRouter로 연결 시 토큰 소모가 큰 편

- **지침(CLAUDE.md 등) 시각화**
  - 전용 플러그인보다, 파일들을 그대로 넘기고 "HTML로 시각화해달라"고 요청하는 방식이 현실적
  - 상충·중복 항목이 있다면 "최신이 아닌 것 인터뷰해달라"고 추가 지시 권장

## 추천·자료

- **[OpenRouter](https://openrouter.ai)** — 구독제 외 가장 안정적인 다중 모델 종량제 중계. 다양한 모델 선택 가능, 충전 시 수수료 발생(소액 충전도 가능). 사용량이 구독 대비 "훅 날아가는" 느낌이 덜함. 커뮤니티 전반에서 1순위 대안으로 꼽힘.

- **[Mercury Agent](https://mercuryagent.sh/)** — 신규 에이전트 서비스. 아직 찍먹 전 단계로 구체적 평가 미정.

- **[Nous Research 굿즈샵](https://shop.nousresearch.com/collections/products)** — Hermes 일러스트·네온사인 등 AI 에이전트 관련 굿즈 판매. 네온사인 약 70만원대로 고가.

- **Qwen 모델** — "짭 Claude"로 접근하면 의외로 괜찮다는 평. Claude 대안 탐색 시 참고.

- **GLM 모델** — 비전(이미지 이해) 기능 없음에도 디자인 작업에 특화되어 있다는 사용자 평. 중국 모델 중 토큰 소모가 느린 편 → 비용 효율적.

## 반응이 컸던 화제

- **Codex 비정상 소모 버그 + 당일 리셋 이벤트** — 여러 사용자가 동시에 "나도 녹았다", "리셋됐다"고 반응. 사용량이 갑자기 급감했다가 99%로 회복되는 이상 동작이 공통 경험으로 이어짐. OpenAI 측 버그 인정 및 수정 진행 중이라는 정보도 공유됨.

- **OpenRouter vs 구독 모델 비교** — 종량제 수수료 부담에도 불구하고 안정성·모델 다양성·비용 효율 면에서 OpenRouter가 압도적 추천을 받으며 여러 사용자 간 경험 공유로 이어짐.

- **Hermes MoA 기능 (GPT-5.5 × Opus 4.8 혼합)** — 신규 패치 소식 공유 후 실사용 경험 실시간 보고, 기대감 표현 다수.

## 자동화·시스템 적용 후보

- **다중 엔진(E1~E5) 병렬 자동매매 구조** — 각 엔진이 독립적인 조건/전략으로 매수·익절·손절을 수행하고 결과를 집계하는 패턴. 단일 전략 의존도를 낮추는 설계.

- **MoA 설정 패턴** — Hermes에서 `/moa` 커맨드로 두 모델(GPT-5.5 + Opus 4.8)을 혼합 호출하는 방식. 비용 민감 작업에는 GLM·Qwen 등 저비용 모델을 조합하는 전략 적용 가능.

## 질문·미해결

- **Lazycodex** — 사용 경험자가 있는지 질문이 나왔으나 답변 없이 종료됨. 구체적 기능·용도 불명.

- **Claude MCP + Rhino3D 조합** — 라이노 7/8 버전 호환 여부 및 크랙 버전에서의 MCP 연동 가능 여부 질문. 답변 미확인.

- **Vercel AI 기능** — 퍼블리싱을 Vercel로 하는 사용자가 Vercel 내 AI 기능 통합을 고려 중. OpenRouter와의 비교 검토 예정이나 결론 미정.

- **GPT 5.6 / Sol Ultra 출시 일정** — "조만간 풀 거라곤 하는데 명확한 날짜 없음" — 기약 없는 약속 상태.
