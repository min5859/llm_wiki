# AI 요약

## 인사이트·노하우

- **Hermes agent 로컬 모델 선택**: 로컬 모델만으로는 기대 이하. 메인은 유료 모델(Claude/GPT 등)을 붙여야 실용적. 로컬 무료 대안으로는 Qwen3.6-27B > Gemma E4B 순서가 추천됨 (속도·품질 모두 Qwen이 우세하다는 현장 의견).

- **OpenClaw ↔ Hermes 스킬 이식 한계**: OpenClaw에서 수백 번 반복해 완성한 서류 양식 스킬을 Hermes에 그대로 적용하면 동작하지 않음. 원인: 스킬 위치·사용법이 에이전트마다 다르기 때문. 해결 방향: "헤르메스가 쓰는 스킬 포맷으로 변환·맞춤화하는 과정 필요" (P18 발언).

- **OpenClaw 동시 작업 막힘 증상**: 오랫동안 미사용 후 재접속 시 작업을 여러 개 동시에 돌리지 못하는 증상 발생 → 동시 작업 자체는 플랫폼 차원에서 지원됨이 확인됨. 원인은 로컬 환경 꼬임으로 추정 (미해결).

## 추천·자료

- **Qwen3.6-27B**: 로컬에서 Hermes agent 등에 물릴 수 있는 오픈소스 모델. Gemma E4B 대비 속도·품질 모두 낫다는 현장 경험 의견. 유료 API 없이 에이전트를 써보려는 경우의 현실적 차선책.

## 반응이 컸던 화제

- **GPT-5.6 제한 공개 + 한국 접근 여부**: 공개 직후 여러 명이 링크를 연달아 공유하고 한국에서의 사용 가능 여부를 논의. GPT-5.6 하위 모델(Terra, Luna)이 5.5보다 낮은 지능으로 알려졌음에도 접근이 막혀 있다는 점에 공감·웃음 반응.

- **OpenClaw 스킬 → Hermes 이식 불가 경험**: P66의 "수백 번 시켜서 완성한 스킬이 Hermes에서 안 된다"는 발언에 P18이 구체적 해결 방향(포맷 변환 필요)을 제시하며 실질적 토론으로 이어짐.

## 자동화·시스템 적용 후보

- **에이전트별 스킬 포맷 분리 관리**: OpenClaw용 스킬과 Hermes용 스킬을 별도 포맷으로 유지하고, 이식 시 변환 레이어를 거치는 워크플로 도입 검토 가치 있음.
