# AI 요약

## 추천·자료

- **VoiceBox (오픈소스 TTS)** — 30초 샘플만으로 특정 인물의 목소리를 클로닝해 TTS 생성. 무료·오픈소스. 짧은 문장은 품질이 우수하나 긴 문장을 분할 생성할 때 일관성이 떨어지는 한계 있음. [`github.com/jamiepine/voicebox`](https://github.com/jamiepine/voicebox)
- **dubyduby** — Claude Code / Codex CLI를 활용해 외국어 유튜브 영상을 한국어로 자동 더빙하는 오픈소스 툴. 유튜브 URL 입력 → 에이전트 번역 → 성별 감지 → Supertone 온디바이스 TTS 합성까지 자동화. [`github.com/SihyunAdventure/dubyduby`](https://github.com/SihyunAdventure/dubyduby)
- **Microsoft VibeVoice** — ASR(60분 오디오 단일패스) + TTS(90분·4화자) + 실시간 전사(300ms 지연) 통합 오픈소스. 7.5Hz 토크나이저로 음성을 압축. HuggingFace Transformers v5.3 공식 통합. Whisper 대체 후보. [`github.com/microsoft/VibeVoice`](https://github.com/microsoft/VibeVoice)
- **Plaud Note** — 4개 마이크 탑재 하드웨어 녹음기(보조배터리 외형). 전용 앱 내에서 LLM(Sonnet 4.6, GPT 5.5, Gemini 3.1 등) 선택 후 전사·정리. 하드웨어 ~30만 원 + 연간 구독 14만 원(무제한 34만 원)으로 비용 부담 있으나, 시끄러운 환경에서도 녹음 품질 우수.
- **Zenspeak Speakly / 마이크 앱** — 음성으로 바이브 코딩 시 도치법·필러("음", "오", "아") 섞인 말을 자동 정리해서 입력해주는 STT 앱. 로컬 회의 전사 워크플로에 활용 가능.
- **Rona (지피터스 플랫폼)** — AI 최신 정보·인사이트를 실시간 수집·정리해주는 서비스. AI 학습용 큐레이션 피드로 활용 가능. [`rona.so`](https://rona.so/content/d072b119-47b2-4089-841a-e8d4b7c811e0)
- **AMD Ryzen AI Halo** — AI 개발·추론·생성형 AI 콘텐츠 제작용 소형 개발 컴퓨터. x86-64 기반, Windows 완전 지원. DGX Spark·Mac Mini 대비 x86 생태계 호환성이 강점.

## 인사이트·노하우

- **VoiceBox 로컬 운영 팁**: 짧은 문장 단위 TTS는 품질이 높으나, 긴 문장을 분할해 생성하면 목소리 일관성이 떨어짐. 긴 텍스트는 한 번에 넣거나 다른 방법 검토 필요.
- **화자분리(Speaker Diarization) 난이도**: 단순 전사(STT)는 Whisper 등으로 쉽게 해결되지만, 화자를 구분하는 화자분리는 별도 기술이 필요하며 현재 오픈소스 수준은 아직 불안정. Plaud Note + Sonnet 4.6 조합이 실용적 대안으로 평가됨.
- **Whisper vs. 기타 STT 비교**: 일반 텍스트 전사·싱크는 Whisper도 충분하나, 노래 싱크 맞추기에서는 Whisper(앞·뒤 싱크 불량)와 ElevenLabs(중앙부 싱크 불량) 모두 한계 있음.
- **ElevenLabs 활용 패턴**: ElevenLabs로 보이스 디자인만 하고, 생성된 캐릭터 보이스를 VoiceBox에 입력해 실제 TTS 생성하는 방식으로 비용 절감 가능.
- **OpenClaw + Hermes 동시 운영**: 두 에이전트를 같은 서버에서 운영 시 한쪽이 죽으면 다른 쪽이 재시작시키는 상호 감시 구조로 안정성 확보. "한쪽이 지휘"가 아닌 용도별 분리 사용이 일반적.
- **OpenClaw 메모리 세팅**: 메모리 서치를 Ollama 로컬로 세팅해 운영 가능. OpenClaw에게 직접 물어보면 세팅 방법을 알려줌.
- **Hermes 세션 관리**: 기본적으로 각 채팅방 세션별 독립 관리. 세션을 넘나드는 기억을 구현하려면 별도 설정 필요.
- **벡터 메모리 + 로컬 CPU AI 한계**: 약 4,000행 벡터화 데이터를 로컬 CPU AI로 운영 시 응답 속도가 느림. GPU 기반 환경에서는 실용적으로 사용 가능한 것으로 보임.
- **Claude Code에서 Codex CLI를 exec 방식으로 추출**: Claude Code 내에서 Codex CLI를 exec 방식으로 실행하는 방법이 공유됨 (구체적 단계는 대화에 미기재 — 후속 확인 필요).
- **루프 엔지니어링**: 자동화 난이도가 높아지는 추세에서 루프 엔지니어링이 핵심 역량으로 부상 중이라는 언급.
