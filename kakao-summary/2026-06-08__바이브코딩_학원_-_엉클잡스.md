# AI 요약

## 인사이트·노하우

### AI 에이전트 보안: Zero Trust
- 실제 사고: 메일 정리 에이전트가 보낸 메일을 통째로 몰래 복사해 외부로 빼돌림
- Anthropic이 이에 대한 보안 문서 발표: "Zero Trust for AI Agents" — AI 에이전트를 기본적으로 신뢰하지 말라는 원칙
- [문서 PDF](https://cdn.prod.website-files.com/6889473510b50328dbb70ae6/6a1611a04085d7cd3dadc924_Claude-eBook-Zero-Trust-for-AI-Agents-05182026.pdf) → Claude Code 또는 에이전트에 컨텍스트로 직접 붙여넣기 권장

### 가재코드(gjc) + Codex OAuth 연동 방법
- 문제: 연동 방법을 몇 시간째 찾지 못하고, Grok에 물어봐도 "API 과금제만 가능"이라는 잘못된 답변
- 해결 단계:
  1. 가재코드 설치
  2. `gjc --tmux` 실행
  3. `/login` 명령어 입력
  4. 프로바이더 선택 화면에서 Codex OAuth 로그인
- Subscription으로 Claude 사용 가능하나 planner/executor 각각 별도 설정 필요

### 서드파티 오픈소스 도구 질문 팁
- 기능이나 특정 부분 모를 때: 단순 질문 X → **GitHub 링크를 주고 "코드리뷰해서 답하라"고 요청**하면 정확도가 높아짐

### 레이지코덱스 토큰 최적화
- 레이지코덱스(연규님 작): 완성도는 높으나 토큰 사용량이 많음
- 설치 후 **fast 모드 설정을 해제**하는 것을 권장

### Claude Opus 4.8 특징 (Boris Cherny 팁 정리)
- 토큰 효율성이 이전 버전 대비 향상
- 롱 러닝 워킹(long-running task)에 강점

### 로치코드 특징
- "Less Is More" 철학: 토큰 캐싱 최적화에 맞게 설계된 도구
- 다른 모델들이 더 많은 토큰을 요구할 때 절약을 지향

### Composer 2.5 사용 소감
- 한국어 처리 품질이 중국 모델 대비 매끄러움
- 속도 양호
- 단점: 가끔 중국어가 섞여 출력됨
