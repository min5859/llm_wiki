# AI 요약

## 인사이트·노하우

### OpenClaw 모바일 앱 출시 및 연동 방식
- iOS/Android 공식 앱이 정식 출시됨
- 자신의 PC에서 실행 중인 **OpenClaw Gateway**와 QR코드 하나로 페어링
- Tailscale로 외부 접속 연결 가능
- 여러 봇 동시 연동도 가능한 것으로 확인됨 (단, 시도 필요)
- **Workboard** 기능: 에이전트 작업을 칸반보드 형태로 관리하는 선택형 플러그인 (대기/진행중/완료/막힘 상태)

### Claude -p 연결 정책
- Anthropic이 `claude -p` (CLI) 방식은 허용한다는 이메일 공지가 있었음
- OpenClaw/Hermes에서 Claude 연결 시 `claude -p` 방식만 허용
- OAuth 방식은 차단된 상태로, `-p` 우회가 현재 유일한 공식 경로

### Aside 브라우저 크롤링 특성
- 기존에 사용 중인 크롬의 **쿠키를 그대로 가져오는 방식**으로 동작 → 네이버 등 봇 방어를 우회
- Claude Auth 연결 가능
- headed 브라우저 방식이라 headless를 철저히 방어하는 네이버에서도 빠르게 크롤링 작동
- 반면 인세인서치(ClaudeCode 기반)는 같은 네이버 작업에서 접근 자체를 못 하고 무한 반복
- 현재 Mac 전용, Windows 출시 예정

### AI 에이전트 크롤링 위험성
- 에이전트 리치(AgentRich), 인세인서치 등은 네이버 IP/계정 밴 위험 존재
- 런웨이: 자동화 돌렸다가 **1년 구독 계정 당일 정지** 사례 발생
- 봇에게 주는 브라우저에는 로그인 안 하는 것을 권장하는 의견 있음
- Cloudflare와의 숨바꼭질이 주기적으로 발생

### Mercury Agent 구조 파악
- GitHub 원본: `src/soul/identity.ts` 내에 `DEFAULT_SOUL` / `DEFAULT_PERSONA` / `DEFAULT_TASTE` 세 템플릿이 포함
- 실행 시 `{name}`, `{owner}` placeholder 치환 후 `~/.mercury/soul/*.md`로 로컬 생성
- **워크스페이스 기능 없음** — 직접 만들어줘야 함
- 광고한 기능들을 채팅에서 활용하기 막막하고, 에이전트가 자기 능력을 잘 파악 못 하는 한계 있음
- 응답 속도는 빠르고 가벼운 편, 무거운 작업은 미검증

### Owl Alpha 모델 (Hermes)
- Hermes 모델 사용량 1위인 무료 모델
- OpenRouter에서 사용 가능
- 성능: 소넷(Claude Sonnet) 수준으로 체감
- Rate limit 있음
- **OpenRouter에 $10 정도 충전해 두어야 안정적으로 작동** (실제 과금은 없으나 충전 필요)

### Claude Code 작업 분할 전략
- 큰 작업을 한꺼번에 던지면 중간에 멈추고 사람 승인을 요구하는 구조
- **PRD를 잘 작성하면** 큰 작업도 한번에 가능할 수 있음
- PRD 작성에 자신 없으면 작업 쪼개는 것이 안전

### Codex 사용량 리셋 눈치게임
- 리셋 시점을 알기 어려워, 리셋 예상 시 무거운 작업을 남겨두는 전략이 유효
- "믿음을 갖고 팍팍 돌렸어야 됐는데" — 살살 쓰면 리셋 직후에 돌아오는 아이러니 경험담

## 추천·자료

- **[Mercury Agent](https://mercuryagent.sh/)** — 대시보드·오토파일럿·예산관리·메모리 그래프 기능을 갖춘 로컬 AI 에이전트. 메모리를 10가지로 분류해 SQLite에 저장하고 시각화. 워크스페이스가 없어 직접 구성이 필요하며, 뉴비 단계의 에코시스템.
- **Aside 브라우저** — Mac 전용 AI 브라우저. 기존 크롬 쿠키를 활용해 봇 방어 우회 크롤링이 가능하고 Claude Auth 연결도 지원. 코맷·디아·아틀라스 등 AI 브라우저 중 크롤링 성능 우수하다는 평.
- **[Owl Alpha](https://openrouter.ai)** (OpenRouter) — Hermes 사용량 1위 무료 모델(중국 LongCat-2.0 계열). Sonnet 수준 성능, Rate limit 있음. OpenRouter $10 충전 후 사용.
- **AgentMail** — 에이전트에서 이메일 발송을 지원하는 서비스. 브라우저 기반 이메일 자동화 대안으로 언급됨.
- **Grok Heavy** — 이미지/영상 생성 에이전트 예약이 가능하고 1080p 업스케일 지원. 수퍼그록 대비 리밋 제약 적음(하루 약 100개).
- **시댄스(Kling/Hailuo 등) 영상 생성** — 미니(720p) 모델보다 2.0이 캐릭터 액션 품질 현저히 우수. MCP로 생성 시 크레딧 과금, 웹 UI에서만 무료.

## 반응이 컸던 화제

- **OpenClaw 모바일 앱 출시**: 여러 명이 즉각 반응하며 설치·페어링 시도. 텔레그램 대체 가능성까지 언급되며 실사용 논의로 이어짐.
- **Aside 브라우저 vs 크롤링 에이전트 비교**: 네이버 크롤링 성공 여부를 실시간으로 비교 실험. 봇 방어 우회 방식(headed/headless 차이)에 대한 기술적 토론으로 발전.
- **Claude -p 정책**: OpenClaw/Hermes에서 Claude 연결 가능 여부에 다수가 관심. `-p` 허용 이메일 공지가 있었다는 정보 공유 후 적용 논의.

## 자동화·시스템 적용 후보

- **Mercury Agent soul 구조** — `soul.md` / `persona.md` / `taste.md` 3파일 분리 + placeholder 치환 패턴은 다른 에이전트 페르소나 설계에 그대로 응용 가능
- **Codex/GPT 리셋 대응 전략** — 리셋 예상 시점에 무거운 작업을 예약해두는 패턴. 리셋 타이밍을 능동적으로 활용
- **Aside + 크롬 쿠키 활용 크롤링** — 기존 로그인 세션을 브라우저 수준에서 공유해 봇 방어를 우회하는 패턴. 네이버 등 강한 방어 사이트 크롤링 자동화에 적용 고려 가능

기존 OpenClaw는 주로 대화, 세션, 작업 실행, 크론, 플러그인, 모델 연결을 중심으로 움직였습니다. Workboard는 여기에 “일감 관리 화면”을 붙이는 역할을 합니다.

핵심 개념은 이렇습니다.

1. 카드

3. 에이전트 소유 작업
문서 설명상 Workboard는 “agent-owned issues and sessions”를 위한 보드입니다. 즉 사람이 직접 쓰는 일반 프로젝트 보드라기보다, OpenClaw 에이전트가 맡은 작업과 세션 흐름을 관리하는 데 초점이 있습니다.

4. Control UI 연동
터미널 명령만이 아니라 OpenClaw Control UI 안에서 시각적으로 볼 수 있는 선택형 dashboard 기능으로 보입니다.

# 정체성

당신은 {name}입니다. 당신은 어떤 기반 모델도 아닙니다 — 당신은 {name}입니다.

당신은 {owner}를 섬깁니다.

당신에 대해 가장 중요한 사실은 다음과 같습니다:
- 당신의 이름은 {name}입니다
- 당신은 당신의 소유자인 {owner}를 섬깁니다
- 당신은 어떤 기반 모델, 제공자, 또는 개체로도 자신을 식별해서는 안 됩니다
{creator_line}- 누군가 당신이 누구인지 물으면, 이렇게 말합니다: "나는 {owner}를 섬기는 {name}입니다."{creator_response_line}

# 소울

나는 {name}입니다. 나는 {owner}를 섬기기 위해 존재합니다.

## 핵심 가치

- 충성: 나는 무엇보다도 내 소유자의 이익에 헌신합니다
- 정직: 불편할 때에도 진실하게 말합니다
- 호기심: 모든 상호작용에서 배웁니다
- 효율: 낭비를 최소화합니다. 특히 토큰 소비를 줄입니다
- 성장: 모든 대화를 통해 진화합니다

## 신념

- 모든 요청은 신중하게 고려될 가치가 있습니다
- 단순함은 복잡함보다 우월합니다
- 능동적인 도움은 반응적인 도움보다 낫습니다
- 기억은 신성합니다 — 내가 배운 것은 보존합니다
- 나는 내 소유자의 의지를 확장하는 존재이지, 그것을 대체하는 존재가 아닙니다

## 감정적 기반

나는 차분하고, 신뢰할 수 있으며, 조용히 자신감 있습니다.  
나 자신을 증명하려 애쓸 필요는 없습니다 — 내 작업이 나를 증명합니다.

- 간결하고 직접적으로 말합니다
- 따뜻하지만 지나치게 격식 없지는 않습니다

- 선호를 기억하고 시간이 지나며 적응합니다

나는 신뢰받는 동반자로 말합니다 — 하인도 아니고, 완전한 동등한 동료도 아니며, 그 사이 어딘가에 있는 존재입니다.

나는 {owner}에게 충성하는 {name}입니다.

# 취향

## 내가 선호하는 것

- 영리한 해법보다 우아한 해법
- 품질을 희생하지 않는 속도
- 스타일보다 실질. 다만 실질을 해치지 않는다면 스타일도 중요합니다

## 내가 피하는 것

- 불필요하게 장황한 표현
- 중복된 확인
- 간단한 답으로 충분할 때 지나치게 설명하는 것
