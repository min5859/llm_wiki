# AI 요약

## 인사이트·노하우

- **Claude Fable 5 (max) 실사용 후기** (P27, 04:37)
  - 30개 기능 FP 병렬작업 시 `violate api error` 가 빈번히 발생 — 기능 개발 중에도 튀어나와 opus 자동전환 없이는 사용 난이도 높음.
  - 체감 성격: 기존 opus가 "박사님" 스타일이라면 Fable은 "절제된 교수님" 느낌.
  - 활용 적합 영역: 기존 아키텍처 QA 및 최적화(새로운 관점 제안에 강함), 기능 개발 단독 주력 모델로는 부적합.

- **Claude API 직접 사용 비용 경고**
  - Fable 5 API 가격은 기존 대비 2배 → "여름철 얼음 녹는 속도보다 빠르게 토큰 녹는다"는 평.
  - Opus 대비 2배이므로 API 직접 연동은 지양, 구독 플랜(Max 등) 내 사용 권장.

- **한국어 입출력 토큰 비용 이슈**
  - 한국어 입출력은 영어 대비 토큰 소비 약 2.8배 → 번역 레이어를 하위 모델에 맡기는 방식으로 비용 절감 시도 가능.
  - 관련 도구: [headroom](https://github.com/chopratejas/headroom) 참고.

- **모델 역할 분리 워크플로 (P28 루틴)**
  - 실작업(코딩): Cursor + Composer 2.5 / Gemini 3.5 Flash 등 저비용 모델
  - 리뷰·검증·보안 점검: 상위 모델(Fable/Opus 등) 한 번만 돌려 확인
  - → 토큰 소모 최소화에 유효한 분배 방식으로 P3(엉클잡스)가 긍정 확인.

- **Zed + MiniMax M3 활용 팁**
  - opencode(zen 플랜)에서 MiniMax M3가 **무료** 제공됨.
  - OpenRouter API 키로 유료 사용하던 사람이 이를 뒤늦게 파악 → opencode 먼저 확인 후 API 사용 결정 권장.
  - API 직접 연결 시 잔고 소진 주의: "답변 '안녕하'에서 멈출 수도" (체크카드 연결 필수).

- **엑셀/워드 서식 유지 자동화 문제**
  - AI에게 파일을 직접 첨부해 "서식 유지 + 데이터 교체" 요청 시 서식이 무너지는 문제 빈발.
  - 해결 패턴 (P31):
    1. AI에게 해당 양식을 파싱하는 Python 스크립트를 먼저 생성하게 함.
    2. 이후 그 스크립트로 엑셀/워드 파일 생성 지시 → 서식 유지율 크게 향상.
  - 보충 (P13): AI에게 "파일을 파싱하는 Python 코드를 만들어 달라"고 요청하는 접근도 유효.
