# AI 요약

## 인사이트·노하우

- **Fable 5 활용 팁**: `claude -p` 비대화형 방식은 Fable 5의 진면모가 드러나기 어려운 환경. 사용량 제한에 금방 걸려 폴백으로 돌아가므로, Hermes 쿠션 없이 Claude Code 직접 실행이 더 효율적이라는 의견.
- **Codex Pro 사용량 감소 원인**: x5/x20 요금제의 2배(또는 4배) 이벤트 종료로 인한 체감 감소. x5 플랜 사용자가 가장 크게 체감함.
- **카톡 봇 단톡방 퇴장 방법**:
  - CLI 방식(kakao-cli)으로는 스스로 나가기 불가.
  - LOCO 프로토콜 방식이면 스킬로 구현 가능.
  - Computer Use(UI 자동화) 방식으로 화면 제어해 버튼 클릭으로 퇴장 가능 (macOS 기본 기능, OpenClaw/Hermes/Codex 모두 활용 가능).
- **LOCO 연결 시 Agent 불필요 가능성**: "카톡을 LOCO로 연결하면 응답할 때 굳이 AI Agent가 필요없지 않나" — 직접 응답 파이프라인 단순화 검토 필요.
- **Hermes 슬랙 vs 텔레그램 톤 차이 해결**: 채널 간 말투가 다를 경우, Personality Preset과 Soul 설정이 각각 어디에 들어가 있는지 확인 → 시스템 프롬프트 비교로 진단. Personality Preset에 말투 설정이 있는지 먼저 체크.
- **멀티에이전트 시스템 설계 사례 (P85)**: 오케스트레이터(아르고) + 역할별 전용 에이전트 구조. 업무용/개인용/교육지원용/세무기장용 등 목적별 분화. Claude 메인에 멀티 LLM 4개 구현, 오케스트레이터가 필요한 에이전트를 호출하는 방식. 4월 말 동기부여 → 5월 1일 본격 구축 → 실무 투입까지 약 6주.
- **웹게시판 봇 운영 방식 (P27)**: 슬랙 대신 웹게시판 사용. 스크립트로 1분마다 자신을 호출한 유저 확인 → `claude -p` 방식으로 답변 댓글 작성. 15일 이후 API 토큰 방식으로 전환 검토 중.
- **데이원컴퍼니(패스트캠퍼스·마이라이트·콜로소) 개인정보 유출**: GitHub 마스터 계정 키 탈취 → 5월 9일 첫 침입. 이름·전화번호·암호화된 비밀번호 유출. 해당 서비스 이용자는 비밀번호 변경 권고.
