# AI 요약

제공된 대화 원문만을 바탕으로, 장기 지식 가치가 있는 내용만 아래 카테고리별로 정리했습니다.

## 인사이트·노하우

- **Raycast 2.0 (macOS 중심)**: 대화에서는 “2020년 이후 최대 규모 업데이트”로 언급되며, 베타 동안 OpenAI의 **GPT‑5.4 mini 무료 제공**, **Spotlight 기능(맥락 기억)**, **받아쓰기 내장**(무료 체험 가능) 같은 포인트가 나옵니다.

- **API/구독 모델 사용량 관리 패턴**: Codex 크레딧·주간 한도 소진 후 **Claude 주간 한도 복귀 여부**(“자고 나니 되돌아왔다” 등)·다른 플랜·도구 폴백 조합 등이 현실적인 운영 이슈로 다뤄집니다. 한 참여자 언급에 따르면 Claude 쪽에서는 “아껴 쓴 부분만 리셋”되는 등 **표시되는 한도·리셋 규칙이 기대와 다를 수 있다**고 합니다.

- **Hermès + Spotify**: “실행 중인 기기의 플레이리스트와 재생”을 원격처럼 제어할 수 있고, 같은 기기가 아니어도 된다고 합니다. Spotify Developer에 앱을 만들어 연동하는 난이도는 높지 않게 느꼈다는 취지의 발언입니다.

- **OpenClaude(오픈클로/오픈클로 패치 명칭) 환경별 이슈**: Mac은 괜찮았지만 **WSL·Windows·Android 패치 이후 OpenAI 모델명이 깨져 채팅이 안 됨**이 제보되었고, **Hermès로 교정**했다는 실제 해결 경로가 적혀 있습니다(원문에 구체 디버깅 단계 나열 없음).

- **Hermès에서 Grok를 GPT처럼 OAuth로 묶어 쓰기**: 오픈클로에는 Grok OAuth가 없어 **Hermès를 경유하는 설정으로 연결**했다는 사례가 있습니다.

- **Grok 활용 성격(발언 요약)**:
  - “기본이 **에이전트 4명 협업**이라 속도가 빠르다”“코딩·정확도는 과대기대 불가”“이미지·영상 속도 빠르고 크레딧 소모 부담이 적다”“웹·앱의 상위 동작에서는 에이전트가 많이 붙어 결과도 나쁘지 않았다” 등 피임상 레벨 특징이 정리되어 있습니다(NSFW 성격은 주관적 진술 참고만).
  - 참고 표현으로 “시댄스(Sedance) 급은 아니고”“영상 자동화는 **GPT 이미지 + 시댄스**가 최상급이라는 식으로 양자가 비교”되는 문장이 있습니다(‘시댄스’의 정식 제품 명칭 불명 시 문맥상 영상 생성·워크플로 도구 일종으로 받아들였습니다).

- **Grok 구독/상품 레벨**(대화 속 정보):
  - **SuperGrok Heavy(약 \$300 안팎이라는 진술)**가 있어야 **Grok Build**(CLI 기반 에이전트 빌드) 베타를 쓸 수 있다는 설명과 [x.ai/cli](https://x.ai/cli) 안내 링크가 공유됩니다.
  - **슈퍼그록 프로모**(탈퇴/결제 버튼 흐름 언급)로 **\$30급 금액으로 3개월 이용**처럼 보이게 되는 패턴 소개 등이 있습니다(약관·변경 가능성 불명 표기).

- **카카오톡 연동 봇 “답변 한 번 하고 접속이 끊김·킥” 트러블슈팅 조각**  
  원인 특정 불가 상태에서 나온 **가설/경험**만 정리했습니다:
  - Mac 미니 등에서 **접근 권한 팝업**(사용자가 직접 확인 전까지 같은 증상) 경험
  - “다른 기기 인증”으로 흔히 킥된다는 일반 논리
  - 카카오톡 PC 메인 로그인·캐시가 **서브(lo로 접속) 계정 킥**을 유발할 수 있다는 추측
  - 증상: 답변과 킥이 거의 동시, 아이패드 접속 불가까지 겹치는 사례
  ⇒ **합의된 단일 원인·재현 절차는 대화 안에 확정되어 있지 않습니다.**

- **매우 구체적인 macOS 트러블 경험: `tccutil`(TCC) 실수 패턴과 복구 한계**
  - **문제**: `tccutil reset ScreenCapture / Microphone`이 **카테고리 전체**(해당 서비스 권한)를 초기화하는 형태였는데, **특정 앱만 초기화**하려면 `tccutil reset <SERVICE> <BUNDLE_ID>`여야 했음을 사후에 깨달음.
  - **복구**: “**TCC.db는 보통 자동 백업이 없어 복구 불가**”“한 번에 원복 방법 없음”이라는 진술이 포함됩니다.
  - 영향 가능 앱 목록이 예시로 열거됩니다: ScreenCapture(Zoom·Teams·Meet·OBS·스크린샷 도구 등), Microphone(카톡·Discord·Slack·통화 앱 등).  
  - 이후 해당 앱을 다시 켜거나 시스템 설정에서 재허용 필요.

- **Codex GPT‑5.5 관련 기능 저하(대화 전달 표현만)**: “Codex에서 GPT‑5.5 기능 저하에 대한 문제 2가지를 발견·수정, 모니터링 후 저녁에 사용 제한 재설정”이라는 **공식/반공식 브리핑을 인용**하는 메시지가 있습니다(사용자가 직접 검증했다는 증언은 해당 인용 블록에 한함).

- **헤르미스 업데이트 시 의존성 diff가 마이너스(다운그레이드처럼) 보이는 이유 논점**: 의존성 핀을 **최신이 아니라 호환성 기준으로 맞춘다**(“최신이 곧 무조건 좋진 않음”)는 설명 방향과, 업데이트가 너무 길면 취소 후 재시도 노하우가 간단히 교환됩니다. 한 참여자는 업데이트 후 **실제 실행은 웹만 되고 게이트웨이 등이 끊긴 상태**였다는 회고도 했습니다(개별 설정 이슈).

- **헤비·고가 구독 모델 툴링 체감**: Grok Heavy에서 “무제한”처럼 느껴졌으나 곧바로 “툴콜링이 느려 응답 상태를 확인해야 한다”“DeepSeek V4 Flash 등 다른 모델이 훨씬 나았다”“환불 고려·환불 결정”까지 이어진 **주관 비교**(도구 이름·숫자는 원문 레이블 근거)가 있습니다.

- **Hermès 업적(achievement)·대시보드**: 기능 사용량 같은 것을 재미 삼아 보여주는 기본 기능으로 언급됩니다(자체 플러그인이라기보다 “대시보드에 있는 기능”이라는 진술).

- **Hermès Whisper(위스크) 관련**(대화 맥락상 Google 일부 기능 통합 후 제품 이름이 헷갈리는 사용자 경험): “위스크 기능이 플로우에 통합”되고 “위스크는 거의 버려진 상태”처럼 **제품 라인 변화 체감**이 서술됩니다(현재 상태는 불명, 대화 속 인상만 보존).
