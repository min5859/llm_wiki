# AI 요약

## 인사이트·노하우

### 텔레그램 봇 단톡방 구조
- 텔레그램 Bot API 는 기본적으로 봇이 다른 봇 메시지를 읽지 못함(privacy mode 꺼도 동일).
- 약 2주 전 업데이트로 **봇파더에서 `/setbot2bot` 토글을 활성화**하면 봇 간 메시지 수신이 가능해짐 — P2 제보.
- 토글 없이 구현하려면: 사용자가 `/team` 입력 → 백엔드(Hermes 등)가 각 봇을 순서대로 호출 → 각자의 토큰으로 같은 단톡방에 답변 발송하는 orchestration 구조로 우회.
- 봇들끼리 자유롭게 떠들게 두면 무한루프 위험이 큼 → 권한·트리거 설계 주의.

### 로컬 LLM 하드웨어 경험치
- **M4 Pro 24GB**: 가벼운 테스트는 되지만 본격 운용은 포기 사례 다수(P5). 로컬 모델 안 돌려도 RAM 20GB 점유.
- **Mac Mini 24GB → 32GB 업그레이드**: Gemma 4 26B 구동 가능, RAM 점유 80–90% (P8).
- **128GB**: Gemma 4 26B 돌려도 108–110GB까지 잡힘. "체험용으로는 좋은데 뭔가를 해보시겠다는 부족함" (P9).
- 결론적 분위기: "로컬 LLM 깔아 쓰는 것보다 AI 구독이 가장 싸다" — P10, P9 의 중론.
- Ollama 허브에서 받는 모델 다수가 이미 **4bit 양자화** 상태라는 점을 P8 이 뒤늦게 발견.

### 모델 선택 노하우
- **Gemma 4 26B**: 헤르메스에서 무난히 돌아감, P8 의 일상 운용 모델.
- **Qwen 3.6 30B (MoE)**: 32GB RAM 이면 양자화 필수. 코딩은 약하지만 그 외에는 Gemma 보다 낫다는 평(P6, P15).
- **Qwen3 Coder Next**: 3.6 보다 코딩 용도에서 추천된다는 언급(P14).
- **NVIDIA Nemotron Nano Omni** 신모델 — 시도해볼 후보로 거론.

### Claude CLI 사용량 / 한도
- Claude 구독(Pro 추정)에서 헤비 질문 시 **5시간 할당량의 약 15% 차감/질문** → 사실상 5시간에 5질문 수준.
- **Max 플랜이라도 ~25질문/5h** 가 한계라는 P14 의 체감치.

### Claude Code 의 "OpenClaw" 차단 이슈
- 깃 커밋 메시지에 `openclaw.inbound_meta.v1` 같은 JSON blob 이 들어가면 Claude Code 가 요청 거부 또는 추가 요금 청구 메시지 띄움.
- `claude -p "..."` 로 직접 호출하면 정상 → **git log 자체를 검사**하는 것으로 의심됨.
- 빈 repo 에서 커밋만 남겨도 재현됨.

### 텔레그램 출력 정뚝떨 (verbose 로그)
- 헤르메스/오픈클로 업데이트 후 봇이 작업 내부 코드·경로를 줄줄 출력하는 증상.
- 해결: **`/verbose off`, `/trace off`** (P19). 또는 `/reasoning off` 시도.
- 디자인 선택지: 진행 상태 확인용으로 일부러 켜두는 사용자도 있음(P18) — 최종 결과 시 자동 정리되는 것이 정상 동작.

### API 키 보안 — `openclaw.json` 평문 저장 회피 3가지
1. 환경변수 처리
2. OS 키체인 저장
3. 도커에 실제 키 주입 + 전역에는 더미 키, 실행 시 도커에서 호출 — P19 정리

### 도커 vs 전역 설치 (OpenClaw/Hermes)
- 도커 능숙하면 샌드박싱이 안전. 다만 도커 내부 파일 추출이 익숙하지 않으면 전역 설치가 단순(P2 경험).
- 하이브리드: 도커에 API 키만 두고 호출 시점에 가져오기.

### 워크플로 사례
- **크론으로 N분마다 채팅 N개 읽고 요약** 까지 `gemini -p` 로 자동화 가능(P10) — 다만 밴 우려로 잠시 꺼둠.
- 오픈클로로 ERP 자동화 구축 → 문서 작업은 헤르메스로 이관 운용(P5).
- 헤르메스 체감: "오픈클로보다 빠릿빠릿, 일 잘함" (P25). 단 토큰 소모량은 더 많을 수 있다는 풍문(P26) — 사용 방식 따라 다름.

### 멀티 에이전트 컨텍스트 분할 고민
- 서브에이전트에 "딱 필요한 것만" 주는 게 어렵다는 P22 의 토로 — 역할별 분리가 애매하고, 통째로 주면 분할 의미가 없는 트레이드오프.
