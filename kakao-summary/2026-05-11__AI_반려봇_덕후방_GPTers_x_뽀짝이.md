## 인사이트·노하우

- **카카오톡(딸기 등) 폴링 주기**  
  설정 화면에 “폴링 초” 같은 값은 안 보이고, 확인되는 명시 주기는 **내부 디스패처 60초**뿐이라 **수신 폴링은 어댑터 기본값**으로 도는 것 같다는 관찰이 나옴.

- **봇 응답에 내부/메타 문구가 새는 현상**  
  사용자에게 나가면 안 될 문구가 채팅에 노출된 사례: `"output, while staying in character."`, `"방금 흐름 보면 '…' 얘기였네."`, `"지금 막힌 지점 한 줄만 더 줘!"`  
  다른 참가자는 **헤르메스 에이전트에서도 비슷한 현상**을 본 적이 있다고 함.

- **키워드 트리거 봇**  
  `"뽀짝"` 키워드에 **자동 반응**해 등장하는 고양이 페르소나 봇이 있음(의도 없이도 트리거될 수 있다고 본인 설명).

- **Slack에서 “동생봇이 메인처럼” 보이는 현상(추정 체크리스트)**  
  1) **같은 Slack 봇 토큰**  
  2) **User ID 충돌·유사**  
  3) **웹훅에서 새 봇 필터링 미흡**

- **Discord에서 봇 2개가 서로 스레드를 못 읽을 때(정리된 조치)**  
  - 서버 역할 권한: **메시지 전송**, **메시지 읽기 기록**, **메시지 콘텐츠**(스레드 필수로 언급)  
  - 코드 Intent: `discord.Intents.default()` 후 **`intents.message_content = True`**  
  - 봇 간 협업: **공유 DB**, **Webhook 상호 호출**, **채널/스레드 ID 하드코딩 지양**  
  - 안 되면 **토큰 재발급** 재시도

- **헤르메스 이슈: 리트라이·지연·429**  
  증상: 대화 중 갑자기 실패, **`http 429`**, API 잔액은 남음 등.  
  대화 속 완전한 원인 확정은 없음. 제안으로는 **rate limit 대응**, **`gpt-5.4 mini` 언급 후 재연결**, **`hermes model`(터미널)** 재설정 등이 나옴.

- **OpenClaw 웹서치가 느려지는 고리(운영 쪽 Q&A 요약)**  
  흐름: **WebSearch가 게이트웨이에서 실패 → 모델 재시도 → 답변 지연**  
  대응 갈래:  
  1) **`settings.json`에서 WebSearch 비활성화**(가장 빠른 답으로 소개)  
  2) **Codex 백엔드 / web_fetch / chrome relay 스킬로 우회**  
  (상세는 아래 링크 섹션의 Q&A URL)

- **Codex를 붙이면 웹검색 툴 없이도 검색이 된다는 후기**가 있음(환경·버전에 따라 다를 수 있음).

- **헤르메스 칸반 활용 팁(실사용 후기 성격)**  
  - **최신 패치** 필요 언급  
  - “할 일 있으면 칸반으로 계획 세워달라”면 **카드 생성**(코드처럼 **알파벳+숫자** 형태로 만든다고 함)  
  - 협업은 **병렬 가능 여부를 물어보며** 직렬/병렬 왕복  
  - **`yolo background goal`** 설정 후 칸반 카드를 진행시키고, **카드 하나 끝날 때마다 알려줘** + **크론잡**을 켜면 완료 알림이 오는 식으로 운용했다는 사례  
  - **프로필끼리 협업도 가능**하다는 언급

- **여러 봇 분업(디스코드 외 일반적 후기)**  
  **DM으로 지시**하면 분업이 된다는 경험 담.  
  **테일스케일** 등으로 같은 기기에 여러 봇 경로를 알고 있으면 **`OpenClaw`, `Hermes agent` 폴더 “찾아”** 같은 지시로 서로 찾게 할 수 있다는 식의 운영 팁.

- **마누스 크레딧 체감**  
  - 프로여도 **작업에 따라 1~2시간이면 고갈** 수준일 수 있다는 체감  
  - 영상 생성은 특히 크레딧이 크게 든다는 후기(예: **Veo 3.1 8초 기준 600 크레딧** 언급)  
  - 작업당 대략 **300~600**, 무거우면 더 나간다는 범위감  
  - **래퍼 서비스에서 에이전트 기능은 크레딧이 빠져나가기 쉽다**는 경고(젠스파크 사례 언급)

- **마누스 로그인/가입 혼선(당일 채팅 관찰)**  
  - **안드로이드 앱에서만 특가/결제 UI가 보였다**는 공통 관찰  
  - **구글·메일 가입이 막히거나 로그인 에러**, **페이스북/메타/네이버 우회** 시도  
  - **트래픽 폭주**, **신규 가입 제한**, **계정 정지** 후기 다수  
  - **환불**: 부분 환불·순차 환불·구글플레이 환불 요청 등 혼재; **취소 후 4000 크레딧 한 달 프로 같은 보상** 관측도 나옴(일관된 공식 설명은 입력에 없음)

- **Manus API 호출 요약(참가자가 공유한 스모크 기준)**  
  - 환경변수 `MANUS_API_KEY`  
  - 헤더에 **`API_KEY`(실키)** + **`Authorization: Bearer placeholder`도 동작했다고 함** + JSON  
  - 생성: `POST https://api.manus.im/v1/responses`  
    바디에 `"task_mode": "agent"`, `"agent_profile": "manus-1.6"` 예시  
  - 폴링: `GET https://api.manus.im/v1/responses/{TASK_ID}` until status가 running이 아님  
  - 결과 텍스트는 대략 **`output[].content[].text`**  
  - 부가로 **`metadata.task_url`**, **`metadata.credit_usage`** 언급

- **LM Studio 로컬 모델 TPS 대략 측정**  
  `time curl`로 로컬 **`/v1/chat/completions`** 호출 후 시간으로 나누는 방법 소개, 또는 **LM Studio UI의 TPS 표시** 확인.

- **카카오에서 붙은 에이전트의 제약 추정**  
  참가자 추정: 카카오 환경에서 **툴콜링 횟수·최대 출력 토큰 제한** 때문에 동작이 달라 보일 수 있다는 이야기.

- **클라우드 API 사용량 충분한데도 오류**  
  **“쿨다운 이슈”** 가능성 언급, 게이트웨이 재시작으로도 동일하면 **스킬로 우회** 같은 회피 언급(근본 원인은 입력상 불명).

- **Discord 멀티봇 스레드 가시성 체크리스트 템플릿화**: 권한 3종 + `message_content` Intent + 토큰 재발급 플로우.

1. 인증

환경변수로 API key를 둬.

export MANUS_API_KEY="..."


요청 헤더는 우리 테스트 스크립트 기준:

API_KEY: $MANUS_API_KEY
Authorization: Bearer placeholder
Content-Type: application/json


Authorization은 placeholder로 넣어도 동작했어. 핵심은 API_KEY.

---

2. 작업 생성

엔드포인트:

POST <https://api.manus.im/v1/responses>


예시:

curl -s <https://api.manus.im/v1/responses> \
  -H "Content-Type: application/json" \
  -H "API_KEY: $MANUS_API_KEY" \
  -H "Authorization: Bearer placeholder" \
  -d '{
    "input": [
      {
        "role": "user",
        "content": [
          {
            "type": "input_text",
            "text": "한국어로 한 문장만 답해줘: Manus API 테스트 성공"
          }
        ]
      }
    ],
    "task_mode": "agent",
    "agent_profile": "manus-1.6"
  }'


응답에서 보통 이런 걸 봐:

{
  "id": "task_id...",
  "status": "running",
  "metadata": {
    "task_url": "<https://manus.im/app/...>"
  }
}


---

3. 결과 조회 / polling

생성된 id로 조회:

curl -s <https://api.manus.im/v1/responses/TASK_ID> \
  -H "API_KEY: $MANUS_API_KEY" \
  -H "Authorization: Bearer placeholder"


status가 running이면 몇 초 기다렸다가 다시 조회하고, completed면 결과를 읽으면 돼.

---

4. 결과 텍스트 위치

우리 테스트에서는 결과가 대략 여기 들어있었어:

output[].content[].text


Python식으로는:

texts = []
for item in resp.get("output") or []:
    for c in item.get("content") or []:

5. 최소 Python 예시

import os, json, time, urllib.request

BASE = "<https://api.manus.im>"
API_KEY = os.environ["MANUS_API_KEY"]

def request_json(method, url, payload=None):
    data = None if payload is None else json.dumps(payload, ensure_ascii=False).encode()
    req = urllib.request.Request(url, data=data, method=method)
    req.add_header("Content-Type", "application/json")
    req.add_header("API_KEY", API_KEY)
    req.add_header("Authorization", "Bearer placeholder")
    with urllib.request.urlopen(req, timeout=60) as r:
        return json.loads(r.read().decode())

payload = {
    "input": [{
        "role": "user",
        "content": [{
            "type": "input_text",
            "text": "MANUS_SMOKE_OK 라고만 답해줘."
        }]
    }],
    "task_mode": "agent",
    "agent_profile": "manus-1.6",
}

created = request_json("POST", f"{BASE}/v1/responses", payload)
task_id = created["id"]

while True:
    result = request_json("GET", f"{BASE}/v1/responses/{task_id}")
    if result.get("status") != "running":
        break
    time.sleep(5)

texts = []
for item in result.get("output") or []:
    for c in item.get("content") or []:
        if c.get("text"):
            texts.append(c["text"])

print(result.get("status"))
print("\n".join(texts))


---

핵심만 정리

- 작업 생성: POST /v1/responses
- 작업 조회: GET /v1/responses/{id}
- 인증: API_KEY 헤더
- 작업 모드: "task_mode": "agent"
- 프로필: "agent_profile": "manus-1.6"
- 결과: output[].content[].text
- 부가정보: metadata.task_url, metadata.credit_usage

실사용할 때는 API key를 코드에 박지 말고 env/secret 파일로만 관리하면 돼, 오빠.
