# AI 요약

## 인사이트·노하우

### GPT 5.6 Sol 사용 시 Effort 설정 주의
- Sol을 `높음(high)` + `빠름` 조합으로 쓰면 5시간 Pro 한도를 20분 만에 소진 가능
- `ultra` 설정 시 3분 내 한도 소진 사례 보고
- 실무 권장 조합: 분량 많은 작업 → `high / 보통`, 짧은 작업 → `high / 빠름`
- Plan 단계에서는 `high`로 설정하고 실행 단계에서 다운그레이드하는 방식도 유효
- 빠름 모드는 같은 시간 대비 토큰 소모가 더 많아 할증 개념으로 이해할 것

### GPT 5.6 Sol — 설정 문서 다이어트로 토큰 절감
- 기존 5.5에서는 경로 이탈·거짓말 방지용 설정 문서가 무거워질 수밖에 없었음
- 5.6에서는 해당 문제가 개선되어 같은 업무에서 **토큰 소모량이 체감상 크게 감소**
- 테스트 방법: 기존 설정 문서 항목을 하나씩 제거하며 동일 작업 결과 비교

### GPT 5.6 Sol API 호출 불가 (2026-07-10 기준)
- Sol은 현재 **Codex CLI 내부에서만 사용 가능**, API로 외부 호출 불가
- 단순히 모델 ID 숫자만 바꿔서 호출하는 방식은 동작하지 않음 (`sol`, `luna` 등 suffix 포함 필요)
- Codex 개발자 문서에서 5.6 관련 모델명 확인 필요

### Sol + Cerebras 조합 처리 속도
- GPT 5.6 Sol에 **Cerebras** 칩 적용 시 응답 속도가 체감상 크게 빠름 (영상 공유됨)
- 초당 750토큰 처리 버전 존재

### Hermes + OpenClaw에서 GPT 5.6 사용하는 법
- Hermes 최신 커밋까지 업데이트 필요 (git pull → origin/main 일치 확인)
- OpenClaw 최신 패치 후 `/model` 명령으로 모델 변경
- 요금제·계정에 따라 순차 롤아웃이라 동시에 모두에게 뜨지는 않음
- Codex CLI는 별도 업데이트 필요 — 업데이트 후 ChatGPT 앱 내 Work 탭에서 확인 가능

### 5.6 모델 출시 후 Claude Fable 토큰 리셋 현상

## 자동화·시스템 적용 후보

- **Plan / Do 단계 모델 분리 전략**: Plan 단계는 `Sol high` (추론 집중), 실행 단계는 `보통` 이하로 낮춰 토큰 절감. 같은 세션 내에서도 단계별로 effort를 명시적으로 분리하는 워크플로.
- **설정 문서 슬리밍 테스트**: 모델 업그레이드 후 기존 시스템 프롬프트/설정 문서를 항목 단위로 제거하며 동작 검증 → 불필요한 가이드라인 제거로 토큰 절감. 5.6에서 특히 효과적이라는 실사용 보고.
- **로컬 + 클라우드 하이브리드 운용**: 로컬 35B 모델(Hy3 등)로 초안/구조 생성 → 클라우드 모델로 품질 보완. 비용 민감한 반복 작업에 적합.
