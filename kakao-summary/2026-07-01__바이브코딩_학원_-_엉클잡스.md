# AI 요약

## 인사이트·노하우

- **Claude Sonnet 5 업데이트 후 미표시 문제**: Claude 앱에서 Sonnet 5가 자동으로 뜨지 않을 경우 수동 업데이트로 해결됨. "자동 업데이트인줄 알았는데 아니군요" — 앱 업데이트를 직접 실행해야 적용.

- **페이블 5 (Fable 5) 가성비 평가**: 자체 루프(loop)가 내장된 느낌으로 "계속해서 도는" 추론 방식이나, 가격이 타 모델 대비 2배 수준. "과한 추론이 필요 없다면 페이블은 의미 없을 것 같다"는 실사용 판단. LLM Wiki·루프·워크플로 하네스 검증 구조가 이미 갖춰진 환경에서는 추가 가치가 불명확 — 직접 A/B 테스트 권장.

- **AI 모델 비용 분류 원칙** (뉴스브리핑 인용):
  - 요약·분류·초안 → 저렴한 모델
  - 계약서·개인정보·자동발송·결제 관련 → 고성능 모델 + 사람 확인
  - "가장 비싼 모델만 써야 한다"는 흐름이 약해지는 중

## 결정·약속·일정

- **페이블 5 (Fable 5) 재개**: 7월 7일까지 주간 사용량 한도의 최대 50%까지 사용 가능. 이후 사용량 기반 과금제 전환.
- **페이블 5 실제 사용 가능 시점**: 채팅 기준 "내일"(2026-07-02)부터로 안내됨.

## 반응이 컸던 화제

- **Claude Sonnet 5 출시**: 새벽 1시부터 출시 전망이 공유되고, 오전까지 여러 참여자가 확인·환호. "소넷 가격으로 오푸스 수준 성능"이라는 가성비 포인트가 공감을 얻었으며, "헤어진 전 여친이 돌아왔다"는 표현이 나올 만큼 이전 Sonnet 성능에 대한 향수와 기대가 맞물림.

## 자동화·시스템 적용 후보

- **모델 라우팅 전략**: OpenRouter 기준 오픈소스 AI 토큰 비중이 1월 34% → 6월 65%로 증가. 업무 태스크를 "가벼운 모델 vs 강한 모델 + 사람 확인"으로 라우팅하는 2-tier 패턴을 자동화 파이프라인에 도입할 수 있음.

- **STT → 상세 기획 활용**: "STT → 상세 기획에 사용, 프로젝트 근간 잡는 데 큰 도움" — 페이블 5 접근이 제한된 기간 동안 음성 입력을 기획 초안 생성에 투입하는 워크플로 대안으로 제안됨.

## 질문·미해결

- **페이블 5 신원 확인 후 사용 가능 여부**: 이전에 언급된 "신원 확인 이후 사용" 정책이 이번 재개와 무관한지 불명. 공식 공지 없음 — 후속 확인 필요.
- **Sonnet 4.6 대비 성능 체감 저하 원인**: "왜 예전만큼 성능이 안 나오는 느낌이 드는 이유가 뭘까요" — 답변 없이 넘어감.
