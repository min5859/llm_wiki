# AI 요약

## 인사이트·노하우

### Thinking 모드 사용 전략
- **원칙**: 항상 켜두지 말고 필요할 때만 활성화. 상시 켜두면 토큰 소모 증가 + 응답 속도 저하.

### 봇 맥락 소실(Context Loss) 대응법
- **원인 후보**: 매 턴 로드하는 `.md` 파일이 너무 많거나, `에이전트.md` / `소울.md` 외 파일이 불필요하게 쌓인 경우.
- **해결 접근**:
  1. 매 턴 로드 파일은 **필수 내용만 압축**해서 최소화.
  2. 나머지 맥락은 워크플로우에 저장 → 작업 시점에 스킬로 불러오기.
  3. 파일 내 **상충 규칙이 몰래 삽입**됐을 수 있으니 봇에게 "현재 기억 보여줘, 상충되는 게 있는지 점검해줘" 요청.
  4. 중요 프로젝트는 **Obsidian 연결** 또는 `메모리.md`로 별도 관리.

### 헤르메스 기억 오염(Memory Corruption) 진단·정리
- **증상**: 같은 말 반복, 완료된 작업 재시도, 중국어 LLM 사용 후 장기기억에 중국어 도배.
- **진단 절차**:
  1. 헤르메스에게 "네가 기억하고 있는 내용을 보여줘" 요청.
  2. "어떻게 정리하면 좋을지 제안해봐"로 스스로 평가하게 유도.
  3. 불필요·잘못된 기억 정리 후 재사용.
- **예방**: LLM 폴백 시 저품질 모델(예: 미니맥스) 사용하면 오히려 기억 오염 심화 → 폴백 모델 선택 주의.
- **오염 정의**: "기억할 필요 없거나 잘못된 정보를 장기기억에 저장하는 것. 이미 종료된 작업을 반복 시도하거나, 완료된 작업에 문제가 있다고 우기는 행동."

### 헤르메스 vs 오픈클로 비교 (사용자 주관 경험)
| 항목 | 헤르메스 | 오픈클로 |
|------|---------|---------|
| 학습 방식 | 대화 중 스스로 업무규칙 생성 | 사용자가 정리해준 매뉴얼 따름 |
| 스킬 | 커스텀 그때그때 생성 | 이미 만들어진 스킬 풍부 |
| 안정성 | 상대적으로 높음 (운영용) | 자유도 높음 (실험용) |
| 기억 오염 | 관리 필요, 주기적 다이어트 필수 | 아무거나 주워먹지 않아 상대적으로 안정 |
| 추천 대상 | AI 활용 경험자 + 꾸준한 관리 가능자 | 일반인도 비교적 무난 |

- **실전 조합**: "오픈클로로 만든 기술·산출물을 헤르메스가 운영"하는 방식.
- **헤르메스 주기적 다이어트 필수**: 자가발전 특성상 불필요한 기억이 쌓여 속도 저하 + 토큰 소모 증가.

### 컨텍스트 관리 채널별 팁
- **디스코드**: 프로필 + 스레드 조합 → 컨텍스트 분리 가장 깔끔.
- **슬랙/디스코드 > 텔레그램**: 스레드 기반 컨텍스트 분리 편의성에서 압도적 우위.
- **프로필 다수화** 시 개별 설정 증가 부담 → 프로필 관리용 봇을 별도로 두는 방법 존재.

### 헤르메스 텔레그램 봇 번호 선택(Clarify) 제거
- **문제**: 텔레그램 헤르메스 봇이 작업 중 번호 선택 프롬프트를 띄워 흐름 중단.
- **해결**: 봇에게 "Clarify 꺼줘" 요청. (켜두면 중간에 멈추는 경우 많음)

### RTK + Caveman 토큰 절약 조합
- `RTK` 미사용자는 설치 권장 (토큰 절약 효과).
- `Caveman`은 의미 손실이 다소 있으므로 **튜닝해서 사용**하는 것이 효과적.

## 자동화·시스템 적용 후보

- **봇 기억 자가 진단 워크플로**: 봇이 이상하다 싶을 때 → "현재 기억 목록 보여줘" → "상충·불필요 항목 분석해줘" → 정리 지시. 주기적 루틴으로 자동화 가능.
