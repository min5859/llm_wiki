## 인사이트·노하우

- **OpenClaw 업데이트 이슈(5.18)**: 업데이트 후 `exec_command` 관련 문제 발생 → 문제 해결을 위해 소스 코드까지 수정해야 겨우 동작하는 수준이었다 → **5.12로 롤백**한 사례가 있음(원인: 버전 업; 시도·해결: 소스 수정 검토 후 이전 버전으로 복귀).
- **OpenClaw 게이트웨이 지연·진단 로그**: `liveness warning: reasons=event_loop_delay`(예: `eventLoopDelayP99Ms`, `interval=30s` 등) 로그 반복 → 공식 문서를 넘겨 Claude Code·코덱스에 물어보며 여러 수정을 시도했으나 며칠간 미해결. 슬랙 응답도 느리고 **헤르메스와 비교해 간단한 답변도 1초 이상** 늦는 체감이 있음.
- **헤르메스 vs 오픈클로 역할 분담(현장감)**: 같은 환경에서 둘 다 쓸 때 한쪽 장애 시 다른 쪽이 “살리는” 식으로 쓰이기도 하나, 문제 생길 때 **“헤르메스보고 살리라 한다”**, **“헤르메스가 레벨이 다르다”**는 평도 있음(코덱스 vs Claude 차이 불명이라는 의견 포함).
- **카카오 톱(헤르메스) 운영·인수인계**: 비개발 인력만 남을 때를 대비하면, **게이트웨이가 꺼질 때가 있어 직접 켜야 했던 경험**이 있으며 **필요한 명령어 정도 인수인계**하면 된다는 의견이 있었음.
- **Grok 사용 맥락**: 채팅 후기로 **후기만 봐도 “뭔가 이상하다”**, **오픈클로나 허미스(헤르미스)·불명에 물려서 에이전트로 쓰기엔 “아직 못 쓸 물건”**이라는 평과, **그록 빌드를 쓰는 사람들이 공통적으로 “작업을 1분 단위로 한다”**고 하는 경우가 많다는 얘기가 나옴. 반면 **OAuth로 영상 생성**, **xAI API로 검색** 등은 활용 가능하다는 구분(**“영상 만들고 검색하고 이미지 만드는 용도” vs “그록 자체를 물려 에이전트 굴리기는 시기상조”**)이 언급됨.
- **Grok 영상 활용 후기**: 한 참가자가 **리밋 걸릴 때까지 반나절 동안 영상 여러 편**(대화에서는 22편 언급) 생성, **품지이 시뎅스 불명 라이벌에게 딱히 밀리지 않았고**, 특정 캐릭터·스타일과는 **찰떡일 수 있다**는 경험이 공유됨; **속도는 매우 빠르지만 코딩·에이전트 쪽 성능 향상은 기대 단계**.
- **Claude 채팅 이미지 첨부 한도**: 한 세션에서 **그림 첨부 한도가 차면 새 대화 필요**하고, 이때 **이전 대화 맥락을 전부 가져가지 못한다**는 제약 언급(서비스 작업 일관성 이슈).
- **컨텍스트 한도 관리**: **컨텍스트 압축** 가능 여지가 제기됨; **서브 에이전트를 쓰면 컨텍스트 저장에 유리하다**는 팁 형태의 의견.
- **퇴사 시 에이전트·봇 남겨두기**: 비전공자만 남을 경우 **관리 책임·책임 소재**(퇴사자에게 넘길 가능성 포함) 우려 제기; **설치·실습·셀프 관리까지 가능하게 하거나**, 안 되면 **깔끔히 포기**하는 편이 낫다는 조언 대비 **GPT Pro 등으로 교체 후 “고쳐달라고 하면 된다”**는 현실적인 대안도 논의됨.
- **멀티 봇·원격 접근 조합**: 참고 안으로 **봇 여러 대 + 다른 PC + Tailscale**(불명: 정확한 구성 미기재) 구성 언급.

## 자동화·시스템 적용 후보

- **컨텍스트 전략 후보 문구 보존**: **“컨텍스트 압축”**, **“서브 에이전트로 컨텍스트 세이브”** 류 워크플로 검토 후보로 기록 가능.
- 인수인계용 **게이트웨이 재기동·필요 CLI·문서 1쪽** 같은 **장애 대응 런북**(대화 속 “명령어 정도 인수인계”와 접합 가능).

## 질문·미해결

- **헤르메스 카카오 톱봇 “어떻게 설정·사용하나요?”** — 답변은 **방법 미숙 지로 사용 못 하는 경우** 등으로 남거나 구체 설정 가이드는 로그상 없음.
- **클로드 Max(예: 월 중 사용 한도 많이 남아 보이는데) 충전 금액이 빠지는 원인**(VS Code로 Claude Code 다수 사용 포함 가설) — **“봇 때문일 거” 수준 의견**만 있고 **정확한 요금·카운팅 근거 불명**.
- **Open크론 불명 크론·스케줄 답변으로 회신되어 질문 맥락을 놓치는 현상**(OpenCl로 추정 등) 해결 방법 — **별도 방법 제시 없이** 같은 불편 공감 수준으로 끝남.
- **맥북 + 윈도우 PC 사용자가 회사에서는 봇을 못 켜는 경우 VPS·라즈베리파이 등 무엇이 좋을지** — **대화 종료 지점까지 구체적인 비교 결론·추천 없음**(질문만 제기됨).
- **`event_loop_delay` 계열 진단이 나는 OpenCl 게이트웨이 지연 문제**의 **근본 원인 및 확정 해결책** — 로그에는 **미해결** 상태로 명시됨.
