# AI 요약

## 인사이트·노하우

### OpenClaw(오픈클로) 운용 PC 선택 기준
- **로컬 LLM 미사용 시**: 컴퓨터 스펙이 크게 중요하지 않음. 외부 API(GPT 등)에 붙여 쓰면 가벼운 작업 처리 수준으로 충분.
- **RAM**: 맥이 아닌 Windows/Linux 미니PC는 RAM 8GB면 버거움. 맥 생태계(M1~M4)는 8GB도 동작 가능하나 애매한 편.
- **전기세 기준**: 미니PC(알리 등) 24시간 가동 시 월 800원 수준. 데스크탑은 여름 누진세 3단계 진입 위험.
- **가성비 순위** (참여자 다수 의견 종합):
  1. 집에 남는 구형 노트북(화면 하자 있어도 서버로 충분) — 당근/번개장터에서 저렴하게 확보
  2. 맥미니 M4 (RAM 24GB) — 안정성·호환성 최고, 입문자 삽질 적음
  3. 알리 미니PC (라이젠5 7430U, RAM 16GB) — 30만 원 후반, 가성비 고려 시 옵션
  4. VPS(텐센트 클라우드 등) — 초기비용 최소이나 입문자에게 추가 진입장벽
- **맥 vs 윈도우**: 오픈클로 계열 작업은 맥이 더 원활. 윈도우는 삽질 더 많음.
- **운용 팁**: 구형 노트북은 절전모드 해제 + 외장 GPU 비활성화 설정 후 원격 접속으로 운용. Ubuntu 서버/헤드리스 설정 시 화면 없어도 무관.

### Codex로 수집한 로컬 HTML을 NotebookLM에 올리는 방법
- **문제**: Codex가 수집한 자료가 `file:///C:/...` 형태의 로컬 HTML이라 NotebookLM이 URL 인식 불가.
- **해결 방법 (원인→시도→해결)**:
  - 원인: NotebookLM은 `http://` 또는 `https://`로 시작하는 URL만 허용. 로컬 경로 및 HTML 파일 포맷 미지원.
  - 시도: HTML 파일 직접 업로드 → 지원 포맷 아님으로 실패.
  - 해결책 3가지:
    1. Codex에게 원문 링크를 다시 찾아오게 해서 실제 URL을 NotebookLM에 입력.
    2. Codex에게 `html2pdf` 변환 요청 후 PDF 업로드.
    3. 아래 프롬프트로 Codex에게 URL 목록 생성 요청:
       > "NotebookLM의 '웹사이트 및 YouTube URL' 입력칸에 그대로 붙여넣을 수 있도록 URL 목록을 만들어 주세요. URL은 반드시 http:// 또는 https:// 로 시작해야 합니다."

### 오픈휴먼(OpenHuman) 사용 평가
- **현재 상태**: GitHub Star는 빠르게 증가 중이나 실사용 시 문제 다수.
  - OAuth가 오픈휴먼 자체 구독 계정 외에는 지원 안 됨.
  - Ollama 연결 지원한다고 하나 버그로 실제 연결 불가.
  - 토큰 절약 기술 적용 시 의미 손실이 큼.
- **결론**: "큰 차이가 없다면 사람들이 많이 쓰는 걸 쓰세요" — 현 시점 도입 불필요.

### 클로드 모델 자동 변경 이슈
- 작업 중 클로드가 갑자기 성능이 떨어진 경우, 모델이 **Sonnet으로 일괄 변경**되어 있는지 확인 필요.

### Codex 관련
- 코딩 품질은 높으나 최근 **의도적으로 속도를 낮춘 것으로 추정**됨.
- 사람이 이해하기 쉬운 설명은 여전히 클로드가 우위.
- Codex 미들 대신 **X-High** 사용으로 개선 경험 사례 있음 (구체적 맥락 불명).
