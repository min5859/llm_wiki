# AI 요약

## 인사이트·노하우

- **실시간 전사(STT)의 한계와 보완 노하우**
  - 녹음 후 전사·요약은 NotebookLM 등으로 잘 되지만, "실시간" 전사는 품질 확보가 어렵다는 경험담(P55). 넷플릭스 BTS 공연 실시간 자막 품질이 최악이었던 사례 인용.
  - 젠스파크 "speakly"가 실시간 전사를 잘 잡아준다는 평가.
  - 크롬 실시간 자막 설정 경로: `chrome://settings/accessibility` (크롬에서 나오는 소리를 자막으로 표시).
  - 전사 도구 개선 아이디어: ① 화자분리 필수, ② 전사 완료 후 화자 이름 일괄 수정 가능, ③ 전사 문장 클릭 시 해당 부분 다시 듣기로 자동 연결.

- **Claude Code 구독으로 Hermes 사용하기 (deck 정리, P61 재공유)** — 비공식·약관 회색지대 경고 포함
  - 구조: Claude Code `/login`/`claude setup-token` → access/refresh 토큰이 macOS Keychain 또는 `~/.claude/.credentials.json`에 저장 → Hermes에서 `provider: anthropic`, `model: claude-opus-4-7` 설정 → `ANTHROPIC_API_KEY` 대신 Claude Code OAuth 토큰을 읽어 사용.
  - 주의: 환경변수에 `ANTHROPIC_API_KEY`가 있으면 OAuth보다 우선될 수 있음 → 구독 경로를 쓰려면 API key 설정과 혼동 금지.
  - patch가 필요한 이유 두 가지:
    1. Claude Code 요청 형태(시스템 프롬프트, 툴 네이밍, beta flag, SDK 헤더)에 맞추는 호환 레이어가 없으면 거부/400 발생, third-party usage 라우팅 위험.
    2. credential 선택 문제 — Keychain에 만료·갱신불가 토큰이 있으면 그걸 버리고 `~/.claude/.credentials.json`의 유효 refreshable 토큰으로 fallback해야 함.
  - 운영 주의: Claude와 Claude Code가 같은 구독 한도를 공유하므로 무제한 아님. `claude-opus-4-7:1m` 같은 큰 컨텍스트나 병렬 subagent는 한도 빠르게 소모. 429는 설정 오류가 아닌 실제 한도일 수 있음.
  - patch 위치: `/Users/deck/.hermes/patches` — 일반 update 시 유지되지만 venv `sitecustomize.py`가 사라지면 patch loader 재설치 필요.

- **헤르메스 칸반 기능 사용법(P39 경험)**: 공식 문서까지 안 봐도 됨. 그냥 "A야 이런이런 거 시킬 건데 이걸로 이렇게 계획 짜봐" → "ㅇㅋ 실행" 식으로 자연어로 지시하면 동작. 단, 결과는 채팅창에 나열되고 대시보드에는 안 뜸. 칸반을 켜두면 알아서 모니터링하는 효과(토큰 소모는 늘어남).

- **카톡 5.6 업데이트 후 슬랙 연동 깨짐 사례(P13)**: 새벽 내내 헤매다 개발자 도움으로 해결. 비개발자가 오픈소스 자주 사용하다 보면 릴리즈 확인 필수라는 P61의 코멘트.

- **윈도우 카톡 DB 추출**: 가능하나 매우 어렵고, 한번 뚫어도 "계속 뚫을 수 있는 열쇠 같은 요소"를 안 만들어두면 금방 막힘(P17). GPT가 못 하겠다고 거절하는 영역.

- **헤르메스를 어디서 돌릴까(P10 추천)**: ~20만원대 N100 미니PC + 16GB RAM / 512GB SSD에 우분투 서버 설치 → 한 달 전기세 900~1000원. 24/7 저전력 운영용으로 적합. 윈도우라면 WSL2+우분투(P32: "네이티브는 고통을 즐기는 사람용").

- **헤르메스 텔레그램 연동 트러블슈팅(P14)**: Codex auth 인증 자체는 hermes console에서 잘 되는데, 텔레그램 연동만 갑자기 안 됨. P3는 "텔레그램 + codex oauth + hermes" 조합 잘 됨. 디스코드를 메인으로 옮긴 사례도 있음(P61).

- **오픈클로 새버전**: 새로 만든 에이전트가 옛 에이전트보다 똑똑한 느낌(P2). 사용자가 많고 대기업급 운영이라 유지보수가 압도적이라는 평(P10). 주기적으로 밀고 재설치하면 더 빠르고 깔끔해진다는 경험(P22).

- **Claude Max 한도 갑작스런 소진(P119)**: 평소보다 훨씬 빨리 Opus 4.7 한도 소진. 명확한 해결책 미제시 — 해당 시점의 한도/속도 제한 가능성.

- **OAuth fallback 패치 패턴(deck)**: 만료 토큰을 Keychain에서 감지해 파일 기반 refreshable 토큰으로 자동 fallback — 다른 OAuth 통합에도 응용 가능한 일반 패턴.
- **칸반 3박스 멘탈모델(홀든 설명)**: "할 일 / 하는 중 / 다 했다" 3박스. 가운데가 차면 신규 작업 중단하고 비우기 우선. 에이전트 작업 분배 룰로 그대로 적용 가능.

클로드 구독으로 Hermes 사용하기
(약관 위반 위험 책임 못짐)

핵심 요약
Claude Code 구독을 쓴다는 건 Anthropic API key를 쓰는 게 아니라, Claude Code가 로그인할 때 받은 Claude 계정 OAuth 자격증명으로 Claude Code 경로에 요청을 보내는 것이다. 그래서 비용/한도도 일반 API 과금이 아니라 Claude Pro/Max/Team/Enterprise 구독 한도 쪽을 따른다. 단, 이건 공식적으로는 Claude Code CLI에서 쓰는 경로이고, Hermes 같은 외부 런타임에서 쓰는 건 호환 레이어가 필요한 비공식 통합이다.

  구조

  1. 사용자가 Claude Code에서 /login 또는 claude setup-token으로 로그인한다.
  2. Claude Code는 access token / refresh token을 macOS Keychain이나 ~/.claude/.credentials.json에 저장한다.
  3. Hermes ops 프로필은 provider: anthropic, model: claude-opus-4-7로 설정한다.
  4. Hermes가 Anthropic provider를 호출할 때 일반 ANTHROPIC_API_KEY 대신 Claude Code OAuth token을 읽어 쓴다.
  5. 요청이 Claude Code OAuth 요청으로 인식되면 Claude Code 구독 한도에서 사용된다.
  6. 실패하면 fallback으로 openai-codex / gpt-5.5를 쓰게 해둔다.

중요한 차이
ANTHROPIC_API_KEY를 쓰면 Claude Code 구독이 아니라 Anthropic API 과금 경로다. 특히 환경변수에  ANTHROPIC_API_KEY가 잡혀 있으면 Claude Code 구독 로그인보다 API key가 우선될 수 있다. 그래서 “구독을 쓰고 싶다”면 API key 기반 설정과 OAuth 기반 설정을 혼동하면 안 된다.

 왜 패치가 필요한가
Claude Code OAuth token만 있다고 끝이 아니다. Anthropic 쪽은 요청이 실제 Claude Code류 요청인지도 본다. 그래서 Hermes가 그냥 일반 Anthropic API 요청처럼 보내면 다음 문제가 생길 수 있다.

  - 인증은 있는데 Claude Code 요청 형태가 아니라서 거부됨
  - third-party usage 또는 extra usage 쪽으로 라우팅됨
  - tool name, system prompt, beta flag, SDK header 구조가 달라서 400이 남
  - Keychain에 만료된 토큰이 남아 있으면 유효한 파일 토큰까지 못 내려감

  우리 환경의 patch는 두 가지 역할을 한다.

  첫째, Claude Code 요청 형태에 맞추는 호환 레이어다.
  Hermes 요청의 system prompt, tool naming, beta flag, SDK 계열 header 구조를 Claude Code OAuth 요청이 기대하는 형태에 가깝게 바꾼다.

  둘째, credential 선택 문제를 고친다.
  macOS Keychain에 만료됐고 refresh 불가능한 Claude token이 있으면 그걸 버리고  ~/.claude/.credentials.json의 유효한 refreshable token으로 fallback한다.

  현재 ops 구성

  model:
    provider: anthropic
    default: claude-opus-4-7

  fallback_providers:
    - provider: openai-codex
