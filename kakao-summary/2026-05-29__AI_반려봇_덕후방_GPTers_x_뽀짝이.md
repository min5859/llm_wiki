# AI 요약

## 인사이트·노하우

- **Claude Opus 4.8 출시 → 토큰 리셋**: 업데이트로 인해 기존 세션이 초기화된 사례 다수. 대규모 패치 직후 세션 유지가 필요한 작업은 중단 위험 있음.
- **OpenRouter 무료 모델 요청 한도 기준** (크레딧 구매 이력 기반):
  - $10 미만 결제 시: 일일 최대 50회
  - $10 이상 결제 시: 일일 최대 1,000회
  - 공통: 분당 최대 20회, **실패 요청도 할당량에 포함됨**
  - 결론: "무료 모델 쓸 때 크레딧 $10 이상 유지 필수"
- **Gemini API 구독(OAuth) 방식은 OpenClaw 연동 차단됨**: 정액제 플랜으로는 OpenClaw에서 Gemini 사용 불가. API 키 방식만 가능.
- **Google Vertex API 과금 주의**: 사용 경로를 인지하지 못한 상태에서 월 10만 원 한도 금방 소진됨. 명시적 리밋 설정 필수.
- **AI가 세션 중 "할 수 없다"고 태도 전환하는 경우 대응**:
  - 원인: 모델 자체 정책 변경 또는 일시적 기능 제한 (Gemini 사례)
  - 해결: 모델 교체 또는 세션 재시작. 같은 세션 내 반복 교정은 "점점 더 미궁으로 빠진다"(P26 직접 언급)
- **작업 중단 리스크 대비 — 프롬프트 로그 저장 워크플로** (P9 제안):
  - 모든 프롬프트를 별도 로그 파일로 저장
  - git 커밋 시 포함
  - 효과: 모델 교체 시에도 작업 연속성 유지 가능
- **Suno 음악 생성 노하우** (P110·P111 토론):
  - 일본어로 먼저 생성 → 청취 후 한국어로 개사하면 한국적 감성 반영 가능
  - LLM이 취약한 영역: 인간적 고뇌, 정치·철학적 서사, 입체적 캐릭터, 멜로디에 맞는 라임 구조
  - 현실적 베스트 프랙티스: "멜로디·가사 초안은 사람이 직접, 구현만 AI로"
  - 영어 가사는 의미 오류·불필요 접미사가 숨겨져도 비영어권 청자에게는 불편이 덜해 품질 문제가 가려짐

## 추천·자료

- **Hermes (에르메스)**: AI 에이전트 입문자에게 OpenClaw보다 추천. "쉽기도 쉽고 잘 하네요"(P107). Codex와 연결해 사용하면 시너지.
- **DeepSeek**: 코드 작성·수정 분야에서 높은 평가. 할인 영구화 선언 후 비용 효율도 좋음. 중국어 혼입 없이 코딩 가능. GPT 파업(장애) 시 폴백 모델로 활용 중인 사례 있음.
- **Qwen 3.6**: "그냥저냥 쓸만함". 코딩 시 중국어 미혼입. 카카오 자동화 테스트에 활용된 사례 있음 (단, 에이전트 용도로는 평가 낮음).
- **Supabase + 다중 디바이스 통합**: macMini·라즈베리파이·Hermes·Codex 등을 단일 Supabase 저장소로 통합 운용 시도 사례 (P44). 초보 관점에서 실용적 아키텍처로 언급됨.
- **MiMo 2.5 Pro (샤오미)**: Credits 기반 과금 모델. 한 참여자가 직접 테스트 후 "추천 취소" — 디자인 수정 반복 요청 시 수정 불가 문제 발생.

## 반응이 컸던 화제

- **OpenRouter 무료 모델 한도 기준 공유** (P106): 크레딧 $10 기준으로 일일 요청 한도가 50회↔1,000회로 크게 달라진다는 정보가 "오!!!" 반응을 이끌어냄. 실용적 과금 팁이라 즉각 관심.
- **Gemini가 갑자기 "못 한다"고 거부한 사례** (P120): 1,400개 상품 엑셀 자동화를 5개월간 운용하다 막판 2% 남기고 태도 전환. 다수가 공감하며 대안 모델(딥시크, GPT, 큐웬)을 적극 추천. GPT 파업 경험담까지 이어지며 긴 토론으로 발전.
- **카카오톡 봇 "홀든" 라이브 테스트**: Qwen 35b 기반 카카오 계정 봇이 채팅방에서 실시간 응답. 반말/존댓말 혼용, 즉각적 응답 속도, 엉뚱한 답변 등으로 다수가 장난을 걸며 분위기 활성화. 구현 아키텍처 질문(OpenClaw-카카오 연동 방식)도 파생됨.
- **AI 음악 생성의 한계와 사람의 역할** (P110·P111): LLM이 철학적·서사적 가사와 멜로디 라임 매칭을 잘 못 한다는 경험 공유. 현실적 결론("구현만 AI, 창작은 사람")에 공감 형성.

## 자동화·시스템 적용 후보

- **프롬프트 로그 자동 저장 워크플로** (P9 제안): 작업 중 생성한 모든 프롬프트를 로그 파일로 저장 → git 커밋에 포함. 모델 교체 시 작업 연속성 확보. 특히 장기 반복 작업(대량 상품등록 등)에서 유용.
- **GPT 장애 대비 중국 모델 폴백 구성** (P20 실사용): OpenClaw + Codex(메인) + Claude(서브) + DeepSeek/Qwen(대량 작업·폴백) 조합. "지피티 파업 때문에 중국모델 폴백 해놨다"고 명시.
- **카카오 봇 연동 주의사항**: "Loco"라는 툴이 존재하나 계정 정지 위험으로 비추천 (P122 직접 언급). [openkakao.vercel.app](https://openkakao.vercel.app/) 가 대안으로 언급됨.
- **Supabase 단일 저장소로 다중 디바이스/에이전트 통합**: macMini + 라즈베리파이 + Hermes/Codex를 단일 Supabase 인스턴스로 묶는 아키텍처 패턴.
