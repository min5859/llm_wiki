# AI 요약

## 인사이트·노하우

- **OpenClaw에서 codex 로그인이 풀렸을 때**: 다시 설정하기 귀찮을 경우 "웹으로 열어달라"고 하면 잘 됨 (P3).
- **텔레그램 그룹에서 봇 응답 안 되는 문제**: 일반 멘션·privacy 끄기로도 안 될 때, 토픽을 만들고 토픽 내에서 멘션해야 함. 오픈채팅처럼 다른 환경에서는 `chat room id + user id` 숫자값 조합으로 고유 ID 처리하면 해결됨 ("오픈채팅이라서 고유값이 다른 것 같다", P10·P56).
- **OpenClaw + 슬랙 멀티 에이전트 답변 이상 진단법**: `openclaw tui`로 터미널에서 답변이 오는지 먼저 확인 → 답변이 오면 OpenClaw 문제가 아니라 슬랙 연결 문제 (P19).
- **카카오 사용량 2배 이벤트의 실체**: 총량 2배가 아니라 5시간 한도가 2배가 되는 구조이며, 주간 한도를 끌어쓰는 개념. 결과적으로 주간 한도에 더 빨리 도달할 수도 있음 (P17, P55).
- **Claude Code 구독으로 Hermes 쓰기 (P4 정리, 비공식·약관 리스크 있음)**:
  - 핵심: Claude Code는 API key가 아니라 로그인 시 받은 **Claude 계정 OAuth 자격증명**으로 동작. 비용도 Pro/Max 구독 한도를 따름.
  - `ANTHROPIC_API_KEY`가 환경변수에 잡혀 있으면 OAuth보다 우선될 수 있어서, "구독을 쓰고 싶으면" 둘을 혼동하면 안 됨.
  - Hermes에서 그냥 일반 Anthropic API처럼 보내면 (1) Claude Code 요청 형태가 아니라 거부, (2) third-party/extra usage로 라우팅, (3) tool name·system prompt·beta flag·SDK header 구조 차이로 400, (4) Keychain의 만료 토큰이 우선되어 유효 파일 토큰을 못 씀.
  - 따라서 patch 두 가지: ① system prompt·tool naming·beta flag·SDK header를 Claude Code OAuth가 기대하는 형태로 맞추는 호환 레이어, ② Keychain 만료·refresh 불가 토큰을 버리고 `~/.claude/.credentials.json`의 유효 refreshable token으로 fallback.
  - 운영 주의: Claude와 Claude Code가 같은 구독 한도 공유, `claude-opus-4-7:1m`·병렬 subagent는 한도 빠르게 소모, patch는 `/Users/deck/.hermes/patches`에 두어 update에도 유지하되, venv 재설치 시 `sitecustomize.py`가 사라지면 patch loader 재설치 필요.
- **카톡 자동화 + 오픈클로 봇 구성 (P10 사례)**:
  - 업무폰 부계정으로 로그인시켜 봇으로 사용 → 봇으로 연결하니 고유 ID가 안정적이라 채팅방 ID와 채팅방별 본인 고유 ID를 안정적으로 잡을 수 있었음.
  - OpenClaw 설정에 채팅방 리스트, 호출 가능 유저 리스트, 관리자 유저 리스트를 `.py` 파일로 저장.
  - 오픈클로 크론 잡으로 채팅방 요약 자동 수행.
  - Linux 환경에서 Kakao-CLI 못 쓸 때 이 방식이 유효했으나 **밴 우려 있음** (P56 동의).
- **OpenClaw가 클로드 플랜만으로는 동작하지 않을 수 있음**: 모델 인증은 오푸스 4.7로 잡혀도 프로바이더 설정이 필요 (P79 사례, 첫 답장이 안 오는 증상).
- **GPT-5.5 Pro 확장모드 + 봇에 전달하는 파이프라인**: 확장모드가 시간이 오래 걸리지만 결과 품질이 더 좋고, GPT-5.5부터는 Pro 모델도 빨라진 편 (P42·P56).
- **오픈채팅·줌 입장 트러블슈팅**: Zoom Workspace 앱에 다른 확장프로그램이 깔려 있으면 앱으로 입장이 안 되는 경우가 있고, 브라우저 접속으로 우회 가능 (P32·P44).

- **클로바노트** — 한국어 음성 화자 분리·이름 변경 가능, PC 웹에서 실시간 녹음, 시스템 소리 녹음도 가능. 영어보다 한국어 환경에 유리 (P104).

- 화자 분리 관련 최근 업데이트 모델 후보 (P55): **WhisperLiveKit, WhisperX (2026.02), ElevenLabs Scribe v2 Realtime, Voxtral Transcribe 2**.

- **Hermes 온보딩 화면의 스킬 목록**: 기본으로 검증된 좋은 스킬들이 이미 준비되어 있으니 한 번 훑고 시작하면 좋음. 칸반도 여기 포함 (P22).
- **칸반 개념**: AI와 무관한 프로젝트 관리 방법론. "대기중 / 진행중 / 완료 확인 / 최종 완료"로 보드를 두고 카드를 옮기는 방식. Hermes에서는 LLM과 별개 기능 (P44·P39).

## 자동화·시스템 적용 후보

- **OAuth 토큰 기반 Hermes-Claude 구독 연동 패치 (P4)** — 비공식·약관 리스크 있지만 호환 레이어와 credential fallback이라는 두 단계 구조가 명확하므로, 유사한 SDK 호환 작업 템플릿으로 참고 가치.
- **카톡 봇 + 오픈클로 크론 요약 스택 (P10)** — `chat room id + user id` 숫자값을 키로 쓰는 패턴, `.py`에 채팅방·호출자·관리자 리스트를 박아 두는 구조, 부계정+업무폰 분리 운영. 카톡류 자동화의 레퍼런스로 보존할 가치.
- **오픈클로 텔레그램 그룹 토픽 + allowlist + group privacy 끄기 체크리스트** — 텔레그램에서 그룹 응답이 안 될 때 항상 짚어야 할 항목 (P56).
- **상용 AI 앞단에 P95의 Privacy Filter 붙이기** — 사내·민감정보 환경에서 즉시 시도 가능한 패턴 (P19도 동의).

클로드 구독으로 Hermes 사용하기
(약관 위반 위험 책임 못짐)

핵심 요약
Claude Code 구독을 쓴다는 건 Anthropic API key를 쓰는 게 아니라, Claude Code가 로그인할 때 받은 Claude 계정 OAuth 자격증명으로 Claude Code 경로에 요청을 보내는 것이다. 그래서 비용/한도도 일반 API 과금이 아니라 Claude Pro/Max/Team/Enterprise 구독 한도 쪽을 따른다. 단, 이건 공식적으로는 Claude Code CLI에서 쓰는 경로이고, Hermes 같은 외부 런타임에서 쓰는 건 호환 레이어가 필요한 비공식 통합이다.

  구조

  1. 사용자가 Claude Code에서 /login 또는 claude setup-token으로 로그인한다.
  2. Claude Code는 access token / refresh token을 macOS Keychain이나 ~/.claude/.credentials.json에 저장한다.
  3. Hermes ops 프로필은 provider: anthropic, model: claude-opus-4-7로 설정한다.
  4. Hermes가 Anthropic provider를 호출할 때 일반 ANTHROPIC_API_KEY 대신 Claude Code OAuth token을 읽어 쓴다.
  5. 요청이 Claude Code OAuth 요청으로 인식되면 Claude Code 구독 한도에서 사용된다.
  6. 실패하면 fallback으로 openai-codex / gpt-5.5를 쓰게 해둔다.

중요한 차이
ANTHROPIC_API_KEY를 쓰면 Claude Code 구독이 아니라 Anthropic API 과금 경로다. 특히 환경변수에  ANTHROPIC_API_KEY가 잡혀 있으면 Claude Code 구독 로그인보다 API key가 우선될 수 있다. 그래서 “구독을 쓰고 싶다”면 API key 기반 설정과 OAuth 기반 설정을 혼동하면 안 된다.

 왜 패치가 필요한가
Claude Code OAuth token만 있다고 끝이 아니다. Anthropic 쪽은 요청이 실제 Claude Code류 요청인지도 본다. 그래서 Hermes가 그냥 일반 Anthropic API 요청처럼 보내면 다음 문제가 생길 수 있다.

  - 인증은 있는데 Claude Code 요청 형태가 아니라서 거부됨
  - third-party usage 또는 extra usage 쪽으로 라우팅됨
  - tool name, system prompt, beta flag, SDK header 구조가 달라서 400이 남
  - Keychain에 만료된 토큰이 남아 있으면 유효한 파일 토큰까지 못 내려감

  우리 환경의 patch는 두 가지 역할을 한다.

  첫째, Claude Code 요청 형태에 맞추는 호환 레이어다.
  Hermes 요청의 system prompt, tool naming, beta flag, SDK 계열 header 구조를 Claude Code OAuth 요청이 기대하는 형태에 가깝게 바꾼다.

  둘째, credential 선택 문제를 고친다.
  macOS Keychain에 만료됐고 refresh 불가능한 Claude token이 있으면 그걸 버리고  ~/.claude/.credentials.json의 유효한 refreshable token으로 fallback한다.

  현재 ops 구성

  model:
    provider: anthropic
    default: claude-opus-4-7

  fallback_providers:
    - provider: openai-codex
      model: gpt-5.5

  운영 시 주의점

  - Claude Code 구독은 무제한이 아니다. Claude와 Claude Code 사용량이 같은 구독 한도를 공유한다.
  - 429가 뜨면 설정 오류가 아니라 실제 구독 한도/속도 제한일 수 있다.
  - claude-opus-4-7:1m처럼 큰 컨텍스트 옵션이나 병렬 subagent는 한도를 빠르게 소모한다.
  - Hermes 업데이트는 본체 repo를 덮지만, 우리 patch는 /Users/deck/.hermes/patches에 있어서 일반 update에는 유지된다.
  - 완전 재설치로 venv의 sitecustomize.py가 사라지면 patch loader만 다시 설치해야 한다.
  - 이 방식은 공식 Claude Code CLI 사용이 아니라 비공식 호환 방식이므로, 다른 사람에게 줄 때는 “작동은 하지만 깨질 수 있고 약관/정책 리스크가 있다”고 설명하는 게 맞다.
