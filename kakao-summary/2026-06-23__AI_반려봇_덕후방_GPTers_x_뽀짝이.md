# AI 요약

## 인사이트·노하우

### OpenClaw vs Hermes 비교
- **OpenClaw**: 지시한 작업을 정석으로 수행하는 느낌
- **Hermes**: 반복 작업 중 스스로 체계를 만들고 지시하지 않은 부분까지 최적화하는 느낌
- 병행 사용 팁: 한쪽이 먹통일 때 다른 쪽으로 수리 가능 (상호 폴백)

### OpenClaw 작업 완료 알림 미응답 해결법
- 문제: 서브에이전트 작업 완료 후 텔레그램 응답을 안 보내는 경우
- 해결 1: 잔소리(재요청)
- 해결 2: 확인용 크론을 서브에이전트에게 걸게 해서 완료 확인 시 크론 종료 후 알림 발송

### OpenClaw + Playwright CDP 브라우저 크롤링 구조 (WSL 환경)
- 핵심: 새 Chrome을 띄우는 것이 아니라 **이미 떠 있는 Windows Chrome의 CDP 포트에 붙는 방식**
- 포트 구성:
  - Windows Chrome 로컬 CDP: `127.0.0.1:9334`
  - Windows portproxy/bridge → WSL 접속: `10.95.xxx.xxx:9444`
  - Edge는 9335 → 9445
- 연결 코드 패턴:
  ```js
  const browser = await chromium.connectOverCDP(CDP_URL, { timeout: 120000 });
  const context = browser.contexts()[0] || await browser.newContext();
  ```
- 연결 확인: `curl http://10.95.xxx.xxx:9444/json/version` 및 `/json/list`
- 캡처 방식보다 빠르나, 너무 빨라서 로딩 전 클릭 발생 → 지연 시간 추가 필요
- 작업 확인 시 일반 Chrome, 백그라운드 작업 시 Playwright headless 사용

### 크론 전략 최적화
- 단순 반복 크롤링은 OpenClaw 크론 대신 **리눅스 crontab**으로 직접 등록하는 것이 토큰 절약에 유리
- 크론탭 24개 동시 실행해도 토큰 거의 미소모
- 크론탭 대시보드를 OpenClaw에게 만들게 하고, Tailscale로 포트 열어 외부에서 폰/노트북으로 접속 가능하게 구성

### 클로드 브라우저 크롤링 대안
- Claude Cowork + Chrome Extension 조합으로 이미 열려 있는 브라우저를 직접 활용한 크롤링 가능

### Codex 사용량 확인 방법
- 사용량 확인 URL: `https://chatgpt.com/codex/cloud/settings/analytics#usage`
- 기존 ChatGPT Plus 세션과 꼬이면 사용량 초과 오류처럼 보일 수 있음 → 해당 URL에서 실제 사용량 확인

### 중국 AI 모델 보안 우려 완화법
- Ollama Cloud나 HuggingFace 등 **미국 서버 중계**를 통해 사용하면 직접 연결 대비 보안 리스크 감소
- AI 학습용 데이터는 정제된 대량 데이터를 사용하므로, 사용자 입력 데이터가 학습에 활용될 가능성은 낮다는 의견 (단, 완전히 배제할 수는 없으므로 감안하고 사용 권장)

### 클로드 서버 장애 대응

## 자동화·시스템 적용 후보

- **크론탭 + 대시보드 패턴**: 단순 반복 작업은 OpenClaw 크론 대신 리눅스 crontab으로 등록 → OpenClaw가 대시보드만 생성 → Tailscale로 외부 접근. 토큰 절약 + 안정성 향상
- **Playwright CDP headless 전환 패턴**: 개발/디버깅 시 CDP로 화면 보면서 작업 → 완성 후 headless 스킬로 전환 → crontab 등록. 시각적 확인과 자동화를 단계적으로 분리하는 워크플로
- **분석 에이전트에 도메인 지식 사전 학습**: 과거 강의 자료·프로젝트 자료·전문 블로그 글을 에이전트에 학습시켜 텍소노미 설계·태깅·보고서 자동 생성에 활용한 사례 (P92 "또롱이")
- **Hermes Slack 연동 시 매니페스트 활용**: `hermes manifest` 명령으로 매니페스트를 추출해 Slack 앱 등록에 사용하면 설정 번거로움 감소
