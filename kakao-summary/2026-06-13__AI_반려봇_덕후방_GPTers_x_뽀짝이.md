# AI 요약

## 인사이트·노하우

- **Fable 5 / Mythos 5 서비스 중단 경위**: 미국 정부가 "특정 코드베이스를 읽고 소프트웨어 결함을 수정하는 제한적 탈옥 가능성"을 이유로 서비스 중단 지시 → Anthropic은 불복 의사를 밝히면서도 법적 지침을 준수해 전 세계 접속을 차단. "하나의 잠재적 탈옥 사례만으로 수억 명에 배포된 상용 모델을 회수해야 할 이유에 동의할 수 없다"는 공식 입장. 복구 시점 미정.

- **Claude 토큰 예고 없이 리셋 후 즉시 Fable 차단**: 2026-06-13 오전 토큰이 갑작스럽게 리셋되었다가 약 30분 뒤 `claude-fable-5` 모델 오류 발생. 리셋과 서비스 차단이 거의 동시에 일어나 실용적 혜택이 없었음. **교훈**: "공지 믿지 말고 주어진 토큰 사용을 내일로 미루지 마라."

- **컨텍스트가 길어지면 Opus 4.8 → 1M 컨텍스트 모델로 자동 전환**: Claude Code 앱에서 세션이 길어지면 자동으로 Opus 4.8 (1M 컨텍스트) 버전으로 전환됨. Fable 차단 이후 해당 전환이 더 이상 동작하지 않는다는 보고 있음. 대응으로 "세션이 길어지면 다른 세션에서 이어받는 명령어"를 별도로 구성해두는 방식을 사용 중.

- **자가개선 루프(Loop Engineering)의 현실적 한계**:
  - 토큰이 병목: Max×5 사용자도 멀티 윈도우로 1~2시간이면 한도 도달.
  - 루프가 돌수록 데이터가 무거워져 관리 자체가 어려워짐.
  - 실용적 운용 방식: "실험을 여러 번 돌려 휴먼이 채택하는 반자동 루프"가 현실적 최선.
  - 개인/소규모 팀에서 완전 자동화 루프는 "앤트로픽 수준의 토큰 무제한이 전제되어야 의미 있다"는 공통 의견.

- **Human-in-the-Loop는 병목이 아니라 품질의 핵심**: 특히 정성평가가 중요한 작업에서는 휴먼 검수를 제거하면 오히려 문제를 단순화하는 경향이 있음. "AI를 신뢰하지 말고, 시스템을 신뢰할 수 있도록 구조화하라."

- **브루(Vrew) 대체 워크플로우**: 코딩 에이전트(Claude Code 등)에 Whisper 엔진이 내장되어 있어 촬영본을 넣으면 오타·문단 의미·화면 비율에 맞는 글자수·타임코드를 고려한 SRT 자막을 자동 생성 가능 → 브루의 무거운 2차가공 문제를 해결. 인터뷰·정보성 영상에서 "자막 작업을 5분 만에 해결"하는 수준. 다큐멘터리처럼 여러 인터뷰를 조합하는 워크플로우에서는 며칠 치 작업을 단축. 단, **NLE 후반 편집은 여전히 전통 방식 유지**가 실무 권장.

- **워크플로우 마이그레이션은 빠름**: 플랫폼 간 워크플로우 마이그레이션은 Codex·Claude로 "금방" 처리 가능하다는 경험담.

- **Hermes SNS 연동 채널 선택 기준**:
  - 단일 채널이라면 텔레그램으로 충분.
  - 봇끼리 협업·대화가 필요하다면 슬랙이 유리 (텔레그램도 봇 간 지시는 가능하나 UI 표현이 안 됨).
  - 텔레그램은 정보 접근성이 높아 정보 전달 용도로도 편리.

- **Brainstorming·Work-plan 단계에서 토큰 과소비 발생**: SuperPowers 플러그인의 brainstorming·work-plan 단계가 가장 많이 소모. 대안으로 Matt Pocock의 `grill-me` 스킬, `handoff` 스킬 사용 권장 (handoff는 그대로 써도 좋다는 평가).

- **LLM Wiki의 구조적 한계**: 방대해질수록 관리 자체가 어려워지고 토큰 소모도 증가. 구조적 개선이 필요한 시점이 반드시 옴.

## 반응이 컸던 화제

- **Fable 5 서비스 차단**: 토큰 리셋과 거의 동시에 서비스가 끊기면서 다수가 실시간으로 피해를 공유. Anthropic의 공식 입장 전문이 채팅방에 공유되며 "억울한 면이 있다", "기준이 업계 전반에 적용되면 신규 모델 배포가 사실상 중단"이라는 내용이 화제가 됨. Kimi 2.7이 Fable을 증류했다는 근거 없는 추정도 나와 분위기를 이어감.

- **루프 엔지니어링의 개인 효용성 논쟁**: "앤트로픽 수준이 아니면 의미 없다"는 회의론과 "하네스·스킬 수준의 루프는 개인도 가능하다"는 의견이 맞붙음. 토큰 비용 문제, 데이터 비대화 문제, 휴먼 인 더 루프 불가피성 등 실질적 고민이 많이 공유됨.

- **코딩 에이전트 기반 영상 자막 자동화**: "브루는 이미 대체되었다", "인터뷰 기반 영상 자막을 5분에 해결", "다큐멘터리 며칠 치 작업을 단축"이라는 구체적 경험담이 영상 제작자들의 적극 질문을 유발.

## 자동화·시스템 적용 후보

- **게임 프로젝트 CLAUDE.md 자가개선 패턴**: Fable에게 기존 지침을 검토해 "시행착오를 줄이고 더 좋은 게임을 만들 수 있도록 지침을 업데이트해달라"고 요청 → 이후 일자별 프롬프트 로그를 사용자별(연우/선우) 폴더로 분리해 커밋에 함께 포함. **적용 포인트**: 장기 프로젝트에서 CLAUDE.md를 AI가 직접 개선하게 하고, 프롬프트 로그를 버전 관리하는 패턴.

- **Kenney 에셋 자동 색인 + 게임 자동 적용**: Kenney.nl 13페이지 분량의 에셋을 Claude Code로 스캔·색인해두고, 게임 제작 시 별도 요청 없이 자동으로 적합한 사운드를 삽입하는 워크플로우. **적용 포인트**: 외부 리소스 사이트를 색인화해 에이전트가 맥락에 맞게 자율 활용하는 패턴.

- **코딩 에이전트로 SRT 자막 자동 생성**: 촬영본 → Whisper 내장 에이전트 → 오타·의미·글자수·타임코드 고려한 SRT 출력. 브루 없이도 자막 완성 가능. 인터뷰·정보성 영상 제작 파이프라인에 바로 적용 가능.

- **SkillOpt 방식의 스킬 자가개선 루프**: AI가 스킬 사용의 성공/실패를 분석해 스킬 문서를 스스로 개선. MS가 오픈소스로 공개. 완전 자동화보다는 "여러 실험 결과 중 휴먼이 채택"하는 반자동 방식이 현실적.

## 질문·미해결

- **OpenClaw 연동 Claude 토큰이 플랜 차감이 아닌 Agent SDK 크레딧($100)에서 차감된다는 정책 변경**: 자세한 내용을 아는 사람이 채팅방에서 확인되지 않음. 후속 조사 필요.

- **Max×5 vs Max 20, 혹은 GPT Pro + Claude Max×5 조합 중 어느 것이 더 효율적인가**: 채팅방에서 Max 20 쪽으로 의견이 모였으나 실제 비교 데이터는 공유되지 않음.
- **롱폼 영상 편집 AI 에이전트 사례**: 질문이 나왔고 일부 경험자가 있었으나 구체적 파이프라인·도구 스택이 완전히 공개되지 않음. After Effects 위주 사용 중이라는 단편적 언급만 있음.
