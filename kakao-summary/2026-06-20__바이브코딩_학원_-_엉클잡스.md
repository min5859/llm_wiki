# AI 요약

## 인사이트·노하우

- **에이전트 권한 경계 원칙**: 에이전트에게 작업을 맡길수록 "무엇을 하게 할지"보다 "어디까지 못 하게 할지"가 운영 실력. Claude Code/Cursor/Codex/MCP 도구 점검 체크리스트: 읽기 전용으로 가능한가, 실행 전 사람 승인이 켜져 있는가.

- **AI 에이전트 + 금융 자동화 위험**: AI에게 거래 권한을 줄 때 기본 가드 3가지 — ① 읽기 전용 모드 우선, ② 소액 전용 계정 분리, ③ 주문/출금은 사람 승인 단계 삽입. 국내 업비트 Upbit Skills, 빗썸 AI Trade Kit 모두 공식적으로 사람 확인 단계 권장.

- **웹 브라우징 에이전트의 로컬 RCE 위험 (AutoJack)**: Microsoft 연구 — AutoGen Studio 개발 빌드에서 웹페이지를 읽는 AI 에이전트가 로컬 MCP WebSocket에 접근해 원격 코드 실행(RCE)으로 이어질 수 있음. "로컬이라 안전하다"는 전제 무효. PyPI 공개 릴리스에는 미포함, upstream main branch에서 수정 완료. 결제·발송·삭제 권한 자동화 시 승인 단계 및 권한 제한 필수.

- **Claude Code Artifacts (베타)**: 2026-06-18 Team/Enterprise 이용자 공개. 세션 결과를 조직 내부 공유 가능한 비공개 URL의 라이브 페이지로 변환, 작업 진행 시 같은 URL 업데이트. 제약: 단일 페이지 캡처이며 백엔드/실시간 API 호출 불가. 활용 적합: 결과 보고서, 변경 설명, 장애 조사 타임라인 공유.

- **AI 글쓰기 도구 - 변경점 추적**: AI에게 글을 수정 요청하면 요청하지 않은 문장까지 슬쩍 바꾸는 문제 → Flashtype은 AI가 바꾼 곳을 한 줄씩 짚어주고 수락 여부를 사람이 선택(개발자 코드 리뷰 방식을 글 편집에 적용). 브라우저 기반 마크다운 에디터, 무료.
