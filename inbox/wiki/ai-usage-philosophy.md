# AI 사용법 철학 — 언어화 능력·컨텍스트 설계·리버럴 아츠

AI를 잘 사용한다는 것이 무엇인지에 대한 철학적 논의. 기술적 스킬보다 "무엇을 묻는가"를 결정하는 사고력이 핵심이다.

## 핵심 내용

### AI 사용에 정답은 없다

AI 활용법은 개인의 맥락, 도메인, 목적에 따라 달라진다. "어떤 프롬프트가 최고다"는 식의 정답은 없으며, 자신의 상황에 맞게 실험하고 조정하는 과정이 본질이다.

### 언어화 능력

AI에게 좋은 결과를 얻으려면 자신이 원하는 것을 정확히 언어로 표현할 수 있어야 한다.

- "좋은 코드 짜줘" → 모호함, 결과 불예측
- "Go 언어로, 에러 처리는 errors.Is()를 사용하고, 단위 테스트를 포함한, Redis 캐싱을 적용한 HTTP 핸들러를 작성해줘" → 구체적, 예측 가능

> 언어화 능력은 AI 시대에 가장 중요한 핵심 역량이다. 생각을 언어로 변환하는 훈련이 필요하다.

강의 녹취에서는 "모르는 것을 AI에게 물어보지 못하는" 문제가 반복적으로 언급된다. 핵심은 지식 부족 자체보다, 자신이 무엇을 모르는지 문장으로 표현하지 못하는 상태다. Claude Code 같은 에이전트 도구는 모호한 요청도 어느 정도 보정하지만, 업무 맥락·제약·완료 조건을 말로 꺼내는 능력이 결과 품질을 좌우한다.

### 컨텍스트 설계

AI가 잘 작동하려면 충분한 맥락(Context)이 필요하다. 컨텍스트 설계는 "AI가 올바른 판단을 내릴 수 있는 환경을 만드는 것"이다.

**좋은 컨텍스트의 3요소:**

1. **배경** — 왜 이 작업을 하는가
2. **제약** — 무엇을 하면 안 되는가
3. **목표** — 최종 결과물이 어떤 상태이어야 하는가

## 세부 사항

### 리버럴 아츠 (Liberal Arts)

AI 시대에 기술 지식 못지않게 인문학·사회과학·예술 등 폭넓은 교양이 중요해진다.

- AI가 실행은 해줄 수 있지만 "무엇을 실행할지"는 사람이 결정
- 좋은 판단력, 비판적 사고, 윤리 감각은 AI가 대체할 수 없음
- 도메인 지식 + AI 활용 능력의 조합이 경쟁력

### AI 활용 수준 분류

| 수준 | 특징 |
|---|---|
| 초급 | 질문-답변 패턴, AI를 검색엔진처럼 사용 |
| 중급 | 맥락 제공, 반복 개선, 역할 부여 |
| 고급 | 에이전트 설계, 파이프라인 구축, 하네스 엔지니어링 |

### 실패하는 패턴

- **과도한 의존**: AI가 틀렸는데 검증하지 않고 사용
- **맥락 생략**: 배경 없이 결과만 요청
- **정답 찾기**: "최고의 프롬프트"를 추구하다 실험을 포기

### 잘 활용하는 패턴

- **반복 실험**: 프롬프트를 조금씩 바꾸며 결과 비교
- **역할 부여**: "시니어 백엔드 엔지니어로서 검토해줘"
- **비판적 수용**: AI 결과를 비판적으로 검토하고 수정
- **축적**: 좋은 프롬프트와 결과를 기록하고 재사용

## 관련 맥락

- 이 철학은 Claude Code 같은 고급 도구를 사용할 때 더욱 중요해짐
- 기술 도구는 빠르게 바뀌지만 언어화 능력·판단력은 장기적 자산
- "AI를 어떻게 사용하는가"는 점점 "어떻게 생각하는가"와 동의어가 되고 있음
